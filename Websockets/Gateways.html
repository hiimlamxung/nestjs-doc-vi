Gateways

Hầu hết các khái niệm đã được thảo luận ở các phần khác trong tài liệu này, chẳng hạn như dependency injection, decorators, exception filters, pipes, guards và interceptors, đều áp dụng tương tự cho gateways. Bất cứ khi nào có thể, Nest trừu tượng hóa các chi tiết triển khai để các component giống nhau có thể chạy trên các nền tảng dựa trên HTTP, WebSockets và Microservices. Phần này đề cập đến các khía cạnh của Nest dành riêng cho WebSockets.

Trong Nest, một gateway đơn giản là một class được đánh dấu bằng decorator @WebSocketGateway(). Về mặt kỹ thuật, gateways không phụ thuộc vào nền tảng (platform-agnostic) khiến chúng tương thích với bất kỳ thư viện WebSockets nào sau khi một adapter được tạo. Có hai nền tảng WS được hỗ trợ sẵn: socket.io và ws. Bạn có thể chọn nền tảng phù hợp nhất với nhu cầu của mình. Ngoài ra, bạn có thể xây dựng adapter riêng bằng cách làm theo hướng dẫn này.

Gợi ý

Gateways có thể được coi là providers; điều này có nghĩa là chúng có thể inject các dependencies thông qua constructor của class. Ngoài ra, gateways cũng có thể được inject bởi các class khác (providers và controllers).

Cài đặt #

Để bắt đầu xây dựng các ứng dụng dựa trên WebSockets, trước tiên hãy cài đặt package cần thiết:

bash 
$ npm i --save @nestjs/websockets @nestjs/platform-socket.io

Tổng quan #

Nói chung, mỗi gateway lắng nghe trên cùng một cổng với HTTP server, trừ khi ứng dụng của bạn không phải là ứng dụng web, hoặc bạn đã thay đổi cổng thủ công. Hành vi mặc định này có thể được sửa đổi bằng cách truyền một đối số vào decorator @WebSocketGateway(80) trong đó 80 là số cổng được chọn. Bạn cũng có thể thiết lập một namespace được sử dụng bởi gateway bằng cách sử dụng cấu trúc sau:

TypeScript 
@WebSocketGateway(80, { namespace: 'events' })
Cảnh báo

Gateways không được khởi tạo cho đến khi chúng được tham chiếu trong mảng providers của một module hiện có.

Bạn có thể truyền bất kỳ tùy chọn được hỗ trợ nào vào socket constructor bằng đối số thứ hai của decorator @WebSocketGateway(), như được hiển thị dưới đây:

TypeScript 
@WebSocketGateway(81, { transports: ['websocket'] })

Gateway hiện đã lắng nghe, nhưng chúng ta vẫn chưa đăng ký bất kỳ tin nhắn đến nào. Hãy tạo một handler sẽ đăng ký nhận tin nhắn events và phản hồi cho người dùng với cùng dữ liệu đó.

events.gateway 
@SubscribeMessage('events')
handleEvent(@MessageBody() data: string): string {
  return data;
}
@Bind(MessageBody())
@SubscribeMessage('events')
handleEvent(data) {
  return data;
}
Gợi ý

Các decorator @SubscribeMessage() và @MessageBody() được import từ package @nestjs/websockets.

Sau khi gateway được tạo, chúng ta có thể đăng ký nó trong module.

events.module 
import { Module } from '@nestjs/common';
import { EventsGateway } from './events.gateway';

@Module({
  providers: [EventsGateway]
})
export class EventsModule {}

Bạn cũng có thể truyền một property key vào decorator để trích xuất nó từ body của tin nhắn đến:

events.gateway 
@SubscribeMessage('events')
handleEvent(@MessageBody('id') id: number): number {
  // id === messageBody.id
  return id;
}
@Bind(MessageBody('id'))
@SubscribeMessage('events')
handleEvent(id) {
  // id === messageBody.id
  return id;
}

Nếu bạn không muốn sử dụng decorators, đoạn code sau có chức năng tương đương:

events.gateway 
@SubscribeMessage('events')
handleEvent(client: Socket, data: string): string {
  return data;
}
@SubscribeMessage('events')
handleEvent(client, data) {
  return data;
}

Trong ví dụ trên, hàm handleEvent() nhận hai đối số. Đối số đầu tiên là một socket instance dành riêng cho nền tảng, trong khi đối số thứ hai là dữ liệu nhận được từ client. Tuy nhiên, cách tiếp cận này không được khuyến khích vì nó yêu cầu mock socket instance trong mỗi unit test.

Sau khi tin nhắn events được nhận, handler sẽ gửi một acknowledgment với cùng dữ liệu đã được gửi qua mạng. Ngoài ra, có thể phát (emit) tin nhắn bằng cách tiếp cận dành riêng cho thư viện, ví dụ, bằng cách sử dụng phương thức client.emit(). Để truy cập socket instance đã kết nối, hãy sử dụng decorator @ConnectedSocket().

events.gateway 
@SubscribeMessage('events')
handleEvent(
  @MessageBody() data: string,
  @ConnectedSocket() client: Socket,
): string {
  return data;
}
@Bind(MessageBody(), ConnectedSocket())
@SubscribeMessage('events')
handleEvent(data, client) {
  return data;
}
Gợi ý

Decorator @ConnectedSocket() được import từ package @nestjs/websockets.

Tuy nhiên, trong trường hợp này, bạn sẽ không thể tận dụng interceptors. Nếu bạn không muốn phản hồi cho người dùng, bạn có thể đơn giản bỏ qua câu lệnh return (hoặc trả về một giá trị "falsy" một cách rõ ràng, ví dụ undefined).

Bây giờ khi client phát tin nhắn như sau:

TypeScript 
socket.emit('events', { name: 'Nest' });

Phương thức handleEvent() sẽ được thực thi. Để lắng nghe các tin nhắn được phát từ handler trên, client phải đính kèm một acknowledgment listener tương ứng:

TypeScript 
socket.emit('events', { name: 'Nest' }, (data) => console.log(data));

Mặc dù việc trả về giá trị từ message handler ngầm gửi một acknowledgement, các tình huống nâng cao thường yêu cầu kiểm soát trực tiếp acknowledgement callback.

Decorator tham số @Ack() cho phép inject hàm callback ack trực tiếp vào message handler.

events.gateway 
@SubscribeMessage('events')
handleEvent(
  @MessageBody() data: string,
  @Ack() ack: (response: { status: string; data: string }) => void,
) {
  ack({ status: 'received', data });
}
@Bind(MessageBody(), Ack())
@SubscribeMessage('events')
handleEvent(data, ack) {
  ack({ status: 'received', data });
}

Nhiều phản hồi #

Acknowledgment chỉ được gửi một lần. Hơn nữa, nó không được hỗ trợ bởi triển khai WebSockets gốc. Để giải quyết hạn chế này, bạn có thể trả về một object bao gồm hai thuộc tính. event là tên của sự kiện được phát và data là dữ liệu cần được chuyển tiếp đến client.

events.gateway 
@SubscribeMessage('events')
handleEvent(@MessageBody() data: unknown): WsResponse<unknown> {
  const event = 'events';
  return { event, data };
}
@Bind(MessageBody())
@SubscribeMessage('events')
handleEvent(data) {
  const event = 'events';
  return { event, data };
}
Gợi ý

Interface WsResponse được import từ package @nestjs/websockets.

Cảnh báo

Bạn nên trả về một class instance implement WsResponse nếu trường data của bạn dựa vào ClassSerializerInterceptor, vì nó bỏ qua các phản hồi là plain JavaScript object.

Để lắng nghe (các) phản hồi đến, client phải áp dụng một event listener khác.

TypeScript 
socket.on('events', (data) => console.log(data));

Phản hồi bất đồng bộ #

Message handlers có thể phản hồi đồng bộ hoặc bất đồng bộ. Do đó, các phương thức async được hỗ trợ. Một message handler cũng có thể trả về một Observable, trong trường hợp đó các giá trị kết quả sẽ được phát cho đến khi stream hoàn thành.

events.gateway 
@SubscribeMessage('events')
onEvent(@MessageBody() data: unknown): Observable<WsResponse<number>> {
  const event = 'events';
  const response = [1, 2, 3];

  return from(response).pipe(
    map(data => ({ event, data })),
  );
}
@Bind(MessageBody())
@SubscribeMessage('events')
onEvent(data) {
  const event = 'events';
  const response = [1, 2, 3];

  return from(response).pipe(
    map(data => ({ event, data })),
  );
}

Trong ví dụ trên, message handler sẽ phản hồi 3 lần (với mỗi phần tử trong mảng).

Lifecycle hooks #

Có 3 lifecycle hooks hữu ích có sẵn. Tất cả đều có interface tương ứng và được mô tả trong bảng sau:

OnGatewayInit Bắt buộc implement phương thức afterInit(). Nhận server instance dành riêng cho thư viện làm đối số (và spread phần còn lại nếu cần).
OnGatewayConnection Bắt buộc implement phương thức handleConnection(). Nhận client socket instance dành riêng cho thư viện làm đối số.
OnGatewayDisconnect Bắt buộc implement phương thức handleDisconnect(). Nhận client socket instance dành riêng cho thư viện làm đối số.
Gợi ý

Mỗi lifecycle interface được export từ package @nestjs/websockets.

Server và Namespace #

Đôi khi, bạn có thể muốn truy cập trực tiếp vào server instance gốc, dành riêng cho nền tảng. Tham chiếu đến object này được truyền làm đối số cho phương thức afterInit() (interface OnGatewayInit). Một tùy chọn khác là sử dụng decorator @WebSocketServer().

TypeScript 
@WebSocketServer()
server: Server;

Ngoài ra, bạn có thể truy xuất namespace tương ứng bằng cách sử dụng thuộc tính namespace:

TypeScript 
@WebSocketGateway({ namespace: 'my-namespace' })
export class EventsGateway {
  @WebSocketServer()
  namespace: Namespace;
}
Lưu ý

Decorator @WebSocketServer() được import từ package @nestjs/websockets.

Nest sẽ tự động gán server instance cho thuộc tính này khi nó sẵn sàng sử dụng.
