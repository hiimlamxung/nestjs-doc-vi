Adapters

Module WebSockets là platform-agnostic, do đó bạn có thể sử dụng thư viện riêng của mình (hoặc thậm chí là một native implementation) bằng cách sử dụng interface WebSocketAdapter. Interface này bắt buộc phải implement một số method được mô tả trong bảng sau:

createTạo một socket instance dựa trên các tham số được truyền vào
bindClientConnectBind sự kiện kết nối của client
bindClientDisconnectBind sự kiện ngắt kết nối của client (tùy chọn*)
bindMessageHandlersBind message đến với message handler tương ứng
closeĐóng một server instance

Extend socket.io #

Package socket.io được bọc trong class IoAdapter. Nếu bạn muốn nâng cao chức năng cơ bản của adapter thì sao? Ví dụ, yêu cầu kỹ thuật của bạn cần khả năng broadcast event qua nhiều load-balanced instance của web service. Để làm điều này, bạn có thể extend IoAdapter và override một method duy nhất có nhiệm vụ khởi tạo các socket.io server mới. Nhưng trước tiên, hãy cài đặt package cần thiết.

⚠ Warning

Để sử dụng socket.io với nhiều load-balanced instance, bạn phải tắt polling bằng cách đặt transports: ['websocket'] trong cấu hình socket.io của client hoặc bạn phải bật cookie based routing trong load balancer. Chỉ dùng Redis là không đủ. Xem tại đây để biết thêm thông tin.

Terminal
$ npm i --save redis socket.io @socket.io/redis-adapter

Sau khi package được cài đặt, chúng ta có thể tạo class RedisIoAdapter.

redis-io-adapter.ts
import { IoAdapter } from '@nestjs/platform-socket.io';
import { ServerOptions } from 'socket.io';
import { createAdapter } from '@socket.io/redis-adapter';
import { createClient } from 'redis';

export class RedisIoAdapter extends IoAdapter {
  private adapterConstructor: ReturnType<typeof createAdapter>;

  async connectToRedis(): Promise<void> {
    const pubClient = createClient({ url: `redis://localhost:6379` });
    const subClient = pubClient.duplicate();

    await Promise.all([pubClient.connect(), subClient.connect()]);

    this.adapterConstructor = createAdapter(pubClient, subClient);
  }

  createIOServer(port: number, options?: ServerOptions): any {
    const server = super.createIOServer(port, options);
    server.adapter(this.adapterConstructor);
    return server;
  }
}

Sau đó, chỉ cần chuyển sang Redis adapter mới tạo.

main.ts
const app = await NestFactory.create(AppModule);
const redisIoAdapter = new RedisIoAdapter(app);
await redisIoAdapter.connectToRedis();

app.useWebSocketAdapter(redisIoAdapter);

Ws library #

Một adapter khác có sẵn là WsAdapter, hoạt động như một proxy giữa framework và tích hợp thư viện ws cực kỳ nhanh và được kiểm thử kỹ lưỡng. Adapter này tương thích hoàn toàn với native browser WebSockets và nhanh hơn nhiều so với package socket.io. Tuy nhiên, nó có ít chức năng có sẵn hơn đáng kể. Trong một số trường hợp, bạn không nhất thiết cần chúng.

💡 Hint

Thư viện ws không hỗ trợ namespace (kênh giao tiếp được phổ biến bởi socket.io). Tuy nhiên, để mô phỏng phần nào tính năng này, bạn có thể mount nhiều ws server trên các path khác nhau (ví dụ: @WebSocketGateway({ path: '/users' })).

Để sử dụng ws, trước tiên chúng ta phải cài đặt package cần thiết:

Terminal
$ npm i --save @nestjs/platform-ws

Sau khi package được cài đặt, chúng ta có thể chuyển đổi adapter:

main.ts
const app = await NestFactory.create(AppModule);
app.useWebSocketAdapter(new WsAdapter(app));
💡 Hint

WsAdapter được import từ @nestjs/platform-ws.

WsAdapter được thiết kế để xử lý message theo định dạng { event: string, data: any }. Nếu bạn cần nhận và xử lý message ở định dạng khác, bạn sẽ cần cấu hình một message parser.

main.ts
const wsAdapter = new WsAdapter(app, {
  messageParser: (data) => {
    const [event, payload] = JSON.parse(data.toString());
    return { event, data: payload };
  },
});

Ngoài ra, bạn cũng có thể cấu hình message parser sau khi adapter được tạo bằng cách sử dụng method setMessageParser.

Advanced (custom adapter) #

Để minh họa, chúng ta sẽ tích hợp thư viện ws thủ công. Như đã đề cập, adapter cho thư viện này đã được tạo sẵn và được expose từ package @nestjs/platform-ws dưới dạng class WsAdapter. Đây là cách mà implementation đơn giản hóa có thể trông như thế nào:

ws-adapter.ts
TS
JS
import * as WebSocket from 'ws';
import { WebSocketAdapter, INestApplicationContext } from '@nestjs/common';
import { MessageMappingProperties } from '@nestjs/websockets';
import { Observable, fromEvent, EMPTY } from 'rxjs';
import { mergeMap, filter } from 'rxjs/operators';

export class WsAdapter implements WebSocketAdapter {
  constructor(private app: INestApplicationContext) {}

  create(port: number, options: any = {}): any {
    return new WebSocket.Server({ port, ...options });
  }

  bindClientConnect(server, callback: Function) {
    server.on('connection', callback);
  }

  bindMessageHandlers(
    client: WebSocket,
    handlers: MessageMappingProperties[],
    process: (data: any) => Observable<any>,
  ) {
    fromEvent(client, 'message')
      .pipe(
        mergeMap(data => this.bindMessageHandler(data, handlers, process)),
        filter(result => result),
      )
      .subscribe(response => client.send(JSON.stringify(response)));
  }

  bindMessageHandler(
    buffer,
    handlers: MessageMappingProperties[],
    process: (data: any) => Observable<any>,
  ): Observable<any> {
    const message = JSON.parse(buffer.data);
    const messageHandler = handlers.find(
      handler => handler.message === message.event,
    );
    if (!messageHandler) {
      return EMPTY;
    }
    return process(messageHandler.callback(message.data));
  }

  close(server) {
    server.close();
  }
}
import * as WebSocket from 'ws';

export class WsAdapter {
  constructor(app) {
    this.app = app;
  }

  create(port, options = {}) {
    return new WebSocket.Server({ port, ...options });
  }

}
