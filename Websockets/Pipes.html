Pipes

Không có sự khác biệt cơ bản nào giữa pipes thông thường và web sockets pipes. Sự khác biệt duy nhất là thay vì throw HttpException, bạn nên sử dụng WsException. Ngoài ra, tất cả pipes sẽ chỉ được áp dụng cho tham số data (vì việc validating hoặc transforming client instance là không cần thiết).

Gợi ý

Class WsException được export từ package @nestjs/websockets.

Binding pipes #

Ví dụ sau sử dụng method-scoped pipe được khởi tạo thủ công. Giống như với các ứng dụng HTTP, bạn cũng có thể sử dụng gateway-scoped pipes (tức là đặt trước gateway class với decorator @UsePipes()).

TS
JS
@UsePipes(new ValidationPipe({ exceptionFactory: (errors) => new WsException(errors) }))
@SubscribeMessage('events')
handleEvent(client: Client, data: unknown): WsResponse<unknown> {
  const event = 'events';
  return { event, data };
}
