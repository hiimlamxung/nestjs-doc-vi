Mapped types

Cảnh báo

Chương này chỉ áp dụng cho phương pháp code first.

Khi bạn xây dựng các tính năng như CRUD (Create/Read/Update/Delete), việc xây dựng các biến thể trên một entity type cơ sở thường rất hữu ích. Nest cung cấp một số hàm tiện ích thực hiện các phép biến đổi type để giúp tác vụ này thuận tiện hơn.

Partial #

Khi xây dựng các input validation type (còn gọi là Data Transfer Object hay DTO), việc xây dựng các biến thể create và update trên cùng một type thường rất hữu ích. Ví dụ, biến thể create có thể yêu cầu tất cả các trường, trong khi biến thể update có thể làm tất cả các trường trở thành tùy chọn.

Nest cung cấp hàm tiện ích PartialType() để giúp tác vụ này dễ dàng hơn và giảm thiểu boilerplate.

Hàm PartialType() trả về một type (class) với tất cả các thuộc tính của input type được đặt thành tùy chọn. Ví dụ, giả sử chúng ta có một create type như sau:

TypeScript
@InputType()
class CreateUserInput {
  @Field()
  email: string;

  @Field()
  password: string;

  @Field()
  firstName: string;
}

Mặc định, tất cả các trường này đều là bắt buộc. Để tạo một type với cùng các trường, nhưng mỗi trường đều là tùy chọn, sử dụng PartialType() và truyền tham chiếu class (CreateUserInput) làm đối số:

TypeScript
@InputType()
export class UpdateUserInput extends PartialType(CreateUserInput) {}
Gợi ý

Hàm PartialType() được import từ package @nestjs/graphql.

Hàm PartialType() nhận một đối số thứ hai tùy chọn là tham chiếu đến một decorator factory. Đối số này có thể được sử dụng để thay đổi hàm decorator được áp dụng cho class kết quả (class con). Nếu không được chỉ định, class con sẽ sử dụng cùng decorator như class cha (class được tham chiếu trong đối số đầu tiên). Trong ví dụ trên, chúng ta đang extend CreateUserInput được chú thích bằng decorator @InputType(). Vì chúng ta muốn UpdateUserInput cũng được xử lý như thể nó được trang trí bằng @InputType(), nên chúng ta không cần truyền InputType làm đối số thứ hai. Nếu class cha và class con có type khác nhau (ví dụ, class cha được trang trí bằng @ObjectType), chúng ta sẽ truyền InputType làm đối số thứ hai. Ví dụ:

TypeScript
@InputType()
export class UpdateUserInput extends PartialType(User, InputType) {}

Pick #

Hàm PickType() xây dựng một type (class) mới bằng cách chọn một tập hợp các thuộc tính từ một input type. Ví dụ, giả sử chúng ta bắt đầu với một type như:

TypeScript
@InputType()
class CreateUserInput {
  @Field()
  email: string;

  @Field()
  password: string;

  @Field()
  firstName: string;
}

Chúng ta có thể chọn một tập hợp các thuộc tính từ class này bằng hàm tiện ích PickType():

TypeScript
@InputType()
export class UpdateEmailInput extends PickType(CreateUserInput, [
  'email',
] as const) {}
Gợi ý

Hàm PickType() được import từ package @nestjs/graphql.

Omit #

Hàm OmitType() xây dựng một type bằng cách chọn tất cả các thuộc tính từ một input type và sau đó loại bỏ một tập hợp key cụ thể. Ví dụ, giả sử chúng ta bắt đầu với một type như:

TypeScript
@InputType()
class CreateUserInput {
  @Field()
  email: string;

  @Field()
  password: string;

  @Field()
  firstName: string;
}

Chúng ta có thể tạo một type dẫn xuất có mọi thuộc tính ngoại trừ email như bên dưới. Trong cấu trúc này, đối số thứ hai của OmitType là một mảng các tên thuộc tính.

TypeScript
@InputType()
export class UpdateUserInput extends OmitType(CreateUserInput, [
  'email',
] as const) {}
Gợi ý

Hàm OmitType() được import từ package @nestjs/graphql.

Intersection #

Hàm IntersectionType() kết hợp hai type thành một type (class) mới. Ví dụ, giả sử chúng ta bắt đầu với hai type như:

TypeScript
@InputType()
class CreateUserInput {
  @Field()
  email: string;

  @Field()
  password: string;
}

@ObjectType()
export class AdditionalUserInfo {
  @Field()
  firstName: string;

  @Field()
  lastName: string;
}

Chúng ta có thể tạo một type mới kết hợp tất cả các thuộc tính từ cả hai type.

TypeScript
@InputType()
export class UpdateUserInput extends IntersectionType(
  CreateUserInput,
  AdditionalUserInfo,
) {}
Gợi ý

Hàm IntersectionType() được import từ package @nestjs/graphql.

Composition #

Các hàm tiện ích type mapping có thể kết hợp được với nhau. Ví dụ, đoạn sau sẽ tạo ra một type (class) có tất cả các thuộc tính của type CreateUserInput ngoại trừ email, và các thuộc tính đó sẽ được đặt thành tùy chọn:

TypeScript
@InputType()
export class UpdateUserInput extends PartialType(
  OmitType(CreateUserInput, ['email'] as const),
) {}
