Extensions

Cảnh báo

Chương này chỉ áp dụng cho cách tiếp cận code first.

Extensions là một tính năng nâng cao, cấp thấp cho phép bạn định nghĩa dữ liệu tùy ý trong cấu hình các type. Việc gắn custom metadata vào các field nhất định cho phép bạn tạo ra các giải pháp tổng quát và tinh vi hơn. Ví dụ, với extensions, bạn có thể định nghĩa các role cấp field cần thiết để truy cập các field cụ thể. Các role như vậy có thể được phản ánh tại runtime để xác định xem caller có đủ quyền để truy xuất một field cụ thể hay không.

Adding custom metadata #

Để gắn custom metadata cho một field, sử dụng decorator @Extensions() được export từ package @nestjs/graphql.

TypeScript
@Field()
@Extensions({ role: Role.ADMIN })
password: string;

Trong ví dụ trên, chúng ta đã gán property metadata role với giá trị Role.ADMIN. Role là một TypeScript enum đơn giản nhóm tất cả các user role có sẵn trong hệ thống của chúng ta.

Lưu ý, ngoài việc thiết lập metadata trên các field, bạn cũng có thể sử dụng decorator @Extensions() ở cấp class và cấp method (ví dụ, trên query handler).

Using custom metadata #

Logic tận dụng custom metadata có thể phức tạp tùy ý. Ví dụ, bạn có thể tạo một interceptor đơn giản lưu trữ/ghi log các sự kiện theo mỗi lần gọi method, hoặc một field middleware khớp các role cần thiết để truy xuất một field với quyền của caller (hệ thống quyền cấp field).

Để minh họa, hãy định nghĩa một checkRoleMiddleware so sánh role của người dùng (được hardcode ở đây) với role cần thiết để truy cập một field mục tiêu:

TypeScript
export const checkRoleMiddleware: FieldMiddleware = async (
  ctx: MiddlewareContext,
  next: NextFn,
) => {
  const { info } = ctx;
  const { extensions } = info.parentType.getFields()[info.fieldName];

  /**
   * In a real-world application, the "userRole" variable
   * should represent the caller's (user) role (for example, "ctx.user.role").
   */
  const userRole = Role.USER;
  if (userRole === extensions.role) {
    // or just "return null" to ignore
    throw new ForbiddenException(
      `User does not have sufficient permissions to access "${info.fieldName}" field.`,
    );
  }
  return next();
};

Với middleware này đã được thiết lập, chúng ta có thể đăng ký middleware cho field password, như sau:

TypeScript
@Field({ middleware: [checkRoleMiddleware] })
@Extensions({ role: Role.ADMIN })
password: string;
