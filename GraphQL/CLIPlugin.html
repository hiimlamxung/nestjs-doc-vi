CLI Plugin

Cảnh báo

Chương này chỉ áp dụng cho cách tiếp cận code first.

Hệ thống metadata reflection của TypeScript có một số hạn chế khiến việc xác định các property của một class hay nhận biết một property là optional hay required trở nên không thể. Tuy nhiên, một số ràng buộc này có thể được giải quyết tại thời điểm biên dịch. Nest cung cấp một plugin giúp cải thiện quá trình biên dịch TypeScript để giảm lượng boilerplate code cần thiết.

Gợi ý

Plugin này là opt-in. Nếu bạn muốn, bạn có thể khai báo tất cả các decorator thủ công, hoặc chỉ các decorator cụ thể ở những nơi bạn cần.

Overview #

GraphQL plugin sẽ tự động:

  • Gắn annotation cho tất cả các property của input object, object type và args class bằng @Field trừ khi @HideField được sử dụng
  • Thiết lập property nullable dựa trên dấu chấm hỏi (ví dụ: name?: string sẽ thiết lập nullable: true)
  • Thiết lập property type dựa trên kiểu dữ liệu (cũng hỗ trợ arrays)
  • Tạo description cho các property dựa trên comments (nếu introspectComments được thiết lập thành true)

Xin lưu ý rằng tên file của bạn phải có một trong các hậu tố sau để được plugin phân tích: ['.input.ts', '.args.ts', '.entity.ts', '.model.ts'] (ví dụ: author.entity.ts). Nếu bạn sử dụng hậu tố khác, bạn có thể điều chỉnh hành vi của plugin bằng cách chỉ định tùy chọn typeFileNameSuffix (xem bên dưới).

Với những gì chúng ta đã học cho đến nay, bạn phải lặp lại rất nhiều code để cho package biết kiểu dữ liệu của bạn nên được khai báo trong GraphQL như thế nào. Ví dụ, bạn có thể định nghĩa một class Author đơn giản như sau:

authors/models/author.model
@ObjectType()
export class Author {
  @Field(type => ID)
  id: number;

  @Field({ nullable: true })
  firstName?: string;

  @Field({ nullable: true })
  lastName?: string;

  @Field(type => [Post])
  posts: Post[];
}

Mặc dù không phải là vấn đề lớn với các dự án cỡ trung bình, nhưng nó trở nên dài dòng và khó bảo trì khi bạn có một tập hợp lớn các class.

Bằng cách bật GraphQL plugin, định nghĩa class ở trên có thể được khai báo đơn giản:

authors/models/author.model
@ObjectType()
export class Author {
  @Field(type => ID)
  id: number;
  firstName?: string;
  lastName?: string;
  posts: Post[];
}

Plugin sẽ thêm các decorator thích hợp một cách tự động dựa trên Abstract Syntax Tree. Do đó, bạn sẽ không phải vất vả với các decorator @Field nằm rải rác khắp code.

Gợi ý

Plugin sẽ tự động tạo bất kỳ property GraphQL nào bị thiếu, nhưng nếu bạn cần ghi đè chúng, chỉ cần thiết lập chúng một cách tường minh thông qua @Field().

Comments introspection #

Với tính năng comments introspection được bật, CLI plugin sẽ tạo description cho các field dựa trên comments.

Ví dụ, cho một property roles như sau:

TypeScript
/**
 * A list of user's roles
 */
@Field(() => [String], {
  description: `A list of user's roles`
})
roles: string[];

Bạn phải lặp lại giá trị description. Với introspectComments được bật, CLI plugin có thể trích xuất các comments này và tự động cung cấp description cho các property. Bây giờ, field ở trên có thể được khai báo đơn giản như sau:

TypeScript
/**
 * A list of user's roles
 */
roles: string[];

Using the CLI plugin #

Để bật plugin, mở nest-cli.json (nếu bạn sử dụng Nest CLI) và thêm cấu hình plugins như sau:

nest-cli.json
{
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "plugins": ["@nestjs/graphql"]
  }
}

Bạn có thể sử dụng property options để tùy chỉnh hành vi của plugin.

nest-cli.json
{
  "collection": "@nestjs/schematics",
  "sourceRoot": "src",
  "compilerOptions": {
    "plugins": [
      {
        "name": "@nestjs/graphql",
        "options": {
          "typeFileNameSuffix": [".input.ts", ".args.ts"],
          "introspectComments": true
        }
      }
    ]
  }
}

Property options phải thỏa mãn interface sau:

TypeScript
export interface PluginOptions {
  typeFileNameSuffix?: string[];
  introspectComments?: boolean;
}
Option Default Mô tả
typeFileNameSuffix ['.input.ts', '.args.ts', '.entity.ts', '.model.ts'] Hậu tố file cho các GraphQL type
introspectComments false Nếu được thiết lập thành true, plugin sẽ tạo description cho các property dựa trên comments

Nếu bạn không sử dụng CLI mà thay vào đó có cấu hình webpack tùy chỉnh, bạn có thể sử dụng plugin này kết hợp với ts-loader:

JavaScript
getCustomTransformers: (program: any) => ({
  before: [require('@nestjs/graphql/plugin').before({}, program)]
}),

SWC builder #

Đối với các thiết lập tiêu chuẩn (non-monorepo), để sử dụng CLI Plugins với SWC builder, bạn cần bật type checking, như mô tả tại đây.

Terminal
$ nest start -b swc --type-check

Đối với thiết lập monorepo, hãy làm theo hướng dẫn tại đây.

Terminal
$ npx ts-node src/generate-metadata.ts
# OR npx ts-node apps/{YOUR_APP}/src/generate-metadata.ts

Bây giờ, file serialized metadata phải được load bởi method GraphQLModule, như hiển thị bên dưới:

TypeScript
import metadata from './metadata'; // <-- file auto-generated by the "PluginMetadataGenerator"

GraphQLModule.forRoot<...>({
  ..., // other options
  metadata,
}),

Integration with ts-jest (e2e tests) #

Khi chạy e2e tests với plugin này được bật, bạn có thể gặp vấn đề khi biên dịch schema. Ví dụ, một trong những lỗi phổ biến nhất là:

JSON
Object type <name> must define one or more fields.

Điều này xảy ra vì cấu hình jest không import plugin @nestjs/graphql/plugin ở bất kỳ đâu.

Để khắc phục, hãy tạo file sau trong thư mục e2e tests của bạn:

JavaScript
const transformer = require('@nestjs/graphql/plugin');

module.exports.name = 'nestjs-graphql-transformer';
// you should change the version number anytime you change the configuration below - otherwise, jest will not detect changes
module.exports.version = 1;

module.exports.factory = (cs) => {
  return transformer.before(
    {
      // @nestjs/graphql/plugin options (can be empty)
    },
    cs.program, // "cs.tsCompiler.program" for older versions of Jest (<= v27)
  );
};

Với file này đã sẵn sàng, hãy import AST transformer trong file cấu hình jest của bạn. Mặc định (trong ứng dụng starter), file cấu hình e2e tests nằm trong thư mục test và có tên là jest-e2e.json.

jest-e2e.json
{
  ... // other configuration
  "globals": {
    "ts-jest": {
      "astTransformers": {
        "before": ["<path to the file created above>"]
      }
    }
  }
}

Nếu bạn sử dụng jest@^29, hãy sử dụng đoạn code bên dưới, vì cách tiếp cận trước đã bị deprecated.

jest-e2e.json
{
  ... // other configuration
  "transform": {
    "^.+\\.(t|j)s$": [
      "ts-jest",
      {
        "astTransformers": {
          "before": ["<path to the file created above>"]
        }
      }
    ]
  }
}
