Quick Start

Khai thác sức mạnh của TypeScript & GraphQL #

GraphQL là một ngôn ngữ truy vấn mạnh mẽ cho API và một runtime để thực thi các truy vấn đó với dữ liệu hiện có của bạn. Đây là một cách tiếp cận tinh tế giúp giải quyết nhiều vấn đề thường gặp với REST API. Để hiểu rõ hơn, chúng tôi khuyên bạn đọc bài so sánh giữa GraphQL và REST. GraphQL kết hợp với TypeScript giúp bạn phát triển type safety tốt hơn với các GraphQL query, mang lại typing end-to-end.

Trong chương này, chúng tôi giả định bạn đã có hiểu biết cơ bản về GraphQL, và tập trung vào cách làm việc với module @nestjs/graphql tích hợp sẵn. GraphQLModule có thể được cấu hình để sử dụng Apollo server (với driver @nestjs/apollo) và Mercurius (với driver @nestjs/mercurius). Chúng tôi cung cấp các tích hợp chính thức cho các package GraphQL đã được chứng minh này để mang lại cách sử dụng GraphQL đơn giản với Nest (xem thêm các tích hợp tại đây).

Bạn cũng có thể xây dựng driver tùy chỉnh của riêng mình (đọc thêm về điều đó tại đây).

Cài đặt #

Bắt đầu bằng cách cài đặt các package cần thiết:

bash
# Cho Express và Apollo (mặc định)
$ npm i @nestjs/graphql @nestjs/apollo @apollo/server @as-integrations/express5 graphql

# Cho Fastify và Apollo
# npm i @nestjs/graphql @nestjs/apollo @apollo/server @as-integrations/fastify graphql

# Cho Fastify và Mercurius
# npm i @nestjs/graphql @nestjs/mercurius graphql mercurius
⚠️ Warning

Các package @nestjs/graphql@>=9 và @nestjs/apollo^10 tương thích với Apollo v3 (xem thêm hướng dẫn migration Apollo Server 3 để biết thêm chi tiết), trong khi @nestjs/graphql@^8 chỉ hỗ trợ Apollo v2 (ví dụ: package apollo-server-express@2.x.x).

Tổng quan #

Nest cung cấp hai cách để xây dựng ứng dụng GraphQL, phương pháp code first và phương pháp schema first. Bạn nên chọn phương pháp phù hợp nhất với mình. Hầu hết các chương trong phần GraphQL này được chia thành hai phần chính: một phần bạn nên theo nếu áp dụng code first, và phần còn lại để sử dụng nếu bạn áp dụng schema first.

Trong phương pháp code first, bạn sử dụng decorator và các class TypeScript để tạo ra GraphQL schema tương ứng. Phương pháp này hữu ích nếu bạn muốn làm việc hoàn toàn với TypeScript và tránh chuyển đổi ngữ cảnh giữa các cú pháp ngôn ngữ.

Trong phương pháp schema first, nguồn sự thật là các file GraphQL SDL (Schema Definition Language). SDL là một cách chia sẻ file schema giữa các nền tảng khác nhau, không phụ thuộc ngôn ngữ. Nest tự động tạo các TypeScript definition (sử dụng class hoặc interface) dựa trên các GraphQL schema để giảm nhu cầu viết code boilerplate thừa.

Bắt đầu với GraphQL & TypeScript #

💡 Hint

Trong các chương tiếp theo, chúng tôi sẽ tích hợp package @nestjs/apollo. Nếu bạn muốn sử dụng package mercurius thay thế, hãy chuyển đến phần này.

Sau khi các package đã được cài đặt, chúng ta có thể import GraphQLModule và cấu hình nó với static method forRoot().

app.module.ts
import { Module } from '@nestjs/common';
import { GraphQLModule } from '@nestjs/graphql';
import { ApolloDriver, ApolloDriverConfig } from '@nestjs/apollo';

@Module({
  imports: [
    GraphQLModule.forRoot<ApolloDriverConfig>({
      driver: ApolloDriver,
    }),
  ],
})
export class AppModule {}
💡 Hint

Đối với tích hợp mercurius, bạn nên sử dụng MercuriusDriver và MercuriusDriverConfig thay thế. Cả hai đều được export từ package @nestjs/mercurius.

Method forRoot() nhận một object tùy chọn làm đối số. Các tùy chọn này được truyền qua cho instance driver bên dưới (đọc thêm về các cài đặt có sẵn tại đây: Apollo và Mercurius). Ví dụ, nếu bạn muốn tắt playground và tắt chế độ debug (cho Apollo), truyền các tùy chọn sau:

app.module.ts
import { Module } from '@nestjs/common';
import { GraphQLModule } from '@nestjs/graphql';
import { ApolloDriver, ApolloDriverConfig } from '@nestjs/apollo';

@Module({
  imports: [
    GraphQLModule.forRoot<ApolloDriverConfig>({
      driver: ApolloDriver,
      playground: false,
    }),
  ],
})
export class AppModule {}

Trong trường hợp này, các tùy chọn sẽ được chuyển tiếp đến constructor ApolloServer.

GraphQL playground #

Playground là một IDE GraphQL đồ họa, tương tác, chạy trong trình duyệt, có sẵn theo mặc định trên cùng URL với GraphQL server. Để truy cập playground, bạn cần một GraphQL server cơ bản đã được cấu hình và đang chạy. Để xem ngay bây giờ, bạn có thể cài đặt và build ví dụ hoạt động tại đây. Ngoài ra, nếu bạn đang theo dõi cùng các mẫu code này, sau khi hoàn thành các bước trong chương Resolvers, bạn có thể truy cập playground.

Khi đã sẵn sàng, và ứng dụng của bạn đang chạy nền, bạn có thể mở trình duyệt web và truy cập http://localhost:3000/graphql (host và port có thể thay đổi tùy thuộc vào cấu hình của bạn). Bạn sẽ thấy GraphQL playground, như hiển thị bên dưới.

💡 Note

Tích hợp @nestjs/mercurius không đi kèm với tích hợp GraphQL Playground tích hợp sẵn. Thay vào đó, bạn có thể sử dụng GraphiQL (đặt graphiql: true).

⚠️ Warning

Cập nhật (04/14/2025): Apollo playground mặc định đã bị deprecated và sẽ bị loại bỏ trong phiên bản major tiếp theo. Thay vào đó, bạn có thể sử dụng GraphiQL, chỉ cần đặt graphiql: true trong cấu hình GraphQLModule, như hiển thị bên dưới:

app.module.ts
GraphQLModule.forRoot<ApolloDriverConfig>({
  driver: ApolloDriver,
  graphiql: true,
}),

Nếu ứng dụng của bạn sử dụng subscriptions, hãy đảm bảo sử dụng graphql-ws, vì subscriptions-transport-ws không được GraphiQL hỗ trợ.

Code first #

Trong phương pháp code first, bạn sử dụng decorator và các class TypeScript để tạo ra GraphQL schema tương ứng.

Để sử dụng phương pháp code first, bắt đầu bằng cách thêm thuộc tính autoSchemaFile vào object tùy chọn:

app.module.ts
GraphQLModule.forRoot<ApolloDriverConfig>({
  driver: ApolloDriver,
  autoSchemaFile: join(process.cwd(), 'src/schema.gql'),
}),

Giá trị thuộc tính autoSchemaFile là đường dẫn nơi schema được tạo tự động sẽ được lưu. Ngoài ra, schema có thể được tạo ngay trong bộ nhớ (on-the-fly). Để kích hoạt điều này, đặt thuộc tính autoSchemaFile thành true:

app.module.ts
GraphQLModule.forRoot<ApolloDriverConfig>({
  driver: ApolloDriver,
  autoSchemaFile: true,
}),

Theo mặc định, các type trong schema được tạo sẽ theo thứ tự chúng được định nghĩa trong các module bao gồm. Để sắp xếp schema theo thứ tự bảng chữ cái (lexicographically), đặt thuộc tính sortSchema thành true:

app.module.ts
GraphQLModule.forRoot<ApolloDriverConfig>({
  driver: ApolloDriver,
  autoSchemaFile: join(process.cwd(), 'src/schema.gql'),
  sortSchema: true,
}),

Ví dụ #

Một mẫu code first hoạt động đầy đủ có sẵn tại đây.

Schema first #

Để sử dụng phương pháp schema first, bắt đầu bằng cách thêm thuộc tính typePaths vào object tùy chọn. Thuộc tính typePaths chỉ ra nơi GraphQLModule nên tìm kiếm các file GraphQL SDL schema definition mà bạn sẽ viết. Các file này sẽ được kết hợp trong bộ nhớ; điều này cho phép bạn chia schema thành nhiều file và đặt chúng gần các resolver của chúng.

app.module.ts
GraphQLModule.forRoot<ApolloDriverConfig>({
  driver: ApolloDriver,
  typePaths: ['./**/*.graphql'],
}),

Bạn thường cũng cần có các TypeScript definition (class và interface) tương ứng với các GraphQL SDL type. Việc tạo các TypeScript definition tương ứng bằng tay là thừa và tẻ nhạt. Nó khiến chúng ta không có một nguồn sự thật duy nhất -- mỗi thay đổi trong SDL buộc chúng ta phải điều chỉnh các TypeScript definition. Để giải quyết vấn đề này, package @nestjs/graphql có thể tự động tạo các TypeScript definition từ abstract syntax tree (AST). Để kích hoạt tính năng này, thêm thuộc tính tùy chọn definitions khi cấu hình GraphQLModule.

app.module.ts
GraphQLModule.forRoot<ApolloDriverConfig>({
  driver: ApolloDriver,
  typePaths: ['./**/*.graphql'],
  definitions: {
    path: join(process.cwd(), 'src/graphql.ts'),
  },
}),

Thuộc tính path của object definitions chỉ ra nơi lưu output TypeScript được tạo. Theo mặc định, tất cả các TypeScript type được tạo dưới dạng interface. Để tạo class thay thế, chỉ định thuộc tính outputAs với giá trị 'class'.

app.module.ts
GraphQLModule.forRoot<ApolloDriverConfig>({
  driver: ApolloDriver,
  typePaths: ['./**/*.graphql'],
  definitions: {
    path: join(process.cwd(), 'src/graphql.ts'),
    outputAs: 'class',
  },
}),

Phương pháp trên tự động tạo TypeScript definition mỗi khi ứng dụng khởi động. Ngoài ra, có thể tốt hơn nếu xây dựng một script đơn giản để tạo theo yêu cầu. Ví dụ, giả sử chúng ta tạo script sau với tên generate-typings.ts:

generate-typings.ts
import { GraphQLDefinitionsFactory } from '@nestjs/graphql';
import { join } from 'node:path';

const definitionsFactory = new GraphQLDefinitionsFactory();
definitionsFactory.generate({
  typePaths: ['./src/**/*.graphql'],
  path: join(process.cwd(), 'src/graphql.ts'),
  outputAs: 'class',
});

Bây giờ bạn có thể chạy script này theo yêu cầu:

bash
$ ts-node generate-typings
💡 Hint

Bạn có thể biên dịch script trước (ví dụ: với tsc) và sử dụng node để thực thi nó.

Để kích hoạt chế độ watch cho script (tự động tạo typing mỗi khi bất kỳ file .graphql nào thay đổi), truyền tùy chọn watch cho method generate().

generate-typings.ts
definitionsFactory.generate({
  typePaths: ['./src/**/*.graphql'],
  path: join(process.cwd(), 'src/graphql.ts'),
  outputAs: 'class',
  watch: true,
});

Để tự động tạo thêm trường __typename cho mỗi object type, kích hoạt tùy chọn emitTypenameField:

generate-typings.ts
definitionsFactory.generate({
  // ...
  emitTypenameField: true,
});

Để tạo các resolver (query, mutation, subscription) dưới dạng plain field không có argument, kích hoạt tùy chọn skipResolverArgs:

generate-typings.ts
definitionsFactory.generate({
  // ...
  skipResolverArgs: true,
});

Để tạo enum dưới dạng TypeScript union type thay vì TypeScript enum thông thường, đặt tùy chọn enumsAsTypes thành true:

generate-typings.ts
definitionsFactory.generate({
  // ...
  enumsAsTypes: true,
});

Apollo Sandbox #

Để sử dụng Apollo Sandbox thay vì graphql-playground làm GraphQL IDE cho phát triển local, sử dụng cấu hình sau:

app.module.ts
import { ApolloDriver, ApolloDriverConfig } from '@nestjs/apollo';
import { Module } from '@nestjs/common';
import { GraphQLModule } from '@nestjs/graphql';
import { ApolloServerPluginLandingPageLocalDefault } from '@apollo/server/plugin/landingPage/default';

@Module({
  imports: [
    GraphQLModule.forRoot<ApolloDriverConfig>({
      driver: ApolloDriver,
      playground: false,
      plugins: [ApolloServerPluginLandingPageLocalDefault()],
    }),
  ],
})
export class AppModule {}

Truy cập schema đã tạo #

Trong một số trường hợp (ví dụ: end-to-end test), bạn có thể muốn lấy tham chiếu đến schema object đã tạo. Trong end-to-end test, bạn có thể chạy query sử dụng object graphql mà không cần bất kỳ HTTP listener nào.

Bạn có thể truy cập schema đã tạo (trong cả phương pháp code first hoặc schema first), sử dụng class GraphQLSchemaHost:

app.ts
const { schema } = app.get(GraphQLSchemaHost);
💡 Hint

Bạn phải gọi getter GraphQLSchemaHost#schema sau khi ứng dụng đã được khởi tạo (sau khi hook onModuleInit đã được kích hoạt bởi method app.listen() hoặc app.init()).

Cấu hình bất đồng bộ #

Khi bạn cần truyền các tùy chọn module một cách bất đồng bộ thay vì tĩnh, sử dụng method forRootAsync(). Như với hầu hết các dynamic module, Nest cung cấp nhiều kỹ thuật để xử lý cấu hình bất đồng bộ.

Một kỹ thuật là sử dụng factory function:

app.module.ts
GraphQLModule.forRootAsync<ApolloDriverConfig>({
  driver: ApolloDriver,
  useFactory: () => ({
    typePaths: ['./**/*.graphql'],
  }),
}),

Giống như các factory provider khác, factory function của chúng ta có thể là async và có thể inject dependency thông qua inject.

app.module.ts
GraphQLModule.forRootAsync<ApolloDriverConfig>({
  driver: ApolloDriver,
  imports: [ConfigModule],
  useFactory: async (configService: ConfigService) => ({
    typePaths: configService.get<string>('GRAPHQL_TYPE_PATHS'),
  }),
  inject: [ConfigService],
}),

Ngoài ra, bạn có thể cấu hình GraphQLModule sử dụng class thay vì factory, như hiển thị bên dưới:

app.module.ts
GraphQLModule.forRootAsync<ApolloDriverConfig>({
  driver: ApolloDriver,
  useClass: GqlConfigService,
}),

Cấu trúc ở trên khởi tạo GqlConfigService bên trong GraphQLModule, sử dụng nó để tạo object tùy chọn. Lưu ý rằng trong ví dụ này, GqlConfigService phải implement interface GqlOptionsFactory, như hiển thị bên dưới. GraphQLModule sẽ gọi method createGqlOptions() trên object đã khởi tạo của class được cung cấp.

gql-config.service.ts
@Injectable()
class GqlConfigService implements GqlOptionsFactory {
  createGqlOptions(): ApolloDriverConfig {
    return {
      typePaths: ['./**/*.graphql'],
    };
  }
}

Nếu bạn muốn tái sử dụng một option provider hiện có thay vì tạo một bản sao riêng bên trong GraphQLModule, sử dụng cú pháp useExisting.

app.module.ts
GraphQLModule.forRootAsync<ApolloDriverConfig>({
  imports: [ConfigModule],
  useExisting: ConfigService,
}),

Tích hợp Mercurius #

Thay vì sử dụng Apollo, người dùng Fastify (đọc thêm tại đây) có thể sử dụng driver @nestjs/mercurius thay thế.

app.module.ts
import { Module } from '@nestjs/common';
import { GraphQLModule } from '@nestjs/graphql';
import { MercuriusDriver, MercuriusDriverConfig } from '@nestjs/mercurius';

@Module({
  imports: [
    GraphQLModule.forRoot<MercuriusDriverConfig>({
      driver: MercuriusDriver,
      graphiql: true,
    }),
  ],
})
export class AppModule {}
💡 Hint

Khi ứng dụng đang chạy, mở trình duyệt và truy cập http://localhost:3000/graphiql. Bạn sẽ thấy GraphQL IDE.

Method forRoot() nhận một object tùy chọn làm đối số. Các tùy chọn này được truyền qua cho instance driver bên dưới. Đọc thêm về các cài đặt có sẵn tại đây.

Nhiều endpoint #

Một tính năng hữu ích khác của module @nestjs/graphql là khả năng phục vụ nhiều endpoint cùng lúc. Điều này cho phép bạn quyết định module nào nên được bao gồm trong endpoint nào. Theo mặc định, GraphQL tìm kiếm resolver trong toàn bộ ứng dụng. Để giới hạn việc quét này chỉ vào một tập hợp con module, sử dụng thuộc tính include.

app.module.ts
GraphQLModule.forRoot({
  include: [CatsModule],
}),
⚠️ Warning

Nếu bạn sử dụng package @apollo/server với @as-integrations/fastify với nhiều GraphQL endpoint trong một ứng dụng duy nhất, hãy đảm bảo kích hoạt cài đặt disableHealthCheck trong cấu hình GraphQLModule.
