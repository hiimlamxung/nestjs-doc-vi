Sharing Models

Cảnh báo

Chương này chỉ áp dụng cho cách tiếp cận code first.

Một trong những lợi thế lớn nhất khi sử dụng Typescript cho backend của dự án là khả năng tái sử dụng cùng các model trong ứng dụng frontend dựa trên Typescript, bằng cách sử dụng một package Typescript chung.

Tuy nhiên có một vấn đề: các model được tạo bằng cách tiếp cận code first được trang trí nhiều decorator liên quan đến GraphQL. Những decorator đó không liên quan trong frontend, ảnh hưởng tiêu cực đến hiệu suất.

Using the model shim #

Để giải quyết vấn đề này, NestJS cung cấp một "shim" cho phép bạn thay thế các decorator gốc bằng code không hoạt động bằng cách sử dụng cấu hình webpack (hoặc tương tự). Để sử dụng shim này, hãy cấu hình một alias giữa package @nestjs/graphql và shim.

Ví dụ, đối với webpack thì được giải quyết theo cách này:

TypeScript
resolve: { // see: https://webpack.js.org/configuration/resolve/
  alias: {
      "@nestjs/graphql": path.resolve(__dirname, "../node_modules/@nestjs/graphql/dist/extra/graphql-model-shim")
  }
}
Gợi ý

Package TypeORM có một shim tương tự có thể tìm thấy tại đây.
