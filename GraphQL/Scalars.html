Scalars

Một GraphQL object type có tên và các field, nhưng đến một lúc nào đó các field đó phải resolve thành dữ liệu cụ thể. Đó là lúc các scalar type phát huy tác dụng: chúng đại diện cho các nút lá của query (đọc thêm tại đây). GraphQL bao gồm các type mặc định sau: Int, Float, String, Boolean và ID. Ngoài các type tích hợp sẵn này, bạn có thể cần hỗ trợ các kiểu dữ liệu nguyên tử tùy chỉnh (ví dụ: Date).

Code first #

Cách tiếp cận code first đi kèm với năm scalar, trong đó ba scalar là các alias đơn giản cho các GraphQL type hiện có.

  • ID (alias cho GraphQLID) - đại diện cho một định danh duy nhất, thường được sử dụng để fetch lại một object hoặc làm key cho cache
  • Int (alias cho GraphQLInt) - một số nguyên có dấu 32 bit
  • Float (alias cho GraphQLFloat) - một giá trị dấu phẩy động có dấu, độ chính xác kép
  • GraphQLISODateTime - một chuỗi date-time ở UTC (được sử dụng mặc định để biểu diễn type Date)
  • GraphQLTimestamp - một số nguyên có dấu đại diện cho ngày và giờ dưới dạng số mili giây tính từ đầu UNIX epoch

GraphQLISODateTime (ví dụ: 2019-12-03T09:54:33Z) được sử dụng mặc định để biểu diễn type Date. Để sử dụng GraphQLTimestamp thay thế, hãy đặt dateScalarMode của object buildSchemaOptions thành 'timestamp' như sau:

TypeScript
GraphQLModule.forRoot({
  buildSchemaOptions: {
    dateScalarMode: 'timestamp',
  }
}),

Tương tự, GraphQLFloat được sử dụng mặc định để biểu diễn type number. Để sử dụng GraphQLInt thay thế, hãy đặt numberScalarMode của object buildSchemaOptions thành 'integer' như sau:

TypeScript
GraphQLModule.forRoot({
  buildSchemaOptions: {
    numberScalarMode: 'integer',
  }
}),

Ngoài ra, bạn có thể tạo các custom scalar.

Override a default scalar #

Để tạo một implementation tùy chỉnh cho scalar Date, chỉ cần tạo một class mới.

TypeScript
import { Scalar, CustomScalar } from '@nestjs/graphql';
import { Kind, ValueNode } from 'graphql';

@Scalar('Date', () => Date)
export class DateScalar implements CustomScalar<number, Date> {
  description = 'Date custom scalar type';

  parseValue(value: number): Date {
    return new Date(value); // value from the client
  }

  serialize(value: Date): number {
    return value.getTime(); // value sent to the client
  }

  parseLiteral(ast: ValueNode): Date {
    if (ast.kind === Kind.INT) {
      return new Date(ast.value);
    }
    return null;
  }
}

Với điều này, hãy đăng ký DateScalar như một provider.

TypeScript
@Module({
  providers: [DateScalar],
})
export class CommonModule {}

Bây giờ chúng ta có thể sử dụng type Date trong các class của mình.

TypeScript
@Field()
creationDate: Date;

Import a custom scalar #

Để sử dụng một custom scalar, hãy import và đăng ký nó như một resolver. Chúng ta sẽ sử dụng package graphql-type-json để minh họa. Package npm này định nghĩa một scalar type JSON cho GraphQL.

Bắt đầu bằng việc cài đặt package:

Terminal
$ npm i --save graphql-type-json

Sau khi package được cài đặt, chúng ta truyền một custom resolver vào method forRoot():

TypeScript
import GraphQLJSON from 'graphql-type-json';

@Module({
  imports: [
    GraphQLModule.forRoot({
      resolvers: { JSON: GraphQLJSON },
    }),
  ],
})
export class AppModule {}

Bây giờ chúng ta có thể sử dụng type JSON trong các class của mình.

TypeScript
@Field(() => GraphQLJSON)
info: JSON;

Để có bộ sưu tập các scalar hữu ích, hãy tham khảo package graphql-scalars.

Create a custom scalar #

Để định nghĩa một custom scalar, hãy tạo một instance GraphQLScalarType mới. Chúng ta sẽ tạo một custom scalar UUID.

TypeScript
const regex = /^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i;

function validate(uuid: unknown): string | never {
  if (typeof uuid !== 'string' || !regex.test(uuid)) {
    throw new Error('invalid uuid');
  }
  return uuid;
}

export const CustomUuidScalar = new GraphQLScalarType({
  name: 'UUID',
  description: 'A simple UUID parser',
  serialize: (value) => validate(value),
  parseValue: (value) => validate(value),
  parseLiteral: (ast) => validate(ast.value),
});

Chúng ta truyền một custom resolver vào method forRoot():

TypeScript
@Module({
  imports: [
    GraphQLModule.forRoot({
      resolvers: { UUID: CustomUuidScalar },
    }),
  ],
})
export class AppModule {}

Bây giờ chúng ta có thể sử dụng type UUID trong các class của mình.

TypeScript
@Field(() => CustomUuidScalar)
uuid: string;

Schema first #

Để định nghĩa một custom scalar (đọc thêm về scalar tại đây), hãy tạo một type definition và một resolver chuyên dụng. Ở đây (giống như trong tài liệu chính thức), chúng ta sẽ sử dụng package graphql-type-json để minh họa. Package npm này định nghĩa một scalar type JSON cho GraphQL.

Bắt đầu bằng việc cài đặt package:

Terminal
$ npm i --save graphql-type-json

Sau khi package được cài đặt, chúng ta truyền một custom resolver vào method forRoot():

TypeScript
import GraphQLJSON from 'graphql-type-json';

@Module({
  imports: [
    GraphQLModule.forRoot({
      typePaths: ['./**/*.graphql'],
      resolvers: { JSON: GraphQLJSON },
    }),
  ],
})
export class AppModule {}

Bây giờ chúng ta có thể sử dụng scalar JSON trong các type definition:

GraphQL
scalar JSON

type Foo {
  field: JSON
}

Một cách khác để định nghĩa scalar type là tạo một class đơn giản. Giả sử chúng ta muốn mở rộng schema với type Date.

TypeScript
import { Scalar, CustomScalar } from '@nestjs/graphql';
import { Kind, ValueNode } from 'graphql';

@Scalar('Date')
export class DateScalar implements CustomScalar<number, Date> {
  description = 'Date custom scalar type';

  parseValue(value: number): Date {
    return new Date(value); // value from the client
  }

  serialize(value: Date): number {
    return value.getTime(); // value sent to the client
  }

  parseLiteral(ast: ValueNode): Date {
    if (ast.kind === Kind.INT) {
      return new Date(ast.value);
    }
    return null;
  }
}

Với điều này, hãy đăng ký DateScalar như một provider.

TypeScript
@Module({
  providers: [DateScalar],
})
export class CommonModule {}

Bây giờ chúng ta có thể sử dụng scalar Date trong các type definition.

GraphQL
scalar Date

Mặc định, TypeScript definition được tạo cho tất cả scalar là any — điều này không đặc biệt an toàn về kiểu. Tuy nhiên, bạn có thể cấu hình cách Nest tạo typing cho các custom scalar khi bạn chỉ định cách tạo type:

TypeScript
import { GraphQLDefinitionsFactory } from '@nestjs/graphql';
import { join } from 'path';

const definitionsFactory = new GraphQLDefinitionsFactory();

definitionsFactory.generate({
  typePaths: ['./src/**/*.graphql'],
  path: join(process.cwd(), 'src/graphql.ts'),
  outputAs: 'class',
  defaultScalarType: 'unknown',
  customScalarTypeMapping: {
    DateTime: 'Date',
    BigNumber: '_BigNumber',
  },
  additionalHeader: "import _BigNumber from 'bignumber.js'",
});
Gợi ý

Ngoài ra, bạn có thể sử dụng type reference, ví dụ: DateTime: Date. Trong trường hợp này, GraphQLDefinitionsFactory sẽ trích xuất property name của type được chỉ định (Date.name) để tạo các TS definition. Lưu ý: việc thêm câu lệnh import cho các type không phải built-in (custom type) là bắt buộc.

Bây giờ, với các custom scalar type GraphQL sau:

GraphQL
scalar DateTime
scalar BigNumber
scalar Payload

Chúng ta sẽ thấy các TypeScript definition được tạo sau đây trong src/graphql.ts:

src/graphql.ts
import _BigNumber from 'bignumber.js';

export type DateTime = Date;
export type BigNumber = _BigNumber;
export type Payload = unknown;

Ở đây, chúng ta đã sử dụng property customScalarTypeMapping để cung cấp một map các type mà chúng ta muốn khai báo cho các custom scalar. Chúng ta cũng đã cung cấp property additionalHeader để thêm bất kỳ import nào cần thiết cho các type definition này. Cuối cùng, chúng ta đã thêm defaultScalarType là 'unknown', để bất kỳ custom scalar nào không được chỉ định trong customScalarTypeMapping sẽ được alias thành unknown thay vì any (điều mà TypeScript khuyến nghị sử dụng kể từ phiên bản 3.0 để tăng tính an toàn về kiểu).

Gợi ý

Lưu ý rằng chúng ta đã import _BigNumber từ bignumber.js; đây là để tránh circular type reference.
