Generating SDL

Cảnh báo

Chương này chỉ áp dụng cho cách tiếp cận code first.

Để tạo GraphQL SDL schema thủ công (tức là không cần chạy ứng dụng, kết nối database, hook up các resolver, v.v.), sử dụng GraphQLSchemaBuilderModule.

TypeScript
async function generateSchema() {
  const app = await NestFactory.create(GraphQLSchemaBuilderModule);
  await app.init();

  const gqlSchemaFactory = app.get(GraphQLSchemaFactory);
  const schema = await gqlSchemaFactory.create([RecipesResolver]);
  console.log(printSchema(schema));
}
Gợi ý

GraphQLSchemaBuilderModule và GraphQLSchemaFactory được import từ package @nestjs/graphql. Hàm printSchema được import từ package graphql.

Usage #

Method gqlSchemaFactory.create() nhận vào một mảng các class reference của resolver. Ví dụ:

TypeScript
const schema = await gqlSchemaFactory.create([
  RecipesResolver,
  AuthorsResolver,
  PostsResolvers,
]);

Nó cũng nhận tham số thứ hai tùy chọn là một mảng các class scalar:

TypeScript
const schema = await gqlSchemaFactory.create(
  [RecipesResolver, AuthorsResolver, PostsResolvers],
  [DurationScalar, DateScalar],
);

Cuối cùng, bạn có thể truyền một object options:

TypeScript
const schema = await gqlSchemaFactory.create([RecipesResolver], {
  skipCheck: true,
  orphanedTypes: [],
});
  • skipCheck: bỏ qua việc validation schema; kiểu boolean, mặc định là false
  • orphanedTypes: danh sách các class không được tham chiếu tường minh (không thuộc object graph) cần được generate. Thông thường, nếu một class được khai báo nhưng không được tham chiếu ở nơi khác trong graph, nó sẽ bị bỏ qua. Giá trị của property là một mảng các class reference.
