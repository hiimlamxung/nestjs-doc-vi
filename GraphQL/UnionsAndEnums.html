Unions and Enums

Unions #

Union type rất giống với interface, nhưng chúng không chỉ định bất kỳ field chung nào giữa các type (đọc thêm tại đây). Union hữu ích để trả về các kiểu dữ liệu rời rạc từ một field duy nhất.

Code first

Để định nghĩa một GraphQL union type, chúng ta phải định nghĩa các class mà union này sẽ được tạo thành. Theo ví dụ từ tài liệu Apollo, chúng ta sẽ tạo hai class. Đầu tiên là Book:

TypeScript
import { Field, ObjectType } from '@nestjs/graphql';

@ObjectType()
export class Book {
  @Field()
  title: string;
}

Và sau đó là Author:

TypeScript
import { Field, ObjectType } from '@nestjs/graphql';

@ObjectType()
export class Author {
  @Field()
  name: string;
}

Với điều này, hãy đăng ký ResultUnion union bằng hàm createUnionType được export từ package @nestjs/graphql:

TypeScript
export const ResultUnion = createUnionType({
  name: 'ResultUnion',
  types: () => [Author, Book] as const,
});
Cảnh báo

Mảng được trả về bởi property types của hàm createUnionType cần được cung cấp const assertion. Nếu không có const assertion, một file declaration sai sẽ được tạo ra tại thời điểm compile, và lỗi sẽ xảy ra khi sử dụng nó từ project khác.

Bây giờ, chúng ta có thể tham chiếu ResultUnion trong query của mình:

TypeScript
@Query(() => [ResultUnion])
search(): Array<typeof ResultUnion> {
  return [new Author(), new Book()];
}

Điều này sẽ tạo ra phần sau của GraphQL schema trong SDL:

GraphQL
type Author {
  name: String!
}

type Book {
  title: String!
}

union ResultUnion = Author | Book

type Query {
  search: [ResultUnion!]!
}

Hàm resolveType() mặc định được tạo bởi thư viện sẽ trích xuất type dựa trên giá trị được trả về từ resolver method. Điều đó có nghĩa là việc trả về các class instance thay vì literal JavaScript object là bắt buộc.

Để cung cấp một hàm resolveType() tùy chỉnh, hãy truyền property resolveType vào object options được truyền vào hàm createUnionType(), như sau:

TypeScript
export const ResultUnion = createUnionType({
  name: 'ResultUnion',
  types: () => [Author, Book] as const,
  resolveType(value) {
    if (value.name) {
      return Author;
    }
    if (value.title) {
      return Book;
    }
    return null;
  },
});

Schema first

Để định nghĩa một union trong cách tiếp cận schema first, chỉ cần tạo một GraphQL union với SDL.

GraphQL
type Author {
  name: String!
}

type Book {
  title: String!
}

union ResultUnion = Author | Book

Sau đó, bạn có thể sử dụng tính năng typings generation (như đã trình bày trong chương quick start) để tạo các TypeScript definition tương ứng:

TypeScript
export class Author {
  name: string;
}

export class Book {
  title: string;
}

export type ResultUnion = Author | Book;

Union yêu cầu một field __resolveType bổ sung trong resolver map để xác định union nên resolve thành type nào. Ngoài ra, lưu ý rằng class ResultUnionResolver phải được đăng ký như một provider trong bất kỳ module nào. Hãy tạo một class ResultUnionResolver và định nghĩa method __resolveType.

TypeScript
@Resolver('ResultUnion')
export class ResultUnionResolver {
  @ResolveField()
  __resolveType(value) {
    if (value.name) {
      return 'Author';
    }
    if (value.title) {
      return 'Book';
    }
    return null;
  }
}
Gợi ý

Tất cả decorator đều được export từ package @nestjs/graphql.

Enums #

Enumeration type là một loại scalar đặc biệt bị giới hạn trong một tập hợp các giá trị được phép cụ thể (đọc thêm tại đây). Điều này cho phép bạn:

  • Xác nhận rằng bất kỳ argument nào thuộc type này đều là một trong các giá trị được phép
  • Truyền đạt thông qua hệ thống type rằng một field sẽ luôn là một trong tập hợp hữu hạn các giá trị

Code first

Khi sử dụng cách tiếp cận code first, bạn định nghĩa một GraphQL enum type bằng cách đơn giản tạo một TypeScript enum.

TypeScript
export enum AllowedColor {
  RED,
  GREEN,
  BLUE,
}

Với điều này, hãy đăng ký enum AllowedColor bằng hàm registerEnumType được export từ package @nestjs/graphql:

TypeScript
registerEnumType(AllowedColor, {
  name: 'AllowedColor',
});

Bây giờ bạn có thể tham chiếu AllowedColor trong các type của mình:

TypeScript
@Field(type => AllowedColor)
favoriteColor: AllowedColor;

Điều này sẽ tạo ra phần sau của GraphQL schema trong SDL:

GraphQL
enum AllowedColor {
  RED
  GREEN
  BLUE
}

Để cung cấp description cho enum, hãy truyền property description vào hàm registerEnumType().

TypeScript
registerEnumType(AllowedColor, {
  name: 'AllowedColor',
  description: 'The supported colors.',
});

Để cung cấp description cho các giá trị enum, hoặc đánh dấu một giá trị là deprecated, hãy truyền property valuesMap, như sau:

TypeScript
registerEnumType(AllowedColor, {
  name: 'AllowedColor',
  description: 'The supported colors.',
  valuesMap: {
    RED: {
      description: 'The default color.',
    },
    BLUE: {
      deprecationReason: 'Too blue.',
    },
  },
});

Điều này sẽ tạo ra GraphQL schema sau trong SDL:

GraphQL
"""
The supported colors.
"""
enum AllowedColor {
  """
  The default color.
  """
  RED
  GREEN
  BLUE @deprecated(reason: "Too blue.")
}

Schema first

Để định nghĩa một enumerator trong cách tiếp cận schema first, chỉ cần tạo một GraphQL enum với SDL.

GraphQL
enum AllowedColor {
  RED
  GREEN
  BLUE
}

Sau đó bạn có thể sử dụng tính năng typings generation (như đã trình bày trong chương quick start) để tạo các TypeScript definition tương ứng:

TypeScript
export enum AllowedColor {
  RED
  GREEN
  BLUE
}

Đôi khi backend sử dụng một giá trị khác cho enum nội bộ so với trong API công khai. Trong ví dụ này, API chứa RED, tuy nhiên trong resolver chúng ta có thể sử dụng #f00 thay thế (đọc thêm tại đây). Để thực hiện điều này, hãy khai báo một resolver object cho enum AllowedColor:

TypeScript
export const allowedColorResolver: Record<keyof typeof AllowedColor, any> = {
  RED: '#f00',
};
Gợi ý

Tất cả decorator đều được export từ package @nestjs/graphql.

Sau đó sử dụng resolver object này cùng với property resolvers của method GraphQLModule#forRoot(), như sau:

TypeScript
GraphQLModule.forRoot({
  resolvers: {
    AllowedColor: allowedColorResolver,
  },
});
