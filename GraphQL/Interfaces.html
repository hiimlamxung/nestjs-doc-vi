Interfaces

Giống như nhiều hệ thống kiểu dữ liệu khác, GraphQL hỗ trợ interface. Một Interface là một kiểu trừu tượng bao gồm một tập hợp các field nhất định mà một type phải bao gồm để implement interface đó (đọc thêm tại đây).

Code first #

Khi sử dụng cách tiếp cận code first, bạn định nghĩa một GraphQL interface bằng cách tạo một abstract class được chú thích bằng decorator @InterfaceType() được export từ @nestjs/graphql.

TypeScript
import { Field, ID, InterfaceType } from '@nestjs/graphql';

@InterfaceType()
export abstract class Character {
  @Field(() => ID)
  id: string;

  @Field()
  name: string;
}
Cảnh báo

Các TypeScript interface không thể được sử dụng để định nghĩa GraphQL interface.

Điều này sẽ tạo ra phần sau trong GraphQL schema bằng SDL:

GraphQL
interface Character {
  id: ID!
  name: String!
}

Bây giờ, để implement interface Character, sử dụng key implements:

TypeScript
@ObjectType({
  implements: () => [Character],
})
export class Human implements Character {
  id: string;
  name: string;
}
Gợi ý

Decorator @ObjectType() được export từ package @nestjs/graphql.

Hàm resolveType() mặc định được tạo bởi thư viện sẽ trích xuất type dựa trên giá trị được trả về từ phương thức resolver. Điều này có nghĩa là bạn phải trả về các class instance (bạn không thể trả về các JavaScript object literal).

Để cung cấp một hàm resolveType() tùy chỉnh, truyền thuộc tính resolveType vào object options được truyền vào decorator @InterfaceType(), như sau:

TypeScript
@InterfaceType({
  resolveType(book) {
    if (book.colors) {
      return ColoringBook;
    }
    return TextBook;
  },
})
export abstract class Book {
  @Field(() => ID)
  id: string;

  @Field()
  title: string;
}

Interface resolvers #

Cho đến nay, khi sử dụng interface, bạn chỉ có thể chia sẻ các định nghĩa field với các object của mình. Nếu bạn cũng muốn chia sẻ triển khai thực tế của field resolver, bạn có thể tạo một interface resolver chuyên biệt, như sau:

TypeScript
import { Resolver, ResolveField, Parent, Info } from '@nestjs/graphql';

@Resolver((type) => Character) // Reminder: Character is an interface
export class CharacterInterfaceResolver {
  @ResolveField(() => [Character])
  friends(
    @Parent() character, // Resolved object that implements Character
    @Info() { parentType }, // Type of the object that implements Character
    @Args('search', { type: () => String }) searchTerm: string,
  ) {
    // Get character's friends
    return [];
  }
}

Bây giờ field resolver friends được tự động đăng ký cho tất cả các object type implement interface Character.

Cảnh báo

Điều này yêu cầu thuộc tính inheritResolversFromInterfaces được đặt thành true trong cấu hình GraphQLModule.

Schema first #

Để định nghĩa một interface trong cách tiếp cận schema first, chỉ cần tạo một GraphQL interface bằng SDL.

GraphQL
interface Character {
  id: ID!
  name: String!
}

Sau đó, bạn có thể sử dụng tính năng tạo typings (như được trình bày trong chương quick start) để tạo các định nghĩa TypeScript tương ứng:

TypeScript
export interface Character {
  id: string;
  name: string;
}

Interface yêu cầu một field __resolveType bổ sung trong resolver map để xác định type nào mà interface sẽ resolve. Hãy tạo một class CharactersResolver và định nghĩa phương thức __resolveType:

TypeScript
@Resolver('Character')
export class CharactersResolver {
  @ResolveField()
  __resolveType(value) {
    if ('age' in value) {
      return Person;
    }
    return null;
  }
}
Gợi ý

Tất cả decorator đều được export từ package @nestjs/graphql.
