Tiếng Việt Tài liệu NestJS được dịch sang tiếng Việt

Authentication

Authentication là một phần thiết yếu của hầu hết các ứng dụng. Có nhiều cách tiếp cận và chiến lược khác nhau để xử lý authentication. Cách tiếp cận được sử dụng cho bất kỳ dự án nào phụ thuộc vào yêu cầu cụ thể của ứng dụng đó. Chương này trình bày một số cách tiếp cận authentication có thể được điều chỉnh cho nhiều yêu cầu khác nhau.

Hãy cụ thể hóa các yêu cầu của chúng ta. Trong trường hợp sử dụng này, client sẽ bắt đầu bằng việc xác thực với username và password. Sau khi được xác thực, server sẽ phát hành một JWT có thể được gửi dưới dạng bearer token trong authorization header của các request tiếp theo để chứng minh đã được xác thực. Chúng ta cũng sẽ tạo một route được bảo vệ chỉ có thể truy cập bởi các request chứa JWT hợp lệ.

Chúng ta sẽ bắt đầu với yêu cầu đầu tiên: xác thực người dùng. Sau đó chúng ta sẽ mở rộng bằng cách phát hành JWT. Cuối cùng, chúng ta sẽ tạo một route được bảo vệ kiểm tra JWT hợp lệ trên request.

Tạo authentication module #

Chúng ta sẽ bắt đầu bằng việc tạo một AuthModule và trong đó, một AuthService và một AuthController. Chúng ta sẽ sử dụng AuthService để triển khai logic authentication, và AuthController để expose các authentication endpoint.

bash 
$ nest g module auth
$ nest g controller auth
$ nest g service auth

Khi chúng ta triển khai AuthService, chúng ta sẽ thấy hữu ích khi đóng gói các thao tác người dùng trong một UsersService, vì vậy hãy tạo module và service đó ngay bây giờ:

bash 
$ nest g module users
$ nest g service users

Thay thế nội dung mặc định của các file được tạo như hiển thị bên dưới. Đối với ứng dụng mẫu của chúng ta, UsersService đơn giản duy trì một danh sách người dùng hard-coded trong bộ nhớ, và một phương thức find để truy xuất một người dùng theo username. Trong ứng dụng thực tế, đây là nơi bạn sẽ xây dựng user model và persistence layer, sử dụng thư viện mà bạn chọn (ví dụ: TypeORM, Sequelize, Mongoose, v.v.).

users/users.service 
import { Injectable } from '@nestjs/common';

// This should be a real class/interface representing a user entity
export type User = any;

@Injectable()
export class UsersService {
  private readonly users = [
    {
      userId: 1,
      username: 'john',
      password: 'changeme',
    },
    {
      userId: 2,
      username: 'maria',
      password: 'guess',
    },
  ];

  async findOne(username: string): Promise<User | undefined> {
    return this.users.find(user => user.username === username);
  }
}

@Injectable()
export class UsersService {
  constructor() {
    this.users = [
      {
        userId: 1,
        username: 'john',
        password: 'changeme',
      },
      {
        userId: 2,
        username: 'maria',
        password: 'guess',
      },
    ];
  }

    return this.users.find(user => user.username === username);
  }
}

Trong UsersModule, thay đổi duy nhất cần thiết là thêm UsersService vào mảng exports của decorator @Module để nó có thể nhìn thấy được bên ngoài module này (chúng ta sẽ sớm sử dụng nó trong AuthService).

  providers: [UsersService],
  exports: [UsersService],

Triển khai endpoint "Sign in" #

AuthService của chúng ta có nhiệm vụ truy xuất người dùng và xác minh password. Chúng ta tạo phương thức signIn() cho mục đích này. Trong đoạn code bên dưới, chúng ta sử dụng toán tử spread ES6 tiện lợi để loại bỏ thuộc tính password khỏi user object trước khi trả về. Đây là một thực hành phổ biến khi trả về user object, vì bạn không muốn expose các trường nhạy cảm như password hoặc các security key khác.

Warning

Tất nhiên trong ứng dụng thực tế, bạn sẽ không lưu trữ password ở dạng plain text. Thay vào đó, bạn sẽ sử dụng một thư viện như bcrypt, với thuật toán hash một chiều có salt. Với cách tiếp cận đó, bạn chỉ lưu trữ password đã được hash, và sau đó so sánh password đã lưu với phiên bản hash của password đầu vào, do đó không bao giờ lưu trữ hoặc expose password người dùng ở dạng plain text. Để giữ cho ứng dụng mẫu đơn giản, chúng ta vi phạm quy tắc tuyệt đối đó và sử dụng plain text. Đừng làm điều này trong ứng dụng thực tế!

Bây giờ, chúng ta cập nhật AuthModule để import UsersModule.

  imports: [UsersModule],
  providers: [AuthService],
  controllers: [AuthController],
})
export class AuthModule {}

@Module({
  imports: [UsersModule],
  providers: [AuthService],
  controllers: [AuthController],
})
export class AuthModule {}

Với điều này đã sẵn sàng, hãy mở AuthController và thêm phương thức signIn() vào đó. Phương thức này sẽ được client gọi để xác thực người dùng. Nó sẽ nhận username và password trong request body, và sẽ trả về một JWT token nếu người dùng được xác thực.

auth/auth.controller 
import { Body, Controller, Post, HttpCode, HttpStatus } from '@nestjs/common';
import { AuthService } from './auth.service';

@Controller('auth')
export class AuthController {
  constructor(private authService: AuthService) {}

  @HttpCode(HttpStatus.OK)
  @Post('login')
  signIn(@Body() signInDto: Record<string, any>) {
    return this.authService.signIn(signInDto.username, signInDto.password);
  }
}
Gợi ý

Lý tưởng nhất, thay vì sử dụng kiểu Record<string, any>, chúng ta nên sử dụng một DTO class để định nghĩa hình dạng của request body. Xem chương validation để biết thêm thông tin.

JWT token #

Chúng ta đã sẵn sàng chuyển sang phần JWT của hệ thống auth. Hãy xem xét và tinh chỉnh các yêu cầu của chúng ta:

  • Cho phép người dùng xác thực bằng username/password, trả về một JWT để sử dụng trong các lần gọi tiếp theo đến các API endpoint được bảo vệ. Chúng ta đang trên đường đạt được yêu cầu này. Để hoàn thành, chúng ta cần viết code phát hành JWT.
  • Tạo các API route được bảo vệ dựa trên sự hiện diện của JWT hợp lệ dưới dạng bearer token

Chúng ta cần cài đặt thêm một package để hỗ trợ yêu cầu JWT:

bash 
$ npm install --save @nestjs/jwt
Gợi ý

Package @nestjs/jwt (xem thêm tại đây) là một utility package giúp thao tác với JWT. Bao gồm việc tạo và xác minh JWT token.

Để giữ cho các service được module hóa gọn gàng, chúng ta sẽ xử lý việc tạo JWT trong authService. Mở file auth.service.ts trong thư mục auth, inject JwtService, và cập nhật phương thức signIn để tạo JWT token:

auth/auth.service 
import { Injectable, UnauthorizedException } from '@nestjs/common';
import { UsersService } from '../users/users.service';
import { JwtService } from '@nestjs/jwt';

@Injectable()
export class AuthService {
  constructor(
    private usersService: UsersService,
    private jwtService: JwtService
  ) {}

  async signIn(
    username: string,
    pass: string,
  ): Promise<{ access_token: string }> {
    const user = await this.usersService.findOne(username);
    if (user?.password !== pass) {
      throw new UnauthorizedException();
    }
    const payload = { sub: user.userId, username: user.username };
    return {
      access_token: await this.jwtService.signAsync(payload),
    };
  }
}
import { Injectable, Dependencies, UnauthorizedException } from '@nestjs/common';
import { UsersService } from '../users/users.service';
import { JwtService } from '@nestjs/jwt';

@Dependencies(UsersService, JwtService)
@Injectable()
export class AuthService {
  constructor(usersService, jwtService) {
    this.usersService = usersService;
    this.jwtService = jwtService;
  }

  async signIn(username, pass) {
    const user = await this.usersService.findOne(username);
    if (user?.password !== pass) {
      throw new UnauthorizedException();
    }
    const payload = { username: user.username, sub: user.userId };
    return {
      access_token: await this.jwtService.signAsync(payload),
    };
  }
}

Chúng ta đang sử dụng thư viện @nestjs/jwt, cung cấp hàm signAsync() để tạo JWT từ một tập hợp con các thuộc tính của user object, sau đó trả về dưới dạng một object đơn giản với thuộc tính access_token duy nhất. Lưu ý: chúng ta chọn tên thuộc tính sub để chứa giá trị userId nhằm nhất quán với tiêu chuẩn JWT.

Bây giờ chúng ta cần cập nhật AuthModule để import các dependency mới và cấu hình JwtModule.

Đầu tiên, tạo constants.ts trong thư mục auth, và thêm đoạn code sau:

auth/constants

  secret: 'DO NOT USE THIS VALUE. INSTEAD, CREATE A COMPLEX SECRET AND KEEP IT SAFE OUTSIDE OF THE SOURCE CODE.',
};

  secret: 'DO NOT USE THIS VALUE. INSTEAD, CREATE A COMPLEX SECRET AND KEEP IT SAFE OUTSIDE OF THE SOURCE CODE.',

Chúng ta sẽ sử dụng điều này để chia sẻ key giữa các bước ký và xác minh JWT.

Warning

Không expose key này công khai. Chúng ta đã làm vậy ở đây để làm rõ code đang làm gì, nhưng trong hệ thống production bạn phải bảo vệ key này bằng các biện pháp phù hợp như secrets vault, environment variable, hoặc configuration service.

Bây giờ, mở auth.module.ts trong thư mục auth và cập nhật nó như sau:

@Module({

    UsersModule,
    JwtModule.register({
      global: true,
      secret: jwtConstants.secret,
      signOptions: { expiresIn: '60s' },
    }),
  ],
  providers: [AuthService],
  controllers: [AuthController],
  exports: [AuthService],
})
export class AuthModule {}

    UsersModule,
    JwtModule.register({
      global: true,

      signOptions: { expiresIn: '60s' },
    }),
  ],
  providers: [AuthService],
  controllers: [AuthController],
  exports: [AuthService],
})

Gợi ý

Chúng ta đăng ký JwtModule là global để mọi thứ dễ dàng hơn. Điều này có nghĩa là chúng ta không cần import JwtModule ở bất kỳ nơi nào khác trong ứng dụng.

Chúng ta cấu hình JwtModule bằng register(), truyền vào một configuration object. Xem tại đây để biết thêm về Nest JwtModule và tại đây để biết thêm chi tiết về các tùy chọn cấu hình có sẵn.

Hãy tiếp tục và kiểm tra các route bằng cURL. Bạn có thể kiểm tra với bất kỳ user object nào được hard-coded trong UsersService.

Triển khai authentication guard #

Bây giờ chúng ta có thể giải quyết yêu cầu cuối cùng: bảo vệ endpoint bằng cách yêu cầu JWT hợp lệ có mặt trên request. Chúng ta sẽ làm điều này bằng cách tạo một AuthGuard mà chúng ta có thể sử dụng để bảo vệ các route.

auth/auth.guard 
import {
  CanActivate,
  ExecutionContext,
  Injectable,
  UnauthorizedException,
} from '@nestjs/common';
import { JwtService } from '@nestjs/jwt';
import { Request } from 'express';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(private jwtService: JwtService) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest();
    const token = this.extractTokenFromHeader(request);
    if (!token) {
      throw new UnauthorizedException();
    }
    try {
      const payload = await this.jwtService.verifyAsync(token);
      request['user'] = payload;
    } catch {
      throw new UnauthorizedException();
    }
    return true;
  }

  private extractTokenFromHeader(request: Request): string | undefined {
    const [type, token] = request.headers.authorization?.split(' ') ?? [];
    return type === 'Bearer' ? token : undefined;
  }
}

Bây giờ chúng ta có thể triển khai route được bảo vệ và đăng ký AuthGuard để bảo vệ nó.

Mở file auth.controller.ts và cập nhật như sau:

auth.controller 
import {
  Body,
  Controller,
  Get,
  HttpCode,
  HttpStatus,
  Post,
  Request,
  UseGuards
} from '@nestjs/common';
import { AuthGuard } from './auth.guard';
import { AuthService } from './auth.service';

@Controller('auth')
export class AuthController {
  constructor(private authService: AuthService) {}

  @HttpCode(HttpStatus.OK)
  @Post('login')
  signIn(@Body() signInDto: Record<string, any>) {
    return this.authService.signIn(signInDto.username, signInDto.password);
  }

  @UseGuards(AuthGuard)
  @Get('profile')
  getProfile(@Request() req) {
    return req.user;
  }
}

Chúng ta đang áp dụng AuthGuard vừa tạo cho route GET /profile để nó được bảo vệ.

Đảm bảo ứng dụng đang chạy, và kiểm tra các route bằng cURL.

bash 
$ # GET /profile
$ curl http://localhost:3000/auth/profile
{"statusCode":401,"message":"Unauthorized"}

$ # POST /auth/login
$ curl -X POST http://localhost:3000/auth/login -d '{"username": "john", "password": "changeme"}' -H "Content-Type: application/json"
{"access_token":"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2Vybm..."}

$ # GET /profile using access_token returned from previous step as bearer code
$ curl http://localhost:3000/auth/profile -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJ1c2Vybm..."
{"sub":1,"username":"john","iat":...,"exp":...}

Lưu ý rằng trong AuthModule, chúng ta đã cấu hình JWT có thời hạn hết hạn là 60 giây. Đây là thời hạn quá ngắn, và việc xử lý chi tiết về hết hạn token và refresh nằm ngoài phạm vi của bài viết này. Tuy nhiên, chúng ta chọn như vậy để minh họa một phẩm chất quan trọng của JWT. Nếu bạn đợi 60 giây sau khi xác thực trước khi thực hiện request GET /auth/profile, bạn sẽ nhận được response 401 Unauthorized. Điều này là vì @nestjs/jwt tự động kiểm tra thời gian hết hạn của JWT, giúp bạn không phải tự làm điều đó trong ứng dụng.

Chúng ta đã hoàn thành việc triển khai JWT authentication. Các JavaScript client (như Angular/React/Vue), và các ứng dụng JavaScript khác, giờ đây có thể xác thực và giao tiếp an toàn với API Server của chúng ta.

Kích hoạt authentication toàn cục #

Nếu phần lớn các endpoint của bạn cần được bảo vệ theo mặc định, bạn có thể đăng ký authentication guard như một global guard và thay vì sử dụng decorator @UseGuards() trên mỗi controller, bạn chỉ cần đánh dấu các route nào là public.

Đầu tiên, đăng ký AuthGuard như một global guard bằng cách sử dụng cấu trúc sau (trong bất kỳ module nào, ví dụ, trong AuthModule):

TypeScript 
providers: [
  {
    provide: APP_GUARD,
    useClass: AuthGuard,
  },
],

Với điều này, Nest sẽ tự động bind AuthGuard vào tất cả endpoint.

Bây giờ chúng ta cần cung cấp một cơ chế để khai báo route là public. Để làm điều này, chúng ta có thể tạo một custom decorator bằng hàm factory SetMetadata.

TypeScript 
import { SetMetadata } from '@nestjs/common';

export const IS_PUBLIC_KEY = 'isPublic';
export const Public = () => SetMetadata(IS_PUBLIC_KEY, true);

Trong file trên, chúng ta đã export hai hằng số. Một là metadata key có tên IS_PUBLIC_KEY, và cái còn lại là decorator mới mà chúng ta sẽ gọi là Public (bạn có thể đặt tên khác là SkipAuth hoặc AllowAnon, bất cứ gì phù hợp với dự án).

Bây giờ chúng ta đã có custom decorator @Public(), chúng ta có thể sử dụng nó để decorate bất kỳ method nào, như sau:

TypeScript 
@Public()
@Get()
findAll() {
  return [];
}

Cuối cùng, chúng ta cần AuthGuard trả về true khi tìm thấy metadata "isPublic". Để làm điều này, chúng ta sẽ sử dụng class Reflector (đọc thêm tại đây).

auth/auth.guard 
@Injectable()
export class AuthGuard implements CanActivate {
  constructor(private jwtService: JwtService, private reflector: Reflector) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const isPublic = this.reflector.getAllAndOverride<boolean>(IS_PUBLIC_KEY, [
      context.getHandler(),
      context.getClass(),
    ]);
    if (isPublic) {
      return true;
    }

    const request = context.switchToHttp().getRequest();
    const token = this.extractTokenFromHeader(request);
    if (!token) {
      throw new UnauthorizedException();
    }
    try {
      const payload = await this.jwtService.verifyAsync(token);
      request['user'] = payload;
    } catch {
      throw new UnauthorizedException();
    }
    return true;
  }

  private extractTokenFromHeader(request: Request): string | undefined {
    const [type, token] = request.headers.authorization?.split(' ') ?? [];
    return type === 'Bearer' ? token : undefined;
  }
}

Tích hợp Passport #

Passport là thư viện authentication node.js phổ biến nhất, được cộng đồng biết đến rộng rãi và được sử dụng thành công trong nhiều ứng dụng production. Việc tích hợp thư viện này với ứng dụng Nest rất đơn giản khi sử dụng module @nestjs/passport.
