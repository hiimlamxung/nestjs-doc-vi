Interceptors

Interceptor là một class được annotate bằng decorator @Injectable() và implement interface NestInterceptor.

Interceptor bọc quanh cả hai phía: trước và sau khi route handler thực thi

Interceptor có một tập các khả năng hữu ích lấy cảm hứng từ kỹ thuật Aspect Oriented Programming (AOP). Chúng cho phép:

⚡ Bind thêm logic trước / sau khi method được thực thi
🔄 Transform kết quả trả về từ function
⚠️ Transform exception được throw từ function
➕ Mở rộng hành vi cơ bản của function
🔁 Override hoàn toàn một function tuỳ theo điều kiện cụ thể (ví dụ: caching)

Basics #

Mỗi interceptor implement method intercept(), nhận hai argument. Argument đầu tiên là instance ExecutionContext (cùng đối tượng như ở guards). ExecutionContext kế thừa từ ArgumentsHost — chúng ta đã thấy trong chương exception filters, nó là wrapper bao quanh các argument được truyền đến handler gốc.

Execution context #

Bằng cách extend ArgumentsHost, ExecutionContext thêm một số helper method mới cung cấp chi tiết bổ sung về quá trình thực thi hiện tại. Những chi tiết này hữu ích để build các interceptor generic có thể hoạt động trên nhiều loại controller, method và execution context. Tìm hiểu thêm về ExecutionContext tại đây.

Call handler #

Argument thứ hai là CallHandler. Interface CallHandler implement method handle(), bạn có thể dùng nó để gọi route handler method tại một điểm nào đó trong interceptor. Nếu bạn không gọi handle() trong implementation của intercept(), route handler sẽ không được thực thi.

Cách tiếp cận này có nghĩa là method intercept() hiệu quả bọc request/response stream. Kết quả là bạn có thể implement custom logic cả trước và sau khi route handler cuối cùng thực thi.

Vì handle() trả về RxJS Observable, chúng ta có nhiều operator mạnh mẽ để xử lý response. Sử dụng thuật ngữ AOP, việc gọi route handler (tức gọi handle()) được gọi là Pointcut — điểm mà logic bổ sung được chèn vào.

Ví dụ: với request POST /cats hướng đến handler create() trong CatsController — nếu một interceptor không gọi handle(), method create() sẽ không được thực thi. Khi handle() được gọi và Observable của nó được trả về, handler create() mới được kích hoạt. Và khi response stream được nhận qua Observable, các thao tác bổ sung có thể được thực hiện trên stream.

Aspect interception #

Use case đầu tiên là dùng interceptor để log tương tác của user (ví dụ: lưu user call, dispatch event bất đồng bộ hoặc tính timestamp). Dưới đây là LoggingInterceptor đơn giản:

logging.interceptor.ts
TS
JS
import { Injectable, NestInterceptor, ExecutionContext, CallHandler } from '@nestjs/common';
import { Observable } from 'rxjs';
import { tap } from 'rxjs/operators';

@Injectable()
export class LoggingInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    console.log('Before...');

    const now = Date.now();
    return next
      .handle()
      .pipe(
        tap(() => console.log(`After... ${Date.now() - now}ms`)),
      );
  }
}
import { Injectable } from '@nestjs/common';
import { Observable } from 'rxjs';
import { tap } from 'rxjs/operators';

@Injectable()
export class LoggingInterceptor {
  intercept(context, next) {
    console.log('Before...');

    const now = Date.now();
    return next
      .handle()
      .pipe(
        tap(() => console.log(`After... ${Date.now() - now}ms`)),
      );
  }
}
💡 Hint

NestInterceptor<T, R> là generic interface trong đó T biểu thị type của Observable<T> (hỗ trợ response stream), và R là type của value được wrap bởi Observable<R>.

📌 Notice

Interceptor, cũng như controller, provider, guard, v.v., có thể inject dependency qua constructor.

Vì handle() trả về RxJS Observable, chúng ta có nhiều operator để xử lý stream. Trong ví dụ trên, operator tap() gọi logging function ẩn danh khi observable stream kết thúc bình thường hoặc có exception, nhưng không can thiệp vào response cycle.

Binding interceptors #

Để thiết lập interceptor, dùng decorator @UseInterceptors() được import từ @nestjs/common. Giống như pipe và guard, interceptor có thể có scope controller-scoped, method-scoped, hoặc global-scoped.

cats.controller.ts
TS
JS
@UseInterceptors(LoggingInterceptor)
export class CatsController {}
@UseInterceptors(LoggingInterceptor)
export class CatsController {}
💡 Hint

Decorator @UseInterceptors() được import từ package @nestjs/common.

Với cấu trúc trên, mỗi route handler trong CatsController sẽ dùng LoggingInterceptor. Khi ai đó gọi GET /cats, output sẽ như sau:

Console output
Before...
After... 1ms

Chúng ta truyền class LoggingInterceptor (không phải instance), để framework đảm nhận khởi tạo và cho phép dependency injection. Cũng có thể truyền in-place instance:

cats.controller.ts
TS
JS
@UseInterceptors(new LoggingInterceptor())
export class CatsController {}
@UseInterceptors(new LoggingInterceptor())
export class CatsController {}

Để giới hạn scope của interceptor cho một method duy nhất, dùng decorator @UseInterceptors() ở cấp method. Để thiết lập global interceptor, dùng method useGlobalInterceptors():

const app = await NestFactory.create(AppModule);
app.useGlobalInterceptors(new LoggingInterceptor());

Global interceptor đăng ký từ ngoài module không thể inject dependency. Để giải quyết, dùng token APP_INTERCEPTOR từ bất kỳ module nào:

app.module.ts
TS
JS
import { Module } from '@nestjs/common';
import { APP_INTERCEPTOR } from '@nestjs/core';

@Module({
  providers: [
    {
      provide:  APP_INTERCEPTOR,
      useClass: LoggingInterceptor,
    },
  ],
})
export class AppModule {}
import { Module } from '@nestjs/common';
import { APP_INTERCEPTOR } from '@nestjs/core';

@Module({
  providers: [
    {
      provide:  APP_INTERCEPTOR,
      useClass: LoggingInterceptor,
    },
  ],
})
export class AppModule {}
💡 Hint

Khi dùng cách này, interceptor thực chất là global bất kể module nào sử dụng cấu trúc này. Nên đặt ở module nơi interceptor (LoggingInterceptor) được định nghĩa. Ngoài ra, useClass không phải cách duy nhất để đăng ký custom provider. Tìm hiểu thêm tại đây.

Response mapping #

Chúng ta đã biết handle() trả về Observable. Stream chứa value được trả về từ route handler, vì vậy chúng ta có thể dễ dàng thay đổi nó bằng operator map() của RxJS.

⚠️ Warning

Tính năng response mapping không hoạt động với library-specific response strategy (dùng trực tiếp @Res() object là bị cấm).

Hãy tạo TransformInterceptor — sẽ wrap mỗi response vào property data của một object mới:

transform.interceptor.ts
TS
JS
import { Injectable, NestInterceptor, ExecutionContext, CallHandler } from '@nestjs/common';
import { Observable } from 'rxjs';
import { map } from 'rxjs/operators';

export interface Response<T> {
  data: T;
}

@Injectable()
export class TransformInterceptor<T> implements NestInterceptor<T, Response<T>> {
  intercept(context: ExecutionContext, next: CallHandler): Observable<Response<T>> {
    return next.handle().pipe(map(data => ({ data })));
  }
}
import { Injectable } from '@nestjs/common';
import { map } from 'rxjs/operators';

@Injectable()
export class TransformInterceptor {
  intercept(context, next) {
    return next.handle().pipe(map(data => ({ data })));
  }
}
💡 Hint

Nest interceptor hoạt động với cả method intercept() đồng bộ và bất đồng bộ. Có thể chuyển sang async nếu cần.

Với cấu trúc trên, khi ai đó gọi GET /cats, response sẽ như sau (giả sử route handler trả về mảng rỗng []):

{
  "data": []
}

Interceptor có giá trị lớn trong việc tạo các giải pháp tái sử dụng cho các yêu cầu xuyên suốt toàn ứng dụng. Ví dụ: transform tất cả giá trị null thành chuỗi rỗng '' chỉ với một dòng code, bind global để tự động áp dụng cho mọi handler:

TS
JS
import { Injectable, NestInterceptor, ExecutionContext, CallHandler } from '@nestjs/common';
import { Observable } from 'rxjs';
import { map } from 'rxjs/operators';

@Injectable()
export class ExcludeNullInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next
      .handle()
      .pipe(map(value => value === null ? '' : value));
  }
}
import { Injectable } from '@nestjs/common';
import { map } from 'rxjs/operators';

@Injectable()
export class ExcludeNullInterceptor {
  intercept(context, next) {
    return next

  }

Exception mapping #

Use case thú vị khác là tận dụng operator catchError() của RxJS để override exception được throw:

errors.interceptor.ts
TS
JS
import {
  Injectable, NestInterceptor, ExecutionContext,
  BadGatewayException, CallHandler,
} from '@nestjs/common';
import { Observable, throwError } from 'rxjs';
import { catchError } from 'rxjs/operators';

@Injectable()
export class ErrorsInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next
      .handle()
      .pipe(
        catchError(err => throwError(() => new BadGatewayException())),
      );
  }
}
import { Injectable, BadGatewayException } from '@nestjs/common';
import { throwError } from 'rxjs';
import { catchError } from 'rxjs/operators';

@Injectable()
export class ErrorsInterceptor {
  intercept(context, next) {
    return next
      .handle()
      .pipe(
        catchError(err => throwError(() => new BadGatewayException())),
      );
  }
}

Stream overriding #

Đôi khi chúng ta muốn hoàn toàn ngăn việc gọi handler và trả về giá trị khác thay thế. Ví dụ điển hình là implement cache để cải thiện response time. Dưới đây là cache interceptor đơn giản trả về response từ cache:

cache.interceptor.ts
TS
JS
import { Injectable, NestInterceptor, ExecutionContext, CallHandler } from '@nestjs/common';
import { Observable, of } from 'rxjs';

@Injectable()
export class CacheInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    const isCached = true;
    if (isCached) {
      return of([]); // trả về stream mới, route handler không được gọi
    }
    return next.handle();
  }
}
import { Injectable } from '@nestjs/common';
import { of } from 'rxjs';

@Injectable()
export class CacheInterceptor {
  intercept(context, next) {
    const isCached = true;
    if (isCached) {
      return of([]);
    }
    return next.handle();
  }
}

CacheInterceptor có biến isCached và response [] được hard-code. Điểm mấu chốt: chúng ta trả về một stream mới được tạo bởi operator of() của RxJS, do đó route handler sẽ không được gọi. Để tạo giải pháp generic, có thể tận dụng Reflector và tạo custom decorator — Reflector được mô tả kỹ trong chương guards.

More operators #

Khả năng xử lý stream bằng RxJS operator cho chúng ta nhiều tính năng mạnh mẽ. Xét use case phổ biến khác — xử lý timeout trên route request. Khi endpoint không trả về gì sau một khoảng thời gian, chúng ta muốn kết thúc với error response:

timeout.interceptor.ts
TS
JS
import {
  Injectable, NestInterceptor, ExecutionContext,
  CallHandler, RequestTimeoutException,
} from '@nestjs/common';
import { Observable, throwError, TimeoutError } from 'rxjs';
import { catchError, timeout } from 'rxjs/operators';

@Injectable()
export class TimeoutInterceptor implements NestInterceptor {
  intercept(context: ExecutionContext, next: CallHandler): Observable<any> {
    return next.handle().pipe(
      timeout(5000),
      catchError(err => {
        if (err instanceof TimeoutError) {
          return throwError(() => new RequestTimeoutException());
        }
        return throwError(() => err);
      }),
    );
  }
}
import { Injectable, RequestTimeoutException } from '@nestjs/common';
import { Observable, throwError, TimeoutError } from 'rxjs';
import { catchError, timeout } from 'rxjs/operators';

@Injectable()
export class TimeoutInterceptor {
  intercept(context, next) {
    return next.handle().pipe(
      timeout(5000),
      catchError(err => {
        if (err instanceof TimeoutError) {
          return throwError(() => new RequestTimeoutException());
        }
        return throwError(() => err);
      }),
    );
  }
}

Sau 5 giây, xử lý request sẽ bị hủy. Bạn cũng có thể thêm logic tùy biến trước khi throw RequestTimeoutException (ví dụ: giải phóng tài nguyên).
