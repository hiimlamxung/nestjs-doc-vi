Pipes

Pipe là một class được annotate bằng decorator @Injectable(), implement interface PipeTransform.

Arguments

Pipe hoạt động trên argument TRƯỚC khi method được gọi

Pipe có hai trường hợp sử dụng điển hình:

🔄 Transformation — chuyển đổi input sang dạng mong muốn (ví dụ: string → integer)
✅ Validation — kiểm tra input; nếu hợp lệ thì truyền tiếp không đổi; ngược lại throw exception

Trong cả hai trường hợp, pipe hoạt động trên các arguments được xử lý bởi controller route handler. Nest đặt pipe ngay trước khi method được gọi — pipe nhận các argument dành cho method và thực hiện thao tác trên chúng. Mọi thao tác transformation hoặc validation diễn ra tại thời điểm đó, sau đó route handler mới được gọi với các argument đã (có thể) được biến đổi.

💡 Hint

Pipe chạy bên trong exceptions zone. Nghĩa là khi Pipe throw exception, nó được xử lý bởi exceptions layer. Khi exception được throw trong Pipe, không có controller method nào được thực thi sau đó — đây là kỹ thuật best-practice để validate dữ liệu đến từ nguồn bên ngoài tại system boundary.

Hãy xem nhanh cách dùng ParseIntPipe — ví dụ về trường hợp transformation, nơi pipe đảm bảo rằng parameter của method handler được convert sang JavaScript integer (hoặc throw exception nếu convert thất bại).

Binding pipes #

Để dùng pipe, chúng ta cần bind một instance của pipe class vào context phù hợp. Trong ví dụ ParseIntPipe, chúng ta muốn liên kết pipe với một route handler method cụ thể và đảm bảo nó chạy trước khi method được gọi — đây được gọi là binding pipe ở cấp method parameter:

@Get(':id')
async findOne(@Param('id', ParseIntPipe) id: number) {
  return this.catsService.findOne(id);
}

Điều này đảm bảo một trong hai điều kiện: hoặc parameter nhận được trong findOne() là number, hoặc exception được throw trước khi route handler được gọi.

Ví dụ nếu route được gọi như:

GET localhost:3000/abc

Nest sẽ throw exception như sau:

{
  "statusCode": 400,
  "message": "Validation failed (numeric string is expected)",
  "error": "Bad Request"
}

Trong ví dụ trên, chúng ta truyền class (ParseIntPipe), không phải instance, để framework đảm nhận việc khởi tạo và cho phép dependency injection. Ngoài ra, có thể truyền in-place instance để tuỳ biến hành vi của built-in pipe bằng cách truyền options:

@Get(':id')
async findOne(
  @Param('id', new ParseIntPipe({ errorHttpStatusCode: HttpStatus.NOT_ACCEPTABLE }))
  id: number,
) {
  return this.catsService.findOne(id);
}

Binding các transformation pipe khác (tất cả Parse* pipe) hoạt động tương tự. Ví dụ với query string parameter:

@Get()
async findOne(@Query('id', ParseIntPipe) id: number) {
  return this.catsService.findOne(id);
}

Đây là ví dụ dùng ParseUUIDPipe để parse string parameter và validate xem đó có phải UUID không:

TS
JS
@Get(':uuid')
async findOne(@Param('uuid', new ParseUUIDPipe()) uuid: string) {
  return this.catsService.findOne(uuid);
}
@Get(':uuid')
@Bind(Param('uuid', new ParseUUIDPipe()))
async findOne(uuid) {
  return this.catsService.findOne(uuid);
}
💡 Hint

Khi dùng ParseUUIDPipe(), bạn đang parse UUID version 3, 4 hoặc 5. Nếu chỉ yêu cầu version cụ thể, có thể truyền version vào pipe options.

Custom pipes #

Như đã đề cập, bạn có thể tự build custom pipe. Hãy build phiên bản đơn giản của ValidationPipe để xem custom pipe được xây dựng như thế nào. Ban đầu, nó chỉ nhận input value và trả về ngay giá trị đó — hoạt động như một identity function.

validation.pipe.ts
TS
JS
import { PipeTransform, Injectable, ArgumentMetadata } from '@nestjs/common';

@Injectable()
export class ValidationPipe implements PipeTransform {
  transform(value: any, metadata: ArgumentMetadata) {
    return value;
  }
}
import { Injectable } from '@nestjs/common';

@Injectable()
export class ValidationPipe {
  transform(value, metadata) {
    return value;
  }
}
💡 Hint

PipeTransform<T, R> là generic interface phải được implement bởi mọi pipe. T biểu thị type của input value, R biểu thị kiểu trả về của method transform().

Mọi pipe phải implement method transform() để hoàn thành contract của interface PipeTransform. Method này có hai parameter:

  • value — method argument hiện tại đang được xử lý (trước khi được route handling method nhận)
  • metadata — metadata của method argument đang được xử lý

Object metadata có các property sau:

export interface ArgumentMetadata {
  type: 'body' | 'query' | 'param' | 'custom';
  metatype?: Type<unknown>;
  data?: string;
}
PropertyMô tả
type Cho biết argument là body @Body(), query @Query(), param @Param(), hay custom parameter.
metatype Cung cấp metatype của argument, ví dụ String. Giá trị là undefined nếu bỏ khai báo type trong chữ ký method hoặc dùng vanilla JavaScript.
data String được truyền vào decorator, ví dụ @Body('string'). Là undefined nếu để trống dấu ngoặc của decorator.
⚠️ Warning

TypeScript interface biến mất sau khi transpile. Do đó nếu type của method parameter được khai báo là interface thay vì class, giá trị metatype sẽ là Object.

Schema based validation #

Hãy làm validation pipe hữu ích hơn. Xem xét method create() của CatsController — chúng ta muốn đảm bảo post body object hợp lệ trước khi chạy service method. Chúng ta có thể validate bên trong route handler, nhưng điều đó vi phạm single responsibility principle (SRP). Middleware validation cũng không phù hợp vì middleware không biết về execution context. Đây chính xác là use case mà pipe được thiết kế để xử lý.

Object schema validation #

Thư viện Zod cho phép tạo schema theo cách đơn giản với API dễ đọc. Hãy build validation pipe sử dụng Zod-based schema.

Cài đặt package cần thiết:

$ npm install --save zod
TS
JS
import { PipeTransform, ArgumentMetadata, BadRequestException } from '@nestjs/common';
import { ZodSchema } from 'zod';

export class ZodValidationPipe implements PipeTransform {
  constructor(private schema: ZodSchema) {}

  transform(value: unknown, metadata: ArgumentMetadata) {
    try {
      const parsedValue = this.schema.parse(value);
      return parsedValue;
    } catch (error) {
      throw new BadRequestException('Validation failed');
    }
  }
}
import { BadRequestException } from '@nestjs/common';

export class ZodValidationPipe {
  constructor(private schema) {}

  transform(value, metadata) {
    try {
      const parsedValue = this.schema.parse(value);
      return parsedValue;
    } catch (error) {
      throw new BadRequestException('Validation failed');
    }
  }
}

Binding validation pipes #

Để dùng ZodValidationPipe, chúng ta cần:

  1. Tạo instance của ZodValidationPipe
  2. Truyền context-specific Zod schema vào constructor của pipe
  3. Bind pipe vào method

Ví dụ Zod schema:

import { z } from 'zod';

export const createCatSchema = z
  .object({
    name:  z.string(),
    age:   z.number(),
    breed: z.string(),
  })
  .required();

export type CreateCatDto = z.infer<typeof createCatSchema>;

Dùng decorator @UsePipes() để bind:

cats.controller.ts
TS
JS
@Post()
@UsePipes(new ZodValidationPipe(createCatSchema))
async create(@Body() createCatDto: CreateCatDto) {
  this.catsService.create(createCatDto);
}
@Post()
@Bind(Body())
@UsePipes(new ZodValidationPipe(createCatSchema))
async create(createCatDto) {
  this.catsService.create(createCatDto);
}
💡 Hint

Decorator @UsePipes() được import từ package @nestjs/common.

⚠️ Warning

Thư viện zod yêu cầu bật cấu hình strictNullChecks trong file tsconfig.json.

Class validator #

⚠️ Warning

Các kỹ thuật trong phần này yêu cầu TypeScript và không khả dụng với vanilla JavaScript.

Nest hoạt động tốt với thư viện class-validator — cho phép dùng decorator-based validation. Kỹ thuật này đặc biệt mạnh mẽ khi kết hợp với khả năng Pipe của Nest vì chúng ta có quyền truy cập vào metatype của property đang xử lý. Cài đặt trước:

$ npm i --save class-validator class-transformer

Sau khi cài đặt, thêm decorator vào class CreateCatDto. Ưu điểm lớn ở đây: CreateCatDto vẫn là nguồn sự thật duy nhất cho Post body object:

create-cat.dto.ts
TS
JS
import { IsString, IsInt } from 'class-validator';

export class CreateCatDto {
  @IsString()
  name: string;

  @IsInt()
  age: number;

  @IsString()
  breed: string;
}
// Vanilla JS không hỗ trợ decorator-based validation
// Sử dụng TypeScript để dùng class-validator

Bây giờ tạo class ValidationPipe sử dụng các annotation này:

validation.pipe.ts
TS
JS
import { PipeTransform, Injectable, ArgumentMetadata, BadRequestException } from '@nestjs/common';
import { validate } from 'class-validator';
import { plainToInstance } from 'class-transformer';

@Injectable()
export class ValidationPipe implements PipeTransform<any> {
  async transform(value: any, { metatype }: ArgumentMetadata) {
    if (!metatype || !this.toValidate(metatype)) {
      return value;
    }
    const object = plainToInstance(metatype, value);
    const errors = await validate(object);
    if (errors.length > 0) {
      throw new BadRequestException('Validation failed');
    }
    return value;
  }

  private toValidate(metatype: Function): boolean {
    const types: Function[] = [String, Boolean, Number, Array, Object];
    return !types.includes(metatype);
  }
}
import { Injectable, BadRequestException } from '@nestjs/common';
import { validate } from 'class-validator';
import { plainToInstance } from 'class-transformer';

@Injectable()
export class ValidationPipe {
  async transform(value, { metatype }) {
    if (!metatype || !this.toValidate(metatype)) {
      return value;
    }
    const object = plainToInstance(metatype, value);
    const errors = await validate(object);
    if (errors.length > 0) {
      throw new BadRequestException('Validation failed');
    }
    return value;
  }

  toValidate(metatype) {
    const types = [String, Boolean, Number, Array, Object];
    return !types.includes(metatype);
  }
}
📌 Notice

Thư viện class-transformer được tạo bởi cùng tác giả với class-validator nên chúng phối hợp rất tốt với nhau.

Method transform() được đánh dấu async vì Nest hỗ trợ cả pipe đồng bộ lẫn bất đồng bộ — một số class-validator validation có thể là async. Chúng ta dùng plainToInstance() từ class-transformer để transform plain JavaScript argument object thành typed object để có thể áp dụng validation, vì body deserialized từ network request không có bất kỳ type information nào.

Bind ValidationPipe vào route handler @Body():

cats.controller.ts
TS
JS
@Post()
async create(
  @Body(new ValidationPipe()) createCatDto: CreateCatDto,
) {
  this.catsService.create(createCatDto);
}
@Post()
async create(
  @Body(new ValidationPipe()) createCatDto,
) {
  this.catsService.create(createCatDto);
}

Global scoped pipes #

Vì ValidationPipe được tạo để generic nhất có thể, chúng ta có thể phát huy tối đa lợi ích của nó bằng cách thiết lập như global-scoped pipe — được áp dụng cho mọi route handler trong toàn bộ ứng dụng:

main.ts
TS
JS
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.useGlobalPipes(new ValidationPipe());
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  app.useGlobalPipes(new ValidationPipe());
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();

Global pipe đăng ký từ bên ngoài module (với useGlobalPipes()) không thể inject dependency. Để giải quyết, đăng ký global pipe trực tiếp từ bất kỳ module nào:

app.module.ts
TS
JS
import { Module } from '@nestjs/common';
import { APP_PIPE } from '@nestjs/core';

@Module({
  providers: [
    {
      provide:  APP_PIPE,
      useClass: ValidationPipe,
    },
  ],
})
export class AppModule {}
import { Module } from '@nestjs/common';
import { APP_PIPE } from '@nestjs/core';

@Module({
  providers: [
    {
      provide:  APP_PIPE,
      useClass: ValidationPipe,
    },
  ],
})
export class AppModule {}

The built-in ValidationPipe #

Nhắc nhở: bạn không cần tự build generic validation pipe vì ValidationPipe đã được Nest cung cấp sẵn. Built-in ValidationPipe cung cấp nhiều option hơn ví dụ chúng ta tự build trong chương này. Xem chi tiết đầy đủ cùng nhiều ví dụ tại đây.

Transformation use case #

Validation không phải là use case duy nhất cho custom pipe. Pipe cũng có thể transform input data sang format mong muốn vì value trả về từ hàm transform hoàn toàn ghi đè giá trị cũ của argument.

Đây là ParseIntPipe đơn giản tự implement — chịu trách nhiệm parse string thành integer:

parse-int.pipe.ts
TS
JS
import { PipeTransform, Injectable, ArgumentMetadata, BadRequestException } from '@nestjs/common';

@Injectable()
export class ParseIntPipe implements PipeTransform<string, number> {
  transform(value: string, metadata: ArgumentMetadata): number {
    const val = parseInt(value, 10);
    if (isNaN(val)) {
      throw new BadRequestException('Validation failed');
    }
    return val;
  }
}
import { Injectable, BadRequestException } from '@nestjs/common';

@Injectable()
export class ParseIntPipe {
  transform(value, metadata) {
    const val = parseInt(value, 10);
    if (isNaN(val)) {
      throw new BadRequestException('Validation failed');
    }
    return val;
  }
}

Bind pipe này vào param được chọn:

TS
JS
@Get(':id')
async findOne(@Param('id', new ParseIntPipe()) id) {
  return this.catsService.findOne(id);
}
@Get(':id')
@Bind(Param('id', new ParseIntPipe()))
async findOne(id) {
  return this.catsService.findOne(id);
}

Một trường hợp transformation hữu ích khác là chọn một user entity từ database bằng id từ request:

TS
JS
@Get(':id')
findOne(@Param('id', UserByIdPipe) userEntity: UserEntity) {
  return userEntity;
}
@Get(':id')
@Bind(Param('id', UserByIdPipe))
findOne(userEntity) {
  return userEntity;
}

Pipe này nhận input id và trả về output (UserEntity object). Điều này làm cho code của bạn trở nên declarative và DRY hơn bằng cách trừu tượng hoá boilerplate ra khỏi handler và đưa vào pipe chung.

Providing defaults #

Các Parse* pipe kỳ vọng giá trị của parameter phải được định nghĩa — chúng throw exception khi nhận giá trị null hoặc undefined. Để cho phép endpoint xử lý các query string parameter bị thiếu, chúng ta phải cung cấp giá trị mặc định trước khi các Parse* pipe hoạt động. DefaultValuePipe phục vụ mục đích đó. Chỉ cần khởi tạo DefaultValuePipe trong decorator @Query() trước Parse* pipe liên quan:

TS
JS
@Get()
async findAll(
  @Query('activeOnly', new DefaultValuePipe(false), ParseBoolPipe) activeOnly: boolean,
  @Query('page', new DefaultValuePipe(0), ParseIntPipe) page: number,
) {
  return this.catsService.findAll({ activeOnly, page });
}
@Get()
@Bind(
  Query('activeOnly', new DefaultValuePipe(false), ParseBoolPipe),
  Query('page', new DefaultValuePipe(0), ParseIntPipe),
)
async findAll(activeOnly, page) {
  return this.catsService.findAll({ activeOnly, page });
}
