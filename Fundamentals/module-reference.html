Module reference

Nest cung cấp class ModuleRef để duyệt qua danh sách nội bộ các provider và lấy tham chiếu đến bất kỳ provider nào bằng cách sử dụng injection token của nó làm lookup key. Class ModuleRef cũng cung cấp cách khởi tạo động cả static và scoped provider. ModuleRef có thể được inject vào một class theo cách thông thường:

cats.service
TS
JS
@Injectable()
export class CatsService {
  constructor(private moduleRef: ModuleRef) {}
}
@Injectable()
@Dependencies(ModuleRef)
export class CatsService {
  constructor(moduleRef) {
    this.moduleRef = moduleRef;
  }
}
💡 Hint

Class ModuleRef được import từ package @nestjs/core.

Retrieving instances #

Instance ModuleRef (sau đây gọi là module reference) có method get(). Mặc định, method này trả về một provider, controller, hoặc injectable (ví dụ: guard, interceptor, v.v.) đã được đăng ký và khởi tạo trong module hiện tại bằng injection token/class name. Nếu không tìm thấy instance, một exception sẽ được throw.

cats.service
TS
JS
@Injectable()
export class CatsService implements OnModuleInit {
  private service: Service;
  constructor(private moduleRef: ModuleRef) {}

  onModuleInit() {
    this.service = this.moduleRef.get(Service);
  }
}
@Injectable()
@Dependencies(ModuleRef)
export class CatsService {
  constructor(moduleRef) {
    this.moduleRef = moduleRef;
  }

  onModuleInit() {
    this.service = this.moduleRef.get(Service);
  }
}
⚠️ Warning

Bạn không thể lấy scoped provider (transient hoặc request-scoped) bằng method get(). Thay vào đó, sử dụng kỹ thuật mô tả bên dưới. Tìm hiểu cách kiểm soát scope tại đây.

Để lấy provider từ global context (ví dụ: nếu provider đã được inject trong module khác), truyền option { strict: false } làm đối số thứ hai cho get().

this.moduleRef.get(Service, { strict: false });

Resolving scoped providers #

Để resolve động một scoped provider (transient hoặc request-scoped), sử dụng method resolve(), truyền injection token của provider làm đối số.

cats.service
TS
JS
@Injectable()
export class CatsService implements OnModuleInit {
  private transientService: TransientService;
  constructor(private moduleRef: ModuleRef) {}

  async onModuleInit() {
    this.transientService = await this.moduleRef.resolve(TransientService);
  }
}
@Injectable()
@Dependencies(ModuleRef)
export class CatsService {
  constructor(moduleRef) {
    this.moduleRef = moduleRef;
  }

  async onModuleInit() {
    this.transientService = await this.moduleRef.resolve(TransientService);
  }
}

Method resolve() trả về một instance duy nhất của provider từ DI container sub-tree riêng của nó. Mỗi sub-tree có một context identifier duy nhất. Do đó, nếu bạn gọi method này hai lần và so sánh các tham chiếu instance, bạn sẽ thấy chúng không bằng nhau.

cats.service
TS
JS
@Injectable()
export class CatsService implements OnModuleInit {
  constructor(private moduleRef: ModuleRef) {}

  async onModuleInit() {
    const transientServices = await Promise.all([
      this.moduleRef.resolve(TransientService),
      this.moduleRef.resolve(TransientService),
    ]);
    console.log(transientServices[0] === transientServices[1]); // false
  }
}
@Injectable()
@Dependencies(ModuleRef)
export class CatsService {
  constructor(moduleRef) {
    this.moduleRef = moduleRef;
  }

  async onModuleInit() {
    const transientServices = await Promise.all([
      this.moduleRef.resolve(TransientService),
      this.moduleRef.resolve(TransientService),
    ]);
    console.log(transientServices[0] === transientServices[1]); // false
  }
}

Để tạo ra một instance duy nhất qua nhiều lần gọi resolve() và đảm bảo chúng chia sẻ cùng một DI container sub-tree đã tạo, bạn có thể truyền một context identifier vào method resolve(). Sử dụng class ContextIdFactory để tạo context identifier. Class này cung cấp method create() trả về một unique identifier phù hợp.

cats.service
TS
JS
@Injectable()
export class CatsService implements OnModuleInit {
  constructor(private moduleRef: ModuleRef) {}

  async onModuleInit() {
    const contextId = ContextIdFactory.create();
    const transientServices = await Promise.all([
      this.moduleRef.resolve(TransientService, contextId),
      this.moduleRef.resolve(TransientService, contextId),
    ]);
    console.log(transientServices[0] === transientServices[1]); // true
  }
}
@Injectable()
@Dependencies(ModuleRef)
export class CatsService {
  constructor(moduleRef) {
    this.moduleRef = moduleRef;
  }

  async onModuleInit() {
    const contextId = ContextIdFactory.create();
    const transientServices = await Promise.all([
      this.moduleRef.resolve(TransientService, contextId),
      this.moduleRef.resolve(TransientService, contextId),
    ]);
    console.log(transientServices[0] === transientServices[1]); // true
  }
}
💡 Hint

Class ContextIdFactory được import từ package @nestjs/core.

Registering REQUEST provider #

Các context identifier được tạo thủ công (với ContextIdFactory.create()) đại diện cho các DI sub-tree trong đó provider REQUEST là undefined vì chúng không được khởi tạo và quản lý bởi hệ thống dependency injection của Nest.

Để đăng ký một custom REQUEST object cho DI sub-tree được tạo thủ công, sử dụng method ModuleRef#registerRequestByContextId(), như sau:

const contextId = ContextIdFactory.create();
this.moduleRef.registerRequestByContextId(/* YOUR_REQUEST_OBJECT */, contextId);

Getting current sub-tree #

Đôi khi, bạn có thể muốn resolve instance của request-scoped provider trong request context. Giả sử CatsService là request-scoped và bạn muốn resolve instance CatsRepository cũng được đánh dấu là request-scoped provider. Để chia sẻ cùng DI container sub-tree, bạn phải lấy context identifier hiện tại thay vì tạo mới. Để lấy context identifier hiện tại, bắt đầu bằng cách inject request object sử dụng decorator @Inject(REQUEST).

cats.service
TS
JS
@Injectable()
export class CatsService {
  constructor(
    @Inject(REQUEST) private request: Record<string, unknown>,
  ) {}
}
@Injectable()
@Dependencies(REQUEST)
export class CatsService {
  constructor(request) {
    this.request = request;
  }
}
💡 Hint

Tìm hiểu thêm về request provider tại đây.

Bây giờ, sử dụng method getByRequest() của class ContextIdFactory để tạo context id dựa trên request object, và truyền nó vào lệnh gọi resolve():

const contextId = ContextIdFactory.getByRequest(this.request);
const catsRepository = await this.moduleRef.resolve(CatsRepository, contextId);

Instantiating custom classes dynamically #

Để khởi tạo động một class chưa từng được đăng ký làm provider, sử dụng method create() của module reference.

cats.service
TS
JS
@Injectable()
export class CatsService implements OnModuleInit {
  private catsFactory: CatsFactory;
  constructor(private moduleRef: ModuleRef) {}

  async onModuleInit() {
    this.catsFactory = await this.moduleRef.create(CatsFactory);
  }
}
@Injectable()
@Dependencies(ModuleRef)
export class CatsService {
  constructor(moduleRef) {
    this.moduleRef = moduleRef;
  }

  async onModuleInit() {
    this.catsFactory = await this.moduleRef.create(CatsFactory);
  }
}

Kỹ thuật này cho phép bạn khởi tạo có điều kiện các class khác nhau bên ngoài framework container.
