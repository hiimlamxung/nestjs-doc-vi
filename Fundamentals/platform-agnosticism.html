Platform agnosticism

Nest là một framework platform-agnostic. Điều này có nghĩa là bạn có thể phát triển các reusable logical parts (phần logic có thể tái sử dụng) mà có thể được sử dụng trên nhiều loại ứng dụng khác nhau. Ví dụ, hầu hết các components có thể được tái sử dụng mà không cần thay đổi trên các HTTP server frameworks khác nhau (ví dụ: Express và Fastify), và thậm chí trên các loại ứng dụng khác nhau (ví dụ: HTTP server frameworks, Microservices với các transport layers khác nhau, và Web Sockets).

Build once, use everywhere #

Phần Overview của tài liệu chủ yếu trình bày các kỹ thuật lập trình sử dụng HTTP server frameworks (ví dụ: ứng dụng cung cấp REST API hoặc cung cấp ứng dụng MVC-style server-side rendered). Tuy nhiên, tất cả các building blocks đó có thể được sử dụng trên các transport layers khác nhau (microservices hoặc websockets).

Hơn nữa, Nest đi kèm với một module GraphQL chuyên dụng. Bạn có thể sử dụng GraphQL làm API layer của mình thay thế cho việc cung cấp REST API.

Ngoài ra, tính năng application context giúp tạo bất kỳ loại ứng dụng Node.js nào - bao gồm các thứ như CRON jobs và CLI apps - trên nền tảng Nest.

Nest hướng tới trở thành một platform toàn diện cho các ứng dụng Node.js, mang đến mức độ modularity và reusability cao hơn cho ứng dụng của bạn. Build once, use everywhere!
