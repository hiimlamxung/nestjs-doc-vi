HTTP module

Axios là một HTTP client package giàu tính năng và được sử dụng rộng rãi. Nest wrap Axios và expose nó thông qua HttpModule tích hợp sẵn. HttpModule export class HttpService, cung cấp các method dựa trên Axios để thực hiện HTTP request. Thư viện cũng transform các HTTP response kết quả thành Observables.

💡 Hint

Bạn cũng có thể sử dụng trực tiếp bất kỳ thư viện HTTP client Node.js nào cho mục đích chung, bao gồm got hoặc undici.

Installation #

Để bắt đầu sử dụng, trước tiên cài đặt các dependency cần thiết.

$ npm i --save @nestjs/axios axios

Getting started #

Sau khi quá trình cài đặt hoàn tất, để sử dụng HttpService, trước tiên import HttpModule.

@Module({
  imports: [HttpModule],
  providers: [CatsService],
})
export class CatsModule {}

Tiếp theo, inject HttpService sử dụng standard constructor injection.

💡 Hint

HttpModule và HttpService được import từ package @nestjs/axios.

@Injectable()
export class CatsService {
  constructor(private readonly httpService: HttpService) {}

  findAll(): Observable<AxiosResponse<Cat[]>> {
    return this.httpService.get('http://localhost:3000/cats');
  }
}
💡 Hint

AxiosResponse là một interface được export từ package axios ($ npm i axios).

Tất cả các method của HttpService đều trả về một AxiosResponse được wrap trong một object Observable.

Configuration #

Axios có thể được cấu hình với nhiều option đa dạng để tùy chỉnh behavior của HttpService. Đọc thêm về chúng tại đây. Để cấu hình Axios instance bên dưới, truyền một options object tùy chọn vào method register() của HttpModule khi import nó. Options object này sẽ được truyền trực tiếp vào Axios constructor bên dưới.

@Module({
  imports: [
    HttpModule.register({
      timeout: 5000,
      maxRedirects: 5,
    }),
  ],
  providers: [CatsService],
})
export class CatsModule {}

Async configuration #

Khi bạn cần truyền module option một cách bất đồng bộ thay vì tĩnh, sử dụng method registerAsync(). Giống như hầu hết các dynamic module, Nest cung cấp một số kỹ thuật để xử lý cấu hình async.

Một kỹ thuật là sử dụng factory function:

HttpModule.registerAsync({
  useFactory: () => ({
    timeout: 5000,
    maxRedirects: 5,
  }),
});

Giống như các factory provider khác, factory function có thể là async và có thể inject dependency thông qua inject.

HttpModule.registerAsync({
  imports: [ConfigModule],
  useFactory: async (configService: ConfigService) => ({
    timeout: configService.get('HTTP_TIMEOUT'),
    maxRedirects: configService.get('HTTP_MAX_REDIRECTS'),
  }),
  inject: [ConfigService],
});

Ngoài ra, bạn có thể cấu hình HttpModule sử dụng class thay vì factory, như bên dưới.

HttpModule.registerAsync({
  useClass: HttpConfigService,
});

Cấu trúc ở trên instantiate HttpConfigService bên trong HttpModule, sử dụng nó để tạo options object. Lưu ý rằng trong ví dụ này, HttpConfigService phải implement interface HttpModuleOptionsFactory như bên dưới. HttpModule sẽ gọi method createHttpOptions() trên object đã khởi tạo của class được cung cấp.

@Injectable()
class HttpConfigService implements HttpModuleOptionsFactory {
  createHttpOptions(): HttpModuleOptions {
    return {
      timeout: 5000,
      maxRedirects: 5,
    };
  }
}

Nếu bạn muốn tái sử dụng một options provider hiện có thay vì tạo bản sao private bên trong HttpModule, sử dụng cú pháp useExisting.

HttpModule.registerAsync({
  imports: [ConfigModule],
  useExisting: HttpConfigService,
});

Bạn cũng có thể truyền cái gọi là extraProviders vào method registerAsync(). Các provider này sẽ được merge với các module provider.

HttpModule.registerAsync({
  imports: [ConfigModule],
  useClass: HttpConfigService,
  extraProviders: [MyAdditionalProvider],
});

Điều này hữu ích khi bạn muốn cung cấp các dependency bổ sung cho factory function hoặc class constructor.

Using Axios directly #

Nếu bạn thấy rằng các option của HttpModule.register là không đủ cho bạn, hoặc nếu bạn chỉ muốn truy cập Axios instance bên dưới được tạo bởi @nestjs/axios, bạn có thể truy cập nó thông qua HttpService#axiosRef như sau:

@Injectable()
export class CatsService {
  constructor(private readonly httpService: HttpService) {}

  findAll(): Promise<AxiosResponse<Cat[]>> {
    return this.httpService.axiosRef.get('http://localhost:3000/cats');
    //              ^ AxiosInstance interface
  }
}

Full example #

Vì giá trị trả về của các method HttpService là một Observable, chúng ta có thể sử dụng rxjs - firstValueFrom hoặc lastValueFrom để lấy dữ liệu của request dưới dạng một promise.

import { catchError, firstValueFrom } from 'rxjs';

@Injectable()
export class CatsService {
  private readonly logger = new Logger(CatsService.name);
  constructor(private readonly httpService: HttpService) {}

  async findAll(): Promise<Cat[]> {
    const { data } = await firstValueFrom(
      this.httpService.get<Cat[]>('http://localhost:3000/cats').pipe(
        catchError((error: AxiosError) => {
          this.logger.error(error.response.data);
          throw 'An error happened!';
        }),
      ),
    );
    return data;
  }
}
💡 Hint

Truy cập tài liệu của RxJS về firstValueFrom và lastValueFrom để biết sự khác biệt giữa chúng.
