Serialization

Serialization là một quá trình xảy ra trước khi object được trả về trong network response. Đây là nơi thích hợp để cung cấp các rule cho việc transform và sanitize data sẽ được trả về cho client. Ví dụ, sensitive data như password nên luôn được exclude khỏi response. Hoặc, một số property nhất định có thể yêu cầu additional transformation, chẳng hạn như chỉ gửi một subset các property của entity. Thực hiện các transformation này một cách thủ công có thể tẻ nhạt và dễ lỗi, và có thể khiến bạn không chắc chắn rằng tất cả các trường hợp đã được bao phủ.

Overview #

Nest cung cấp một built-in capability để giúp đảm bảo rằng các operation này có thể được thực hiện một cách đơn giản. ClassSerializerInterceptor interceptor sử dụng package class-transformer mạnh mẽ để cung cấp cách khai báo và mở rộng cho việc transform object. Basic operation mà nó thực hiện là lấy giá trị được trả về bởi method handler và áp dụng function instanceToPlain() từ class-transformer. Khi làm như vậy, nó có thể áp dụng các rule được biểu thị bởi class-transformer decorator trên entity/DTO class, như mô tả bên dưới.

💡 Hint

Serialization không áp dụng cho StreamableFile response.

Exclude properties #

Giả sử chúng ta muốn tự động exclude property password khỏi user entity. Chúng ta annotate entity như sau:

import { Exclude } from 'class-transformer';

export class UserEntity {
  id: number;
  firstName: string;
  lastName: string;

  @Exclude()
  password: string;

  constructor(partial: Partial<UserEntity>) {
    Object.assign(this, partial);
  }
}

Bây giờ hãy xem xét một controller với method handler trả về một instance của class này.

@UseInterceptors(ClassSerializerInterceptor)
@Get()
findOne(): UserEntity {
  return new UserEntity({
    id: 1,
    firstName: 'John',
    lastName: 'Doe',
    password: 'password',
  });
}
⚠️ Warning

Lưu ý rằng chúng ta phải trả về một instance của class. Nếu bạn trả về một plain JavaScript object, ví dụ, { user: new UserEntity() }, object sẽ không được serialize đúng cách.

💡 Hint

ClassSerializerInterceptor được import từ @nestjs/common.

Khi endpoint này được request, client nhận được response sau:

{
  "id": 1,
  "firstName": "John",
  "lastName": "Doe"
}

Lưu ý rằng interceptor có thể được áp dụng cho toàn bộ application (như đã đề cập tại đây). Sự kết hợp giữa interceptor và entity class declaration đảm bảo rằng bất kỳ method nào trả về UserEntity sẽ chắc chắn loại bỏ property password. Điều này giúp bạn có một mức độ centralized enforcement cho business rule này.

Expose properties #

Bạn có thể sử dụng decorator @Expose() để cung cấp alias name cho property, hoặc để thực thi một function để tính toán property value (tương tự như getter function), như bên dưới.

@Expose()
get fullName(): string {
  return `${this.firstName} ${this.lastName}`;
}

Transform #

Bạn có thể thực hiện additional data transformation sử dụng decorator @Transform(). Ví dụ, construct sau đây trả về name property của RoleEntity thay vì trả về toàn bộ object.

@Transform(({ value }) => value.name)
role: RoleEntity;

Pass options #

Bạn có thể muốn modify default behavior của transformation function. Để override default setting, truyền chúng trong options object với decorator @SerializeOptions().

@SerializeOptions({
  excludePrefixes: ['_'],
})
@Get()
findOne(): UserEntity {
  return new UserEntity();
}
💡 Hint

Decorator @SerializeOptions() được import từ @nestjs/common.

Các option được truyền qua @SerializeOptions() được truyền dưới dạng second argument của function instanceToPlain() bên dưới. Trong ví dụ này, chúng ta tự động excluding tất cả property bắt đầu bằng prefix _.

Transform plain objects #

Bạn có thể enforce transformation ở controller level bằng cách sử dụng decorator @SerializeOptions. Điều này đảm bảo rằng tất cả response được transform thành instance của specified class, áp dụng bất kỳ decorator nào từ class-validator hoặc class-transformer, ngay cả khi plain object được trả về. Approach này giúp code sạch hơn mà không cần phải liên tục instantiate class hoặc gọi plainToInstance.

Trong ví dụ bên dưới, mặc dù trả về plain JavaScript object trong cả hai conditional branch, chúng sẽ được tự động convert thành UserEntity instance, với các relevant decorator được áp dụng:

@UseInterceptors(ClassSerializerInterceptor)
@SerializeOptions({ type: UserEntity })
@Get()
findOne(@Query() { id }: { id: number }): UserEntity {
  if (id === 1) {
    return {
      id: 1,
      firstName: 'John',
      lastName: 'Doe',
      password: 'password',
    };
  }

  return {
    id: 2,
    firstName: 'Kamil',
    lastName: 'Mysliwiec',
    password: 'password2',
  };
}
💡 Hint

Bằng cách chỉ định expected return type cho controller, bạn có thể tận dụng khả năng type-checking của TypeScript để đảm bảo rằng returned plain object tuân thủ shape của DTO hoặc entity. Function plainToInstance không cung cấp mức độ type hinting này, điều này có thể dẫn đến potential bug nếu plain object không match với expected DTO hoặc entity structure.
