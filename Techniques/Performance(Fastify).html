Performance (Fastify)

Mặc định, Nest sử dụng framework Express. Như đã đề cập trước đó, Nest cũng cung cấp khả năng tương thích với các library khác như, ví dụ, Fastify. Nest đạt được sự độc lập framework này bằng cách triển khai một framework adapter mà chức năng chính của nó là proxy middleware và handler đến các implementation tương ứng theo từng library cụ thể.

💡 Hint

Lưu ý rằng để một framework adapter được triển khai, library đích phải cung cấp pipeline xử lý request/response tương tự như được tìm thấy trong Express.

Fastify cung cấp một framework thay thế tốt cho Nest bởi vì nó giải quyết các vấn đề thiết kế theo cách tương tự như Express. Tuy nhiên, fastify nhanh hơn nhiều so với Express, đạt được kết quả benchmark gần gấp đôi. Một câu hỏi hợp lý là tại sao Nest sử dụng Express làm HTTP provider mặc định? Lý do là Express được sử dụng rộng rãi, nổi tiếng, và có một bộ middleware tương thích khổng lồ, có sẵn cho người dùng Nest ngay lập tức.

Nhưng vì Nest cung cấp sự độc lập framework, bạn có thể dễ dàng di chuyển giữa chúng. Fastify có thể là lựa chọn tốt hơn khi bạn đặt giá trị cao vào hiệu suất rất nhanh. Để sử dụng Fastify, chỉ cần chọn FastifyAdapter tích hợp sẵn như được trình bày trong chương này.

Installation #

Trước tiên, chúng ta cần cài đặt package cần thiết:

$ npm i --save @nestjs/platform-fastify

Adapter #

Khi Fastify platform đã được cài đặt, chúng ta có thể sử dụng FastifyAdapter.

main
import { NestFactory } from '@nestjs/core';
import {
  FastifyAdapter,
  NestFastifyApplication,
} from '@nestjs/platform-fastify';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter()
  );
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();

Mặc định, Fastify chỉ lắng nghe trên interface localhost 127.0.0.1 (đọc thêm). Nếu bạn muốn chấp nhận kết nối từ các host khác, bạn nên chỉ định '0.0.0.0' trong lời gọi listen():

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter(),
  );
  await app.listen(3000, '0.0.0.0');
}

Platform specific packages #

Hãy nhớ rằng khi bạn sử dụng FastifyAdapter, Nest sử dụng Fastify làm HTTP provider. Điều này có nghĩa là mỗi recipe dựa trên Express có thể không còn hoạt động. Thay vào đó, bạn nên sử dụng các package tương đương của Fastify.

Redirect response #

Fastify xử lý redirect response hơi khác so với Express. Để thực hiện redirect đúng cách với Fastify, trả về cả status code và URL, như sau:

@Get()
index(@Res() res) {
  res.status(302).redirect('/login');
}

Fastify options #

Bạn có thể truyền options vào Fastify constructor thông qua constructor của FastifyAdapter. Ví dụ:

new FastifyAdapter({ logger: true });

Middleware #

Các hàm middleware nhận các object req và res raw thay vì các wrapper của Fastify. Đây là cách mà package middie hoạt động (được sử dụng ngầm bên dưới) và fastify - xem trang này để biết thêm thông tin.

logger.middleware
import { Injectable, NestMiddleware } from '@nestjs/common';
import { FastifyRequest, FastifyReply } from 'fastify';

@Injectable()
export class LoggerMiddleware implements NestMiddleware {
  use(req: FastifyRequest['raw'], res: FastifyReply['raw'], next: () => void) {
    console.log('Request...');
    next();
  }
}
import { Injectable } from '@nestjs/common';

@Injectable()
export class LoggerMiddleware {
  use(req, res, next) {
    console.log('Request...');
    next();
  }
}

Route Config #

Bạn có thể sử dụng tính năng route config của Fastify với decorator @RouteConfig().

@RouteConfig({ output: 'hello world' })
@Get()
index(@Req() req) {
  return req.routeConfig.output;
}

Route Constraints #

Kể từ v10.3.0, @nestjs/platform-fastify hỗ trợ tính năng route constraints của Fastify với decorator @RouteConstraints.

@RouteConstraints({ version: '1.2.x' })
newFeature() {
  return 'This works only for version >= 1.2.x';
}
💡 Hint

@RouteConfig() và @RouteConstraints được import từ @nestjs/platform-fastify.
