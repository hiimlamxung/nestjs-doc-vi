File upload

Để xử lý file uploading, Nest cung cấp một module tích hợp dựa trên multer middleware package cho Express. Multer xử lý dữ liệu được post ở định dạng multipart/form-data, chủ yếu được sử dụng để upload file qua HTTP POST request. Module này hoàn toàn có thể cấu hình và bạn có thể điều chỉnh behavior của nó theo yêu cầu ứng dụng.

⚠️ Warning

Multer không thể xử lý dữ liệu không ở định dạng multipart được hỗ trợ (multipart/form-data). Ngoài ra, lưu ý rằng package này không tương thích với FastifyAdapter.

Để có type safety tốt hơn, hãy cài đặt Multer typings package:

$ npm i -D @types/multer

Với package này được cài đặt, chúng ta có thể sử dụng type Express.Multer.File (bạn có thể import type này như sau: import { Express } from 'express').

Basic example #

Để upload một file đơn lẻ, chỉ cần gắn FileInterceptor() interceptor vào route handler và extract file từ request sử dụng decorator @UploadedFile().

@Post('upload')
@UseInterceptors(FileInterceptor('file'))
uploadFile(@UploadedFile() file: Express.Multer.File) {
  console.log(file);
}
💡 Hint

Decorator FileInterceptor() được export từ package @nestjs/platform-express. Decorator @UploadedFile() được export từ @nestjs/common.

Decorator FileInterceptor() nhận hai argument:

  • fieldName: string cung cấp tên của field từ HTML form chứa file
  • options: object tùy chọn có type MulterOptions. Đây là cùng object được sử dụng bởi multer constructor (chi tiết hơn tại đây).
⚠️ Warning

FileInterceptor() có thể không tương thích với các third party cloud provider như Google Firebase hoặc các provider khác.

File validation #

Thông thường, việc validate metadata của file đến, như file size hoặc file mime-type, có thể rất hữu ích. Để làm điều này, bạn có thể tạo Pipe của riêng mình và bind nó vào parameter được annotate bằng decorator UploadedFile. Ví dụ bên dưới minh họa cách một file size validator pipe cơ bản có thể được implement:

import { PipeTransform, Injectable, ArgumentMetadata } from '@nestjs/common';

@Injectable()
export class FileSizeValidationPipe implements PipeTransform {
  transform(value: any, metadata: ArgumentMetadata) {
    // "value" is an object containing the file's attributes and metadata
    const oneKb = 1000;
    return value.size < oneKb;
  }
}

Điều này có thể được sử dụng kết hợp với FileInterceptor như sau:

@Post('file')
@UseInterceptors(FileInterceptor('file'))
uploadFileAndValidate(@UploadedFile(
  new FileSizeValidationPipe(),
  // other pipes can be added here
) file: Express.Multer.File, ) {
  return file;
}

Nest cung cấp một pipe tích hợp để xử lý các use case phổ biến và tạo thuận lợi/chuẩn hóa việc thêm mới. Pipe này có tên ParseFilePipe, và bạn có thể sử dụng nó như sau:

@Post('file')
@UseInterceptors(FileInterceptor('file'))
uploadFileAndPassValidation(
  @Body() body: SampleDto,
  @UploadedFile(
    new ParseFilePipe({
      validators: [
        // ... Set of file validator instances here
      ]
    })
  )
  file: Express.Multer.File,
) {
  return {
    body,
    file: file.buffer.toString(),
  };
}

Như bạn có thể thấy, cần phải chỉ định một mảng file validator sẽ được thực thi bởi ParseFilePipe. Chúng ta sẽ thảo luận về interface của validator, nhưng đáng lưu ý rằng pipe này cũng có hai option tùy chọn bổ sung:

  • errorHttpStatusCode: HTTP status code sẽ được throw trong trường hợp bất kỳ validator nào fail. Mặc định là 400 (BAD REQUEST)
  • exceptionFactory: Một factory nhận error message và trả về một error.

Quay lại interface FileValidator. Để tích hợp validator với pipe này, bạn phải sử dụng built-in implementation hoặc cung cấp FileValidator tùy chỉnh của riêng bạn. Xem ví dụ bên dưới:

export abstract class FileValidator<TValidationOptions = Record<string, any>> {
  constructor(protected readonly validationOptions: TValidationOptions) {}

  /**
   * Indicates if this file should be considered valid, according to the options passed in the constructor.
   * @param file the file from the request object
   */
  abstract isValid(file?: any): boolean | Promise<boolean>;

  /**
   * Builds an error message in case the validation fails.
   * @param file the file from the request object
   */
  abstract buildErrorMessage(file: any): string;
}
💡 Hint

Interface FileValidator hỗ trợ async validation thông qua function isValid. Để tận dụng type security, bạn cũng có thể type parameter file thành Express.Multer.File trong trường hợp bạn sử dụng express (mặc định) làm driver.

FileValidator là một class thông thường có quyền truy cập vào file object và validate nó theo các option được cung cấp bởi client. Nest có hai built-in FileValidator implementation bạn có thể sử dụng trong project:

  • MaxFileSizeValidator - Kiểm tra xem kích thước file có nhỏ hơn giá trị được cung cấp hay không (đo bằng bytes)
  • FileTypeValidator - Kiểm tra xem mime-type của file có khớp với string hoặc RegExp được cho hay không. Mặc định, validate mime-type sử dụng file content magic number

Để hiểu cách chúng có thể được sử dụng kết hợp với FileParsePipe đã đề cập ở trên, chúng ta sẽ sử dụng một đoạn code đã chỉnh sửa từ ví dụ cuối cùng được trình bày:

@UploadedFile(
  new ParseFilePipe({
    validators: [
      new MaxFileSizeValidator({ maxSize: 1000 }),
      new FileTypeValidator({ fileType: 'image/jpeg' }),
    ],
  }),
)
file: Express.Multer.File,
💡 Hint

Nếu số lượng validator tăng lên nhiều hoặc các option của chúng làm rối file, bạn có thể định nghĩa mảng này trong một file riêng và import nó ở đây dưới dạng named constant như fileValidators.

Cuối cùng, bạn có thể sử dụng class đặc biệt ParseFilePipeBuilder cho phép bạn compose & construct các validator. Bằng cách sử dụng nó như bên dưới, bạn có thể tránh việc khởi tạo thủ công từng validator và chỉ cần truyền trực tiếp các option của chúng:

@UploadedFile(
  new ParseFilePipeBuilder()
    .addFileTypeValidator({
      fileType: 'jpeg',
    })
    .addMaxSizeValidator({
      maxSize: 1000
    })
    .build({
      errorHttpStatusCode: HttpStatus.UNPROCESSABLE_ENTITY
    }),
)
file: Express.Multer.File,
💡 Hint

File presence được yêu cầu mặc định, nhưng bạn có thể làm cho nó optional bằng cách thêm parameter fileIsRequired: false vào trong options của function build (cùng level với errorHttpStatusCode).

Array of files #

Để upload một mảng các file (được xác định bằng một tên field duy nhất), sử dụng decorator FilesInterceptor() (lưu ý từ Files ở dạng số nhiều trong tên decorator). Decorator này nhận ba argument:

  • fieldName: như mô tả ở trên
  • maxCount: số tùy chọn xác định số lượng file tối đa được chấp nhận
  • options: object MulterOptions tùy chọn, như mô tả ở trên

Khi sử dụng FilesInterceptor(), extract file từ request sử dụng decorator @UploadedFiles() (lưu ý từ Files ở dạng số nhiều trong tên decorator).

@Post('upload')
@UseInterceptors(FilesInterceptor('files'))
uploadFile(@UploadedFiles() files: Array<Express.Multer.File>) {
  console.log(files);
}
💡 Hint

Decorator FilesInterceptor() được export từ package @nestjs/platform-express. Decorator @UploadedFiles() được export từ @nestjs/common.

Multiple files #

Để upload nhiều file (tất cả với các field name key khác nhau), sử dụng decorator FileFieldsInterceptor(). Decorator này nhận hai argument:

  • uploadedFields: một mảng các object, trong đó mỗi object chỉ định một property name bắt buộc với giá trị string chỉ định tên field, như mô tả ở trên, và một property maxCount tùy chọn, như mô tả ở trên
  • options: object MulterOptions tùy chọn, như mô tả ở trên

Khi sử dụng FileFieldsInterceptor(), extract file từ request với decorator @UploadedFiles().

@Post('upload')
@UseInterceptors(FileFieldsInterceptor([
  { name: 'avatar', maxCount: 1 },
  { name: 'background', maxCount: 1 },
]))
uploadFile(@UploadedFiles() files: { avatar?: Express.Multer.File[], background?: Express.Multer.File[] }) {
  console.log(files);
}

Any files #

Để upload tất cả các field với arbitrary field name key, sử dụng decorator AnyFilesInterceptor(). Decorator này có thể chấp nhận một object options tùy chọn như mô tả ở trên.

Khi sử dụng AnyFilesInterceptor(), extract file từ request với decorator @UploadedFiles().

@Post('upload')
@UseInterceptors(AnyFilesInterceptor())
uploadFile(@UploadedFiles() files: Array<Express.Multer.File>) {
  console.log(files);
}

No files #

Để chấp nhận multipart/form-data nhưng không cho phép upload bất kỳ file nào, sử dụng NoFilesInterceptor. Điều này đặt multipart data làm attribute trên request body. Bất kỳ file nào được gửi cùng request sẽ throw một BadRequestException.

@Post('upload')
@UseInterceptors(NoFilesInterceptor())
handleMultiPartData(@Body() body) {
  console.log(body)
}

Default options #

Bạn có thể chỉ định multer option trong file interceptor như mô tả ở trên. Để đặt option mặc định, bạn có thể gọi static method register() khi bạn import MulterModule, truyền vào các option được hỗ trợ. Bạn có thể sử dụng tất cả các option được liệt kê tại đây.

MulterModule.register({
  dest: './upload',
});
💡 Hint

Class MulterModule được export từ package @nestjs/platform-express.

Async configuration #

Khi bạn cần đặt các option MulterModule một cách bất đồng bộ thay vì tĩnh, sử dụng method registerAsync(). Giống như hầu hết các dynamic module, Nest cung cấp một số kỹ thuật để xử lý cấu hình async.

Một kỹ thuật là sử dụng factory function:

MulterModule.registerAsync({
  useFactory: () => ({
    dest: './upload',
  }),
});

Giống như các factory provider khác, factory function có thể là async và có thể inject dependency thông qua inject.

MulterModule.registerAsync({
  imports: [ConfigModule],
  useFactory: async (configService: ConfigService) => ({
    dest: configService.get<string>('MULTER_DEST'),
  }),
  inject: [ConfigService],
});

Ngoài ra, bạn có thể cấu hình MulterModule sử dụng class thay vì factory, như bên dưới:

MulterModule.registerAsync({
  useClass: MulterConfigService,
});

Cấu trúc ở trên instantiate MulterConfigService bên trong MulterModule, sử dụng nó để tạo required options object. Lưu ý rằng trong ví dụ này, MulterConfigService phải implement interface MulterOptionsFactory, như bên dưới. MulterModule sẽ gọi method createMulterOptions() trên object đã khởi tạo của class được cung cấp.

@Injectable()
class MulterConfigService implements MulterOptionsFactory {
  createMulterOptions(): MulterModuleOptions {
    return {
      dest: './upload',
    };
  }
}

Nếu bạn muốn tái sử dụng một options provider hiện có thay vì tạo bản sao private bên trong MulterModule, sử dụng cú pháp useExisting.

MulterModule.registerAsync({
  imports: [ConfigModule],
  useExisting: ConfigService,
});

Bạn cũng có thể truyền cái gọi là extraProviders vào method registerAsync(). Các provider này sẽ được merge với các module provider.

MulterModule.registerAsync({
  imports: [ConfigModule],
  useClass: ConfigService,
  extraProviders: [MyAdditionalProvider],
});

Điều này hữu ích khi bạn muốn cung cấp các dependency bổ sung cho factory function hoặc class constructor.
