Versioning

💡 Hint

Chapter này chỉ liên quan đến các HTTP-based application.

Versioning cho phép bạn có các version khác nhau của controller hoặc individual route chạy trong cùng một application. Application thay đổi rất thường xuyên và không hiếm khi có các breaking change mà bạn cần thực hiện trong khi vẫn cần hỗ trợ version trước đó của application.

Có 4 loại versioning được hỗ trợ:

URI Versioning Version sẽ được truyền trong URI của request (mặc định)
Header Versioning Một custom request header sẽ chỉ định version
Media Type Versioning Header Accept của request sẽ chỉ định version
Custom Versioning Bất kỳ khía cạnh nào của request có thể được sử dụng để chỉ định version(s). Một custom function được cung cấp để extract version(s) đó.

URI Versioning Type #

URI Versioning sử dụng version được truyền trong URI của request, chẳng hạn như https://example.com/v1/route và https://example.com/v2/route.

⚠️ Notice

Với URI Versioning, version sẽ được tự động thêm vào URI sau global path prefix (nếu có), và trước bất kỳ controller hoặc route path nào.

Để enable URI Versioning cho application của bạn, thực hiện như sau:

main.ts
JS
const app = await NestFactory.create(AppModule);
// or "app.enableVersioning()"
app.enableVersioning({
  type: VersioningType.URI,
});
await app.listen(process.env.PORT ?? 3000);
⚠️ Notice

Version trong URI sẽ được tự động prefix bằng v theo mặc định, tuy nhiên giá trị prefix có thể được cấu hình bằng cách đặt key prefix thành prefix mong muốn hoặc false nếu bạn muốn tắt nó.

💡 Hint

Enum VersioningType có sẵn để sử dụng cho property type và được import từ package @nestjs/common.

Header Versioning Type #

Header Versioning sử dụng một custom, user specified, request header để chỉ định version trong đó value của header sẽ là version được sử dụng cho request.

Ví dụ HTTP Request cho Header Versioning:

Để enable Header Versioning cho application của bạn, thực hiện như sau:

main.ts
JS
const app = await NestFactory.create(AppModule);
app.enableVersioning({
  type: VersioningType.HEADER,
  header: 'Custom-Header',
});
await app.listen(process.env.PORT ?? 3000);

Property header nên là tên của header sẽ chứa version của request.

💡 Hint

Enum VersioningType có sẵn để sử dụng cho property type và được import từ package @nestjs/common.

Media Type Versioning Type #

Media Type Versioning sử dụng header Accept của request để chỉ định version.

Trong header Accept, version sẽ được tách biệt khỏi media type bằng dấu chấm phẩy, ;. Sau đó nó nên chứa một key-value pair đại diện cho version được sử dụng cho request, chẳng hạn như Accept: application/json;v=2. They key được xử lý giống như một prefix khi xác định version sẽ được cấu hình để bao gồm key và separator.

Để enable Media Type Versioning cho application của bạn, thực hiện như sau:

main.ts
JS
const app = await NestFactory.create(AppModule);
app.enableVersioning({
  type: VersioningType.MEDIA_TYPE,
  key: 'v=',
});
await app.listen(process.env.PORT ?? 3000);

Property key nên là key và separator của key-value pair chứa version. Ví dụ Accept: application/json;v=2, property key sẽ được đặt thành v=.

💡 Hint

Enum VersioningType có sẵn để sử dụng cho property type và được import từ package @nestjs/common.

Custom Versioning Type #

Custom Versioning sử dụng bất kỳ khía cạnh nào của request để chỉ định version (hoặc versions). Incoming request được phân tích bằng cách sử dụng một extractor function trả về string hoặc array of string.

Nếu multiple version được cung cấp bởi requester, extractor function có thể trả về một array of string, được sắp xếp theo thứ tự từ greatest/highest version đến smallest/lowest version. Các version được matched với route theo thứ tự từ highest đến lowest.

Nếu một empty string hoặc array được trả về từ extractor, không có route nào được matched và 404 được trả về.

Ví dụ, nếu incoming request chỉ định nó hỗ trợ version 1, 2, và 3, extractor MUST trả về [3, 2, 1]. Điều này đảm bảo rằng highest possible route version được chọn trước.

Nếu version [3, 2, 1] được extracted, nhưng route chỉ tồn tại cho version 2 và 1, route match với version 2 được chọn (version 3 tự động bị bỏ qua).

⚠️ Notice

Việc chọn highest matching version dựa trên array trả về từ extractor > không hoạt động đáng tin cậy với Express adapter do design limitation. Một single version (hoặc string hoặc array gồm 1 element) hoạt động tốt trong Express. Fastify hỗ trợ đúng cả highest matching version selection và single version selection.

Để enable Custom Versioning cho application của bạn, tạo một extractor function và truyền nó vào application như sau:

main.ts
JS
// Example extractor that pulls out a list of versions from a custom header and turns it into a sorted array.
// This example uses Fastify, but Express requests can be processed in a similar way.
const extractor = (request: FastifyRequest): string | string[] =>
  [request.headers['custom-versioning-field'] ?? '']
    .flatMap(v => v.split(','))
    .filter(v => !!v)
    .sort()
    .reverse()

const app = await NestFactory.create(AppModule);
app.enableVersioning({
  type: VersioningType.CUSTOM,
  extractor,
});
await app.listen(process.env.PORT ?? 3000);

Usage #

Versioning cho phép bạn version controller, individual route, và cũng cung cấp cách để một số resource opt-out khỏi versioning. Cách sử dụng versioning là giống nhau bất kể Versioning Type mà application của bạn sử dụng.

⚠️ Notice

Nếu versioning được enable cho application nhưng controller hoặc route không chỉ định version, bất kỳ request nào đến controller/route đó sẽ trả về status 404. Tương tự, nếu một request được nhận chứa version không có controller hoặc route tương ứng, nó cũng sẽ trả về status 404.

Controller versions #

Một version có thể được áp dụng cho controller, thiết lập version cho tất cả các route trong controller.

Để thêm version cho controller, thực hiện như sau:

cats.controller.ts
JS
@Controller({
  version: '1',
})
export class CatsControllerV1 {
  @Get('cats')
  findAll(): string {
    return 'This action returns all cats for version 1';
  }
}

Route versions #

Một version có thể được áp dụng cho individual route. Version này sẽ override bất kỳ version nào khác có thể ảnh hưởng đến route, chẳng hạn như Controller Version.

Để thêm version cho individual route, thực hiện như sau:

cats.controller.ts
JS
import { Controller, Get, Version } from '@nestjs/common';

@Controller()
export class CatsController {
  @Version('1')
  @Get('cats')
  findAllV1(): string {
    return 'This action returns all cats for version 1';
  }

  @Version('2')
  @Get('cats')
  findAllV2(): string {
    return 'This action returns all cats for version 2';
  }
}

Multiple versions #

Multiple version có thể được áp dụng cho controller hoặc route. Để sử dụng multiple version, bạn đặt version thành một Array.

Để thêm multiple version, thực hiện như sau:

cats.controller.ts
JS
@Controller({
  version: ['1', '2'],
})
export class CatsController {
  @Get('cats')
  findAll(): string {
    return 'This action returns all cats for version 1 or 2';
  }
}

Version "Neutral" #

Một số controller hoặc route có thể không quan tâm đến version và sẽ có cùng functionality bất kể version nào. Để đáp ứng điều này, version có thể được đặt thành symbol VERSION_NEUTRAL.

Incoming request sẽ được mapped đến controller hoặc route VERSION_NEUTRAL bất kể version được gửi trong request ngoài ra nếu request không chứa version nào cả.

⚠️ Notice

Đối với URI Versioning, resource VERSION_NEUTRAL sẽ không có version trong URI.

Để thêm version neutral controller hoặc route, thực hiện như sau:

cats.controller.ts
JS
import { Controller, Get, VERSION_NEUTRAL } from '@nestjs/common';

@Controller({
  version: VERSION_NEUTRAL,
})
export class CatsController {
  @Get('cats')
  findAll(): string {
    return 'This action returns all cats regardless of version';
  }
}

Global default version #

Nếu bạn không muốn cung cấp version cho mỗi controller/hoặc individual route, hoặc nếu bạn muốn có một specific version được đặt làm default version cho mọi controller/route không có version được chỉ định, bạn có thể đặt defaultVersion như sau:

main.ts
JS
app.enableVersioning({
  // ...
  defaultVersion: '1'
  // or
  defaultVersion: ['1', '2']
  // or
  defaultVersion: VERSION_NEUTRAL
});

Middleware versioning #

Middlewares cũng có thể sử dụng versioning metadata để cấu hình middleware cho version của một specific route. Để làm điều đó, cung cấp version number làm một trong các parameter cho method MiddlewareConsumer.forRoutes():

app.module.ts
JS
import { Module, NestModule, MiddlewareConsumer } from '@nestjs/common';
import { LoggerMiddleware } from './common/middleware/logger.middleware';
import { CatsModule } from './cats/cats.module';
import { CatsController } from './cats/cats.controller';

@Module({
  imports: [CatsModule],
})
export class AppModule implements NestModule {
  configure(consumer: MiddlewareConsumer) {
    consumer
      .apply(LoggerMiddleware)
      .forRoutes({ path: 'cats', method: RequestMethod.GET, version: '2' });
  }
}

Với cấu hình trên, LoggerMiddleware sẽ chỉ được áp dụng cho version '2' của endpoint /cats.

⚠️ Notice

Middleware hoạt động với bất kỳ versioning type nào được mô tả trong section này: URI, Header, Media Type hoặc Custom.
