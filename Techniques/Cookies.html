Cookies

Một HTTP cookie là một mẩu dữ liệu nhỏ được lưu trữ bởi browser của người dùng. Cookie được thiết kế để là một cơ chế đáng tin cậy cho các website để ghi nhớ thông tin stateful. Khi người dùng truy cập lại website, cookie sẽ tự động được gửi kèm với request.

Use with Express (default) #

Trước tiên, cài đặt package cần thiết (và type của nó cho người dùng TypeScript):

$ npm i cookie-parser
$ npm i -D @types/cookie-parser

Sau khi cài đặt xong, áp dụng cookie-parser middleware như global middleware (ví dụ: trong file main.ts).

import * as cookieParser from 'cookie-parser';

// somewhere in your initialization file
app.use(cookieParser());

Bạn có thể truyền một số option cho cookieParser middleware:

  • secret một string hoặc array dùng để signing cookie. Đây là tùy chọn và nếu không được chỉ định, sẽ không parse signed cookie. Nếu một string được cung cấp, nó sẽ được dùng làm secret. Nếu một array được cung cấp, sẽ cố gắng unsign cookie với từng secret theo thứ tự.
  • options một object được truyền cho cookie.parse làm option thứ hai. Xem cookie để biết thêm thông tin.

Middleware sẽ parse Cookie header trên request và expose cookie data dưới property req.cookies và, nếu một secret được cung cấp, dưới property req.signedCookies. Các property này là các cặp name value của cookie name với cookie value.

Khi một secret được cung cấp, module này sẽ unsign và validate bất kỳ signed cookie value nào và di chuyển những name value pair đó từ req.cookies vào req.signedCookies. Một signed cookie là một cookie có giá trị được prefix với s:. Signed cookie thất bại trong việc xác thực chữ ký sẽ có giá trị false thay vì giá trị đã bị tampered.

Với thiết lập này, bạn có thể đọc cookie từ bên trong các route handler, như sau:

@Get()
findAll(@Req() request: Request) {
  console.log(request.cookies); // or "request.cookies['cookieKey']"
  // or console.log(request.signedCookies);
}
💡 Hint

Decorator @Req() được import từ @nestjs/common, trong khi Request từ package express.

Để đính kèm cookie vào outgoing response, sử dụng method Response#cookie():

@Get()
findAll(@Res({ passthrough: true }) response: Response) {
  response.cookie('key', 'value');
}
⚠️ Warning

Nếu bạn muốn để response handling logic cho framework, nhớ đặt option passthrough thành true, như đã thấy ở trên. Đọc thêm tại đây.

💡 Hint

Decorator @Res() được import từ @nestjs/common, trong khi Response từ package express.

Use with Fastify #

Trước tiên, cài đặt package cần thiết:

$ npm i @fastify/cookie

Sau khi cài đặt xong, đăng ký plugin @fastify/cookie:

import fastifyCookie from '@fastify/cookie';

// somewhere in your initialization file
const app = await NestFactory.create<NestFastifyApplication>(
  AppModule,
  new FastifyAdapter(),
);
await app.register(fastifyCookie, {
  secret: 'my-secret', // for cookies signature
});

Với thiết lập này, bạn có thể đọc cookie từ bên trong các route handler, như sau:

@Get()
findAll(@Req() request: FastifyRequest) {
  console.log(request.cookies); // or "request.cookies['cookieKey']"
}
💡 Hint

Decorator @Req() được import từ @nestjs/common, trong khi FastifyRequest từ package fastify.

Để đính kèm cookie vào outgoing response, sử dụng method FastifyReply#setCookie():

@Get()
findAll(@Res({ passthrough: true }) response: FastifyReply) {
  response.setCookie('key', 'value');
}

Để đọc thêm về method FastifyReply#setCookie(), xem trang này.

⚠️ Warning

Nếu bạn muốn để response handling logic cho framework, nhớ đặt option passthrough thành true, như đã thấy ở trên. Đọc thêm tại đây.

💡 Hint

Decorator @Res() được import từ @nestjs/common, trong khi FastifyReply từ package fastify.

Creating a custom decorator (cross-platform) #

Để cung cấp một cách thuận tiện, declarative để truy cập incoming cookie, chúng ta có thể tạo một custom decorator.

import { createParamDecorator, ExecutionContext } from '@nestjs/common';

export const Cookies = createParamDecorator(
  (data: string, ctx: ExecutionContext) => {
    const request = ctx.switchToHttp().getRequest();
    return data ? request.cookies?.[data] : request.cookies;
  },
);

Decorator @Cookies() sẽ extract tất cả cookie, hoặc một named cookie từ object req.cookies và populate decorated parameter với giá trị đó.

Với thiết lập này, chúng ta có thể sử dụng decorator trong route handler signature, như sau:

@Get()
findAll(@Cookies('name') name: string) {}
