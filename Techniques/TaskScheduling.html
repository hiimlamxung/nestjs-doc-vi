Task Scheduling

Task scheduling cho phép bạn lên lịch thực thi code tùy ý (method/function) tại một thời điểm cố định, theo chu kỳ lặp lại, hoặc một lần sau một khoảng thời gian nhất định. Trong thế giới Linux, việc này thường được xử lý bởi các package như cron ở cấp OS. Đối với ứng dụng Node.js, có một số package mô phỏng chức năng giống cron. Nest cung cấp package @nestjs/schedule, tích hợp với package cron phổ biến của Node.js. Chúng ta sẽ tìm hiểu package này trong chapter hiện tại.

Installation #

Để bắt đầu sử dụng, trước tiên chúng ta cài đặt các dependency cần thiết.

bash
$ npm install --save @nestjs/schedule

Để kích hoạt job scheduling, import ScheduleModule vào root AppModule và chạy static method forRoot() như sau:

app.module.ts
JS
import { Module } from '@nestjs/common';
import { ScheduleModule } from '@nestjs/schedule';

@Module({
  imports: [
    ScheduleModule.forRoot()
  ],
})
export class AppModule {}

Lệnh gọi .forRoot() khởi tạo scheduler và đăng ký bất kỳ declarative cron job, timeout và interval nào tồn tại bên trong app của bạn. Việc đăng ký xảy ra khi onApplicationBootstrap lifecycle hook xảy ra, đảm bảo rằng tất cả các module đã được load và khai báo bất kỳ scheduled job nào.

Declarative cron jobs #

Một cron job lên lịch thực thi một function tùy ý (method call) để chạy tự động. Cron job có thể chạy:

  • Một lần, tại một date/time xác định.
  • Theo chu kỳ lặp lại; recurring job có thể chạy tại một thời điểm xác định trong một khoảng thời gian nhất định (ví dụ: mỗi giờ một lần, mỗi tuần một lần, mỗi 5 phút một lần)

Khai báo một cron job với decorator @Cron() đặt trước method definition chứa code cần thực thi, như sau:

task.service.ts
JS
import { Injectable, Logger } from '@nestjs/common';
import { Cron } from '@nestjs/schedule';

@Injectable()
export class TasksService {
  private readonly logger = new Logger(TasksService.name);

  @Cron('45 * * * * *')
  handleCron() {
    this.logger.debug('Called when the current second is 45');
  }
}

Trong ví dụ này, method handleCron() sẽ được gọi mỗi khi giây hiện tại là 45. Nói cách khác, method sẽ chạy mỗi phút một lần, tại mốc giây thứ 45.

Decorator @Cron() hỗ trợ tất cả các cron pattern tiêu chuẩn:

  • Asterisk (ví dụ: *)
  • Ranges (ví dụ: 1-3,5)
  • Steps (ví dụ: */2)

Trong ví dụ trên, chúng ta đã truyền 45 * * * * * cho decorator. Key sau đây cho thấy cách mỗi vị trí trong cron pattern string được hiểu:

* * * * * *
| | | | | |
| | | | | day of week
| | | | months
| | | day of month
| | hours
| minutes
seconds (optional)

Một số cron pattern mẫu:

* * * * * * every second
45 * * * * * every minute, on the 45th second
0 10 * * * * every hour, at the start of the 10th minute
0 */30 9-17 * * * every 30 minutes between 9am and 5pm
0 30 11 * * 1-5 Monday to Friday at 11:30am

Package @nestjs/schedule cung cấp một enum tiện lợi chứa các cron pattern thường dùng. Bạn có thể sử dụng enum này như sau:

task.service.ts
JS
import { Injectable, Logger } from '@nestjs/common';
import { Cron, CronExpression } from '@nestjs/schedule';

@Injectable()
export class TasksService {
  private readonly logger = new Logger(TasksService.name);

  @Cron(CronExpression.EVERY_30_SECONDS)
  handleCron() {
    this.logger.debug('Called every 30 seconds');
  }
}

Trong ví dụ này, method handleCron() sẽ được gọi mỗi 30 giây. Nếu một exception xảy ra, nó sẽ được log ra console, vì mọi method được annotate với @Cron() đều được tự động wrap trong một try-catch block.

Ngoài ra, bạn có thể cung cấp một JavaScript Date object cho decorator @Cron(). Làm như vậy sẽ khiến job thực thi đúng một lần, tại date được chỉ định.

💡 Hint

Sử dụng JavaScript date arithmetic để lên lịch job tương đối với date hiện tại. Ví dụ, @Cron(new Date(Date.now() + 10 * 1000)) để lên lịch một job chạy 10 giây sau khi app khởi động.

Ngoài ra, bạn có thể cung cấp các option bổ sung làm tham số thứ hai cho decorator @Cron().

name Hữu ích để truy cập và điều khiển một cron job sau khi nó đã được khai báo.
timeZone Chỉ định timezone cho việc thực thi. Điều này sẽ thay đổi thời gian thực tế tương đối với timezone của bạn. Nếu timezone không hợp lệ, một error sẽ được throw. Bạn có thể kiểm tra tất cả các timezone có sẵn tại website Moment Timezone.
utcOffset Cho phép bạn chỉ định offset của timezone thay vì sử dụng tham số timeZone.
waitForCompletion Nếu true, không có instance bổ sung nào của cron job sẽ chạy cho đến khi onTick callback hiện tại đã hoàn thành.
disabled Cho biết liệu job có được thực thi hay không.
notification.service.ts
JS
import { Injectable } from '@nestjs/common';
import { Cron, CronExpression } from '@nestjs/schedule';

@Injectable()
export class NotificationService {
  @Cron('* * 0 * * *', {
    name: 'notifications',
    timeZone: 'Europe/Paris',
  })
  triggerNotifications() {}
}

Bạn có thể truy cập và điều khiển một cron job sau khi nó đã được khai báo, hoặc tạo động một cron job (nơi cron pattern của nó được xác định tại runtime) với Dynamic API. Để truy cập một declarative cron job thông qua API, bạn phải liên kết job với một tên bằng cách truyền property name trong một optional options object làm tham số thứ hai của decorator.

Declarative intervals #

Để khai báo rằng một method nên chạy tại một khoảng thời gian (lặp lại) xác định, đặt decorator @Interval() trước method definition. Truyền giá trị interval, dưới dạng số tính bằng millisecond, cho decorator như sau:

task.service.ts
JS
@Interval(10000)
handleInterval() {
  this.logger.debug('Called every 10 seconds');
}
💡 Hint

Cơ chế này sử dụng JavaScript setInterval() function bên trong. Bạn cũng có thể sử dụng cron job để lên lịch các recurring job.

Nếu bạn muốn điều khiển declarative interval của mình từ bên ngoài declaring class thông qua Dynamic API, hãy liên kết interval với một tên bằng cách sử dụng construction sau:

task.service.ts
JS
@Interval('notifications', 2500)
handleInterval() {}

Nếu một exception xảy ra, nó sẽ được log ra console, vì mọi method được annotate với @Interval() đều được tự động wrap trong một try-catch block.

Dynamic API cũng cho phép creating dynamic interval, nơi các property của interval được xác định tại runtime, và listing and deleting chúng.

Declarative timeouts #

Để khai báo rằng một method nên chạy (một lần) tại một timeout xác định, đặt decorator @Timeout() trước method definition. Truyền relative time offset (tính bằng milliseconds), từ thời điểm application startup, cho decorator như sau:

task.service.ts
JS
@Timeout(5000)
handleTimeout() {
  this.logger.debug('Called once after 5 seconds');
}
💡 Hint

Cơ chế này sử dụng JavaScript setTimeout() function bên trong.

Nếu một exception xảy ra, nó sẽ được log ra console, vì mọi method được annotate với @Timeout() đều được tự động wrap trong một try-catch block.

Nếu bạn muốn điều khiển declarative timeout của mình từ bên ngoài declaring class thông qua Dynamic API, hãy liên kết timeout với một tên bằng cách sử dụng construction sau:

task.service.ts
JS
@Timeout('notifications', 2500)
handleTimeout() {}

Dynamic API cũng cho phép creating dynamic timeout, nơi các property của timeout được xác định tại runtime, và listing and deleting chúng.

Dynamic schedule module API #

Module @nestjs/schedule cung cấp một dynamic API cho phép quản lý declarative cron job, timeout và interval. API cũng cho phép creating và managing dynamic cron job, timeout và interval, nơi các property được xác định tại runtime.

Dynamic cron jobs #

Lấy reference đến một CronJob instance theo tên từ bất kỳ đâu trong code bằng cách sử dụng SchedulerRegistry API. Đầu tiên, inject SchedulerRegistry bằng standard constructor injection:

constructor(private schedulerRegistry: SchedulerRegistry) {}
💡 Hint

Import SchedulerRegistry từ package @nestjs/schedule.

Sau đó sử dụng nó trong một class như sau. Giả sử một cron job đã được tạo với khai báo sau:

@Cron('* * * * * *', {
  name: 'notifications',
})
triggerNotifications() {}

Truy cập job này bằng cách sau:

const job = this.schedulerRegistry.getCronJob('notifications');

job.stop();
console.log(job.lastDate());

Method getCronJob() trả về named cron job. Object CronJob được trả về có các method sau:

  • stop() - dừng một job đã được lên lịch để chạy.
  • start() - khởi động lại một job đã bị dừng.
  • setTime(time: CronTime) - dừng một job, đặt thời gian mới cho nó, và sau đó khởi động lại nó.
  • lastDate() - trả về một biểu diễn DateTime của date mà lần thực thi cuối cùng của job đã xảy ra.
  • nextDate() - trả về một biểu diễn DateTime của date khi lần thực thi tiếp theo của job được lên lịch.
  • nextDates(count: number) - Cung cấp một array (kích thước count) of DateTime biểu diễn cho tập hợp date tiếp theo sẽ trigger job execution. count mặc định là 0, trả về một empty array.
💡 Hint

Sử dụng toJSDate() trên DateTime object để render chúng dưới dạng JavaScript Date tương đương với DateTime này.

Create một cron job mới dynamically bằng method SchedulerRegistry#addCronJob, như sau:

addCronJob(name: string, seconds: string) {
  const job = new CronJob(`${seconds} * * * * *`, () => {
    this.logger.warn(`time (${seconds}) for job ${name} to run!`);
  });

  this.schedulerRegistry.addCronJob(name, job);
  job.start();

  this.logger.warn(
    `job ${name} added for each minute at ${seconds} seconds!`,
  );
}

Trong code này, chúng ta sử dụng CronJob object từ package cron để tạo cron job. Constructor CronJob nhận một cron pattern (giống như decorator @Cron()) làm tham số đầu tiên, và một callback sẽ được thực thi khi cron timer fire làm tham số thứ hai. Method SchedulerRegistry#addCronJob nhận hai tham số: một tên cho CronJob, và chính CronJob object.

⚠️ Warning

Import CronJob từ package cron.

Delete một named cron job bằng method SchedulerRegistry#deleteCronJob, như sau: 

deleteCron(name: string) {
  this.schedulerRegistry.deleteCronJob(name);
  this.logger.warn(`job ${name} deleted!`);
}

List tất cả cron job bằng method SchedulerRegistry#getCronJobs như sau:

getCrons() {
  const jobs = this.schedulerRegistry.getCronJobs();
  jobs.forEach((value, key, map) => {
    let next;
    try {
      next = value.nextDate().toJSDate();
    } catch (e) {
      next = 'error: next fire date is in the past!';
    }
    this.logger.log(`job: ${key} -> next: ${next}`);
  });
}

Method getCronJobs() trả về một map. Trong code này, chúng ta iterate qua map và cố gắng truy cập method nextDate() của mỗi CronJob. Trong CronJob API, nếu một job đã fire xong và không có future firing date, nó sẽ throw một exception.

Dynamic intervals #

Lấy reference đến một interval bằng method SchedulerRegistry#getInterval. Như trên, inject SchedulerRegistry bằng standard constructor injection:

constructor(private schedulerRegistry: SchedulerRegistry) {}

Và sử dụng nó như sau:

const interval = this.schedulerRegistry.getInterval('notifications');
clearInterval(interval);

Create một interval mới dynamically bằng method SchedulerRegistry#addInterval, như sau:

addInterval(name: string, milliseconds: number) {
  const callback = () => {
    this.logger.warn(`Interval ${name} executing at time (${milliseconds})!`);
  };

  const interval = setInterval(callback, milliseconds);
  this.schedulerRegistry.addInterval(name, interval);
}

Trong code này, chúng ta tạo một standard JavaScript interval, sau đó truyền nó cho method SchedulerRegistry#addInterval. Method đó nhận hai tham số: một tên cho interval, và chính interval đó.

Delete một named interval bằng method SchedulerRegistry#deleteInterval, như sau: 

deleteInterval(name: string) {
  this.schedulerRegistry.deleteInterval(name);
  this.logger.warn(`Interval ${name} deleted!`);
}

List tất cả interval bằng method SchedulerRegistry#getIntervals như sau:

getIntervals() {
  const intervals = this.schedulerRegistry.getIntervals();
  intervals.forEach(key => this.logger.log(`Interval: ${key}`));
}
