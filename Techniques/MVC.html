Model-View-Controller

Nest, mặc định, sử dụng thư viện Express bên dưới. Do đó, mọi kỹ thuật sử dụng pattern MVC (Model-View-Controller) trong Express đều áp dụng được cho Nest.

Đầu tiên, hãy scaffold một ứng dụng Nest đơn giản bằng công cụ CLI:

$ npm i -g @nestjs/cli
$ nest new project

Để tạo một ứng dụng MVC, chúng ta cũng cần một template engine để render các HTML view:

$ npm install --save hbs

Chúng ta đã sử dụng engine hbs (Handlebars), mặc dù bạn có thể sử dụng bất kỳ engine nào phù hợp với yêu cầu của bạn. Sau khi quá trình cài đặt hoàn tất, chúng ta cần cấu hình express instance bằng code sau:

main.ts
import { NestFactory } from '@nestjs/core';
import { NestExpressApplication } from '@nestjs/platform-express';
import { join } from 'node:path';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create<NestExpressApplication>(
    AppModule,
  );

  app.useStaticAssets(join(__dirname, '..', 'public'));
  app.setBaseViewsDir(join(__dirname, '..', 'views'));
  app.setViewEngine('hbs');

  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();

Chúng ta đã cho Express biết rằng thư mục public sẽ được sử dụng để lưu trữ static asset, views sẽ chứa template, và template engine hbs sẽ được sử dụng để render HTML output.

Template rendering #

Bây giờ, hãy tạo một thư mục views và file template index.hbs bên trong nó. Trong template, chúng ta sẽ in một message được truyền từ controller:

views/index.hbs
<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8" />
    <title>App</title>
  </head>
  <body>
    {{ message }}
  </body>
</html>

Tiếp theo, mở file app.controller và thay thế method root() bằng code sau:

app.controller.ts
import { Get, Controller, Render } from '@nestjs/common';

@Controller()
export class AppController {
  @Get()
  @Render('index')
  root() {
    return { message: 'Hello world!' };
  }
}

Trong code này, chúng ta đang chỉ định template sẽ sử dụng trong decorator @Render(), và giá trị trả về của route handler method được truyền vào template để rendering. Lưu ý rằng giá trị trả về là một object với property message, khớp với placeholder message mà chúng ta đã tạo trong template.

Trong khi ứng dụng đang chạy, mở trình duyệt và truy cập http://localhost:3000. Bạn sẽ thấy thông báo Hello world!.

Dynamic template rendering #

Nếu application logic cần quyết định động template nào sẽ render, thì chúng ta nên sử dụng decorator @Res(), và cung cấp tên view trong route handler, thay vì trong decorator @Render():

💡 Hint

Khi Nest phát hiện decorator @Res(), nó inject object response dành riêng cho thư viện. Chúng ta có thể sử dụng object này để render template một cách dynamic. Tìm hiểu thêm về API của response object tại đây.

app.controller.ts
import { Get, Controller, Res, Render } from '@nestjs/common';
import { Response } from 'express';
import { AppService } from './app.service';

@Controller()
export class AppController {
  constructor(private appService: AppService) {}

  @Get()
  root(@Res() res: Response) {
    return res.render(
      this.appService.getViewName(),
      { message: 'Hello world!' },
    );
  }
}

Fastify #

Như đã đề cập trong chương này, chúng ta có thể sử dụng bất kỳ HTTP provider tương thích nào cùng với Nest. Một thư viện như vậy là Fastify. Để tạo một ứng dụng MVC với Fastify, chúng ta cần cài đặt các package sau:

$ npm install --save @fastify/view @fastify/static handlebars

Các bước tiếp theo bao gồm gần như cùng một quy trình được sử dụng với Express, với một số khác biệt nhỏ dành riêng cho platform. Sau khi quá trình cài đặt hoàn tất, mở file main.ts và cập nhật nội dung của nó:

main.ts
import { NestFactory } from '@nestjs/core';
import { NestFastifyApplication, FastifyAdapter } from '@nestjs/platform-fastify';
import { AppModule } from './app.module';
import { join } from 'node:path';

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter(),
  );

  app.useStaticAssets({
    root: join(__dirname, '..', 'public'),
    prefix: '/public/',
  });
  app.setViewEngine({
    engine: {
      handlebars: require('handlebars'),
    },
    templates: join(__dirname, '..', 'views'),
  });

  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();

Fastify API có một số khác biệt, nhưng kết quả cuối cùng của các lời gọi method này là giống nhau. Một khác biệt đáng chú ý là khi sử dụng Fastify, tên template mà bạn truyền vào decorator @Render() phải bao gồm phần mở rộng file.

Đây là cách bạn có thể thiết lập:

app.controller.ts
import { Get, Controller, Render } from '@nestjs/common';

@Controller()
export class AppController {
  @Get()
  @Render('index.hbs')
  root() {
    return { message: 'Hello world!' };
  }
}

Ngoài ra, bạn có thể sử dụng decorator @Res() để inject trực tiếp response và chỉ định view bạn muốn render, như bên dưới:

app.controller.ts
import { Res, Get, Controller } from '@nestjs/common';
import { FastifyReply } from 'fastify';

@Controller()
export class AppController {
  @Get()
  root(@Res() res: FastifyReply) {
    return res.view('index.hbs', { message: 'Hello world!' });
  }
}
