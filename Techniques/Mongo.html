MongoDB (Mongoose)

Nest hỗ trợ hai phương pháp tích hợp với database MongoDB. Bạn có thể dùng module TypeORM tích hợp sẵn (được mô tả tại đây) với connector cho MongoDB, hoặc dùng Mongoose — công cụ object modeling phổ biến nhất cho MongoDB. Trong chương này chúng ta sẽ mô tả cách sau, sử dụng package @nestjs/mongoose chuyên dụng.

Bắt đầu bằng cài đặt các dependency cần thiết:

$ npm i @nestjs/mongoose mongoose

Sau khi hoàn tất cài đặt, chúng ta có thể import MongooseModule vào root AppModule:

app.module.ts
TS
JS
import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';

@Module({
  imports: [MongooseModule.forRoot('mongodb://localhost/nest')],
})
export class AppModule {}
import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';

@Module({
  imports: [MongooseModule.forRoot('mongodb://localhost/nest')],
})
export class AppModule {}

Method forRoot() chấp nhận cùng configuration object như mongoose.connect() từ package Mongoose, được mô tả tại đây.

Model injection #

Với Mongoose, mọi thứ đều được derived từ một Schema. Mỗi schema map đến một MongoDB collection và định nghĩa hình dạng của các document trong collection đó. Schema được dùng để định nghĩa Model. Model chịu trách nhiệm tạo và đọc document từ underlying MongoDB database.

Schema có thể được tạo bằng NestJS decorator hoặc với Mongoose thủ công. Dùng decorator để tạo schema giảm đáng kể boilerplate và cải thiện khả năng đọc code tổng thể.

Hãy định nghĩa CatSchema:

schemas/cat.schema.ts
TS
JS
import { Prop, Schema, SchemaFactory } from '@nestjs/mongoose';
import { HydratedDocument } from 'mongoose';

export type CatDocument = HydratedDocument<Cat>;

@Schema()
export class Cat {
  @Prop()
  name: string;

  @Prop()
  age: number;

  @Prop()
  breed: string;
}

export const CatSchema = SchemaFactory.createForClass(Cat);
import { Prop, Schema, SchemaFactory } from '@nestjs/mongoose';

@Schema()
export class Cat {
  @Prop()
  name;

  @Prop()
  age;

  @Prop()
  breed;
}

export const CatSchema = SchemaFactory.createForClass(Cat);
💡 Hint

Bạn cũng có thể tạo raw schema definition bằng class DefinitionsFactory (từ nestjs/mongoose). Điều này cho phép modify thủ công schema definition được tạo dựa trên metadata bạn cung cấp. Hữu ích cho một số edge case.

Decorator @Schema() đánh dấu một class là schema definition. Nó map class Cat vào MongoDB collection cùng tên nhưng thêm "s" ở cuối — collection MongoDB cuối cùng sẽ là cats. Decorator này chấp nhận một argument duy nhất tùy chọn là schema options object. Hãy nghĩ về nó như object bạn thường truyền như argument thứ hai của constructor class mongoose.Schema (ví dụ: new mongoose.Schema(_, options)). Tìm hiểu thêm về schema option tại đây.

Decorator @Prop() định nghĩa property trong document. Ví dụ trong schema definition trên, chúng ta định nghĩa ba property: name, age, và breed. Schema type cho các property này được tự động suy ra nhờ khả năng TypeScript metadata (và reflection). Tuy nhiên, trong các trường hợp phức tạp hơn khi type không thể được reflected ngầm định (ví dụ: array hoặc nested object structure), type phải được chỉ định rõ ràng như sau:

@Prop([String])
tags: string[];

Ngoài ra, decorator @Prop() chấp nhận options object argument (đọc thêm về option có sẵn). Với nó bạn có thể chỉ định property có bắt buộc không, giá trị mặc định, hoặc đánh dấu là immutable. Ví dụ:

@Prop({ required: true })
name: string;

Nếu muốn chỉ định relation đến model khác để populate sau, có thể dùng @Prop(). Ví dụ nếu Cat có Owner được lưu trong collection khác tên owners, property nên có type và ref:

import * as mongoose from 'mongoose';
import { Owner } from '../owners/schemas/owner.schema';

// bên trong class definition
@Prop({ type: mongoose.Schema.Types.ObjectId, ref: 'Owner' })
owner: Owner;

Nếu có nhiều owner, cấu hình property như sau:

@Prop({ type: [{ type: mongoose.Schema.Types.ObjectId, ref: 'Owner' }] })
owners: Owner[];

Nếu không có ý định luôn populate reference đến collection khác, hãy xem xét dùng mongoose.Types.ObjectId làm type thay thế:

@Prop({ type: { type: mongoose.Schema.Types.ObjectId, ref: 'Owner' } })
// Đảm bảo field không bị nhầm lẫn với populated reference
owner: mongoose.Types.ObjectId;

Sau đó, khi cần selectively populate sau này, bạn có thể dùng repository function chỉ định đúng type:

import { Owner } from './schemas/owner.schema';

// ví dụ: inside a service or repository
async findAllPopulated() {
  return this.catModel.find().populate<{ owner: Owner }>("owner");
}
💡 Hint

Nếu không có foreign document để populate, type có thể là Owner | null, tuỳ thuộc vào cấu hình Mongoose. Hoặc nó có thể throw error, trong trường hợp đó type sẽ là Owner.

Cuối cùng, raw schema definition cũng có thể được truyền vào decorator. Hữu ích khi property đại diện cho nested object không được định nghĩa là class. Dùng function raw() từ package @nestjs/mongoose:

@Prop(raw({
  firstName: { type: String },
  lastName:  { type: String }
}))
details: Record<string, any>;

Ngoài ra, nếu bạn thích không dùng decorator, có thể định nghĩa schema thủ công:

export const CatSchema = new mongoose.Schema({
  name: String,
  age: Number,
  breed: String,
});

File cat.schema nằm trong thư mục cats, nơi chúng ta cũng định nghĩa CatsModule. Bạn có thể lưu schema file ở đâu tuỳ thích, nhưng chúng tôi khuyến nghị lưu gần các domain object liên quan, trong thư mục module thích hợp.

Hãy xem CatsModule:

cats.module.ts
TS
JS
import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';
import { CatsController } from './cats.controller';
import { CatsService } from './cats.service';
import { Cat, CatSchema } from './schemas/cat.schema';

  imports: [MongooseModule.forFeature([{ name: Cat.name, schema: CatSchema }])],
  controllers: [CatsController],
  providers: [CatsService],
})

import { MongooseModule } from '@nestjs/mongoose';

import { Cat, CatSchema } from './schemas/cat.schema';

MongooseModule cung cấp method forFeature() để cấu hình module, bao gồm định nghĩa model nào nên được đăng ký trong scope hiện tại. Nếu muốn dùng model trong module khác, thêm MongooseModule vào phần exports của CatsModule và import CatsModule trong module khác.

Sau khi đăng ký schema, bạn có thể inject model Cat vào CatsService bằng decorator @InjectModel():

cats.service.ts
TS
JS
import { Model } from 'mongoose';
import { Injectable } from '@nestjs/common';
import { InjectModel } from '@nestjs/mongoose';
import { Cat } from './schemas/cat.schema';
import { CreateCatDto } from './dto/create-cat.dto';

@Injectable()
export class CatsService {
  constructor(@InjectModel(Cat.name) private catModel: Model<Cat>) {}

  async create(createCatDto: CreateCatDto): Promise<Cat> {
    const createdCat = new this.catModel(createCatDto);
    return createdCat.save();
  }

  async findAll(): Promise<Cat[]> {
    return this.catModel.find().exec();
  }
}
import { Model } from 'mongoose';
import { Injectable, Dependencies } from '@nestjs/common';
import { getModelToken } from '@nestjs/mongoose';
import { Cat } from './schemas/cat.schema';

@Injectable()
@Dependencies(getModelToken(Cat.name))
export class CatsService {
  constructor(catModel) {
    this.catModel = catModel;
  }

  async create(createCatDto) {
    const createdCat = new this.catModel(createCatDto);
    return createdCat.save();
  }

  async findAll() {
    return this.catModel.find().exec();
  }
}

Connection #

Đôi khi bạn cần truy cập native Mongoose Connection object. Ví dụ: bạn có thể muốn thực hiện native API call trên connection object. Bạn có thể inject Mongoose Connection bằng decorator @InjectConnection():

import { Injectable } from '@nestjs/common';
import { InjectConnection } from '@nestjs/mongoose';
import { Connection } from 'mongoose';

@Injectable()
export class CatsService {
  constructor(@InjectConnection() private connection: Connection) {}
}

Sessions #

Để bắt đầu session với Mongoose, khuyến nghị inject database connection bằng @InjectConnection thay vì gọi mongoose.startSession() trực tiếp. Cách này tích hợp tốt hơn với NestJS dependency injection system, đảm bảo proper connection management.

Đây là ví dụ cách bắt đầu session:

import { InjectConnection } from '@nestjs/mongoose';
import { Connection } from 'mongoose';

@Injectable()
export class CatsService {
  constructor(@InjectConnection() private readonly connection: Connection) {}

  async startTransaction() {
    const session = await this.connection.startSession();
    session.startTransaction();
    // Logic transaction của bạn ở đây
  }
}

Trong ví dụ này, @InjectConnection() được dùng để inject Mongoose connection vào service. Sau khi connection được inject, bạn có thể dùng connection.startSession() để bắt đầu session mới. Session này có thể được dùng để quản lý database transaction, đảm bảo atomic operation trên nhiều query. Sau khi start session, nhớ commit hoặc abort transaction dựa trên logic của bạn.

Multiple databases #

Một số project yêu cầu nhiều database connection. Điều này cũng có thể đạt được với module này. Để làm việc với nhiều connection, đầu tiên tạo các connection. Trong trường hợp này, đặt tên connection là bắt buộc.

app.module.ts
TS
JS
import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';

@Module({
  imports: [
    MongooseModule.forRoot('mongodb://localhost/test', {
      connectionName: 'cats',
    }),
    MongooseModule.forRoot('mongodb://localhost/users', {
      connectionName: 'users',
    }),
  ],
})
export class AppModule {}
import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';

@Module({
  imports: [
    MongooseModule.forRoot('mongodb://localhost/test', {
      connectionName: 'cats',
    }),
    MongooseModule.forRoot('mongodb://localhost/users', {
      connectionName: 'users',
    }),
  ],
})
export class AppModule {}
📌 Notice

Lưu ý: bạn không nên có nhiều connection mà không có tên, hoặc có cùng tên, nếu không chúng sẽ bị override.

Với setup này, bạn phải chỉ định connection nào sẽ được dùng cho function MongooseModule.forFeature():

@Module({
  imports: [
    MongooseModule.forFeature([{ name: Cat.name, schema: CatSchema }], 'cats'),
  ],
})
export class CatsModule {}

Bạn cũng có thể inject Connection cho connection cụ thể:

import { Injectable } from '@nestjs/common';
import { InjectConnection } from '@nestjs/mongoose';
import { Connection } from 'mongoose';

@Injectable()
export class CatsService {
  constructor(@InjectConnection('cats') private connection: Connection) {}
}

Để inject Connection cho custom provider (ví dụ: factory provider), dùng function getConnectionToken() truyền tên connection làm argument:

{
  provide: CatsService,
  useFactory: (catsConnection: Connection) => {
    return new CatsService(catsConnection);
  },
  inject: [getConnectionToken('cats')],
}

Nếu bạn chỉ muốn inject model từ named database, có thể dùng tên connection làm parameter thứ hai cho decorator @InjectModel():

cats.service.ts
TS
JS
@Injectable()
export class CatsService {
  constructor(@InjectModel(Cat.name, 'cats') private catModel: Model<Cat>) {}
}
@Injectable()
@Dependencies(getModelToken(Cat.name, 'cats'))
export class CatsService {
  constructor(catModel) {
    this.catModel = catModel;
  }
}

Hooks (middleware) #

Middleware (còn gọi là pre và post hook) là function được truyền control trong quá trình thực thi async function. Middleware được chỉ định ở schema level và hữu ích để viết plugin (nguồn). Gọi pre() hoặc post() sau khi compile model không hoạt động trong Mongoose. Để đăng ký hook trước khi model được đăng ký, dùng method forFeatureAsync() của MongooseModule cùng với factory provider (tức useFactory). Với kỹ thuật này, bạn có thể truy cập schema object, rồi dùng method pre() hoặc post() để đăng ký hook trên schema đó. Xem ví dụ dưới đây:

@Module({
  imports: [
    MongooseModule.forFeatureAsync([
      {
        name: Cat.name,
        useFactory: () => {
          const schema = CatsSchema;
          schema.pre('save', function () {
            console.log('Hello from pre save');
          });
          return schema;
        },
      },
    ]),
  ],
})
export class AppModule {}

Giống như các factory provider khác, factory function có thể là async và có thể inject dependency qua inject:

@Module({
  imports: [
    MongooseModule.forFeatureAsync([
      {
        name: Cat.name,
        imports: [ConfigModule],
        useFactory: (configService: ConfigService) => {
          const schema = CatsSchema;
          schema.pre('save', function() {
            console.log(
              `${configService.get('APP_NAME')}: Hello from pre save`,
            ),
          });
          return schema;
        },
        inject: [ConfigService],
      },
    ]),
  ],
})
export class AppModule {}

Plugins #

Để đăng ký plugin cho schema cụ thể, dùng method forFeatureAsync():

@Module({
  imports: [
    MongooseModule.forFeatureAsync([
      {
        name: Cat.name,
        useFactory: () => {
          const schema = CatsSchema;
          schema.plugin(require('mongoose-autopopulate'));
          return schema;
        },
      },
    ]),
  ],
})
export class AppModule {}

Để đăng ký plugin cho tất cả schema cùng lúc, gọi method .plugin() của Connection object. Bạn nên truy cập connection trước khi model được tạo; để làm điều này, dùng connectionFactory:

JS
import { Module } from '@nestjs/common';

@Module({
  imports: [
    MongooseModule.forRoot('mongodb://localhost/test', {

@Module({

Discriminators #

Discriminator là cơ chế schema inheritance. Chúng cho phép bạn có nhiều model với overlapping schema trên cùng underlying MongoDB collection.

Giả sử bạn muốn track các loại event khác nhau trong một collection duy nhất. Mỗi event sẽ có timestamp:

event.schema.ts
TS
JS
@Schema({ discriminatorKey: 'kind' })
export class Event {
  @Prop({
    type: String,
    required: true,
    enum: [ClickedLinkEvent.name, SignUpEvent.name],
  })
  kind: string;

  @Prop({ type: Date, required: true })
  time: Date;
}

export const EventSchema = SchemaFactory.createForClass(Event);
@Schema({ discriminatorKey: 'kind' })
export class Event {
  @Prop({
    type: String,
    required: true,
    enum: [ClickedLinkEvent.name, SignUpEvent.name],
  })
  kind;

  @Prop({ type: Date, required: true })
  time;
}

export const EventSchema = SchemaFactory.createForClass(Event);
💡 Hint

Cách mongoose phân biệt các discriminator model khác nhau là qua "discriminator key", mặc định là __t. Mongoose thêm một String path gọi là __t vào schema để track document này là instance của discriminator nào. Bạn cũng có thể dùng option discriminatorKey để định nghĩa path cho discrimination.

Instance SignedUpEvent và ClickedLinkEvent sẽ được lưu trong cùng collection như generic event.

Bây giờ định nghĩa class ClickedLinkEvent:

JS

@Schema()

export const ClickedLinkEventSchema = SchemaFactory.createForClass(ClickedLinkEvent);

        schema: EventSchema,

          { name: ClickedLinkEvent.name, schema: ClickedLinkEventSchema },

Testing #

Khi unit test một ứng dụng, chúng ta thường muốn tránh mọi database connection, làm cho test suite đơn giản hơn để setup và nhanh hơn để thực thi. Nhưng class của chúng ta có thể phụ thuộc vào model được kéo từ connection instance. Làm sao resolve các class này? Giải pháp là tạo mock model.

Để dễ dàng hơn, package @nestjs/mongoose expose một function getModelToken() trả về một injection token đã chuẩn bị dựa trên token name. Dùng token này, bạn có thể dễ dàng cung cấp mock implementation bằng bất kỳ kỹ thuật custom provider tiêu chuẩn nào, bao gồm useClass, useValue, và useFactory. Ví dụ:

@Module({
  providers: [
    CatsService,
    {
      provide: getModelToken(Cat.name),
      useValue: catModel,
    },
  ],
})
export class CatsModule {}

Trong ví dụ này, một hardcoded catModel (object instance) sẽ được cung cấp bất cứ khi nào consumer nào inject Model<Cat> bằng decorator @InjectModel().

Async configuration #

Khi bạn cần truyền module option bất đồng bộ thay vì tĩnh, dùng method forRootAsync(). Giống như hầu hết dynamic module, Nest cung cấp nhiều kỹ thuật để xử lý async configuration.

Một kỹ thuật là dùng factory function:

MongooseModule.forRootAsync({
  useFactory: () => ({
    uri: 'mongodb://localhost/nest',
  }),
});

Giống như các factory provider khác, factory function có thể là async và có thể inject dependency qua inject:

MongooseModule.forRootAsync({
  imports: [ConfigModule],
  useFactory: async (configService: ConfigService) => ({
    uri: configService.get<string>('MONGODB_URI'),
  }),
  inject: [ConfigService],
});

Ngoài ra, bạn có thể cấu hình MongooseModule bằng class thay vì factory, như dưới đây:

MongooseModule.forRootAsync({
  useClass: MongooseConfigService,
});

Cấu trúc trên khởi tạo MongooseConfigService bên trong MongooseModule, dùng nó để tạo required options object. Lưu ý trong ví dụ này, MongooseConfigService phải implement interface MongooseOptionsFactory, như dưới đây. MongooseModule sẽ gọi method createMongooseOptions() trên instantiated object của class được cung cấp:

@Injectable()
export class MongooseConfigService implements MongooseOptionsFactory {
  createMongooseOptions(): MongooseModuleOptions {
    return {
      uri: 'mongodb://localhost/nest',
    };
  }
}

Nếu muốn tái sử dụng existing options provider thay vì tạo private copy trong MongooseModule, dùng syntax useExisting:

MongooseModule.forRootAsync({
  imports: [ConfigModule],
  useExisting: ConfigService,
});

Connection events #

Bạn có thể lắng nghe connection event của Mongoose bằng cách dùng option onConnectionCreate. Điều này cho phép bạn implement custom logic bất cứ khi nào connection được thiết lập. Ví dụ, bạn có thể đăng ký event listener cho các event connected, open, disconnected, reconnected, và disconnecting, như dưới đây:

MongooseModule.forRoot('mongodb://localhost/test', {
  onConnectionCreate: (connection: Connection) => {
    connection.on('connected', () => console.log('connected'));
    connection.on('open', () => console.log('open'));
    connection.on('disconnected', () => console.log('disconnected'));
    connection.on('reconnected', () => console.log('reconnected'));
    connection.on('disconnecting', () => console.log('disconnecting'));

    return connection;
  },
}),

Trong code snippet này, chúng ta đang thiết lập connection đến MongoDB database tại mongodb://localhost/test. Option onConnectionCreate cho phép bạn setup event listener cụ thể để monitor status của connection:

  • connected: Được trigger khi connection được thiết lập thành công.
  • open: Fire khi connection đã mở hoàn toàn và sẵn sàng cho operation.
  • disconnected: Được gọi khi connection bị mất.
  • reconnected: Được invoke khi connection được thiết lập lại sau khi bị disconnect.
  • disconnecting: Xảy ra khi connection đang trong quá trình đóng.

Bạn cũng có thể kết hợp property onConnectionCreate vào async configuration được tạo bằng MongooseModule.forRootAsync():

MongooseModule.forRootAsync({
  useFactory: () => ({
    uri: 'mongodb://localhost/test',
    onConnectionCreate: (connection: Connection) => {
      // Đăng ký event listener ở đây
      return connection;
    },
  }),
}),

Điều này cung cấp cách linh hoạt để quản lý connection event, cho phép bạn xử lý thay đổi trong connection status một cách hiệu quả.

Để nest subdocument trong parent document, bạn có thể định nghĩa schema như sau:

  @Prop()

Rồi reference subdocument trong parent schema:

person.schema.ts
TS
JS
@Schema()
export class Person {
  @Prop(NameSchema)
  name: Name;
}

export const PersonSchema = SchemaFactory.createForClass(Person);

export type PersonDocumentOverride = {
  name: Types.Subdocument<Types.ObjectId> & Name;
};

export type PersonDocument = HydratedDocument<Person, PersonDocumentOverride>;
@Schema()
export class Person {
  @Prop(NameSchema)
  name;
}

export const PersonSchema = SchemaFactory.createForClass(Person);

Nếu muốn bao gồm nhiều subdocument, có thể dùng array của subdocument. Quan trọng là phải override type của property phù hợp:

person.schema.ts
TS
JS
@Schema()
export class Person {
  @Prop([NameSchema])
  name: Name[];
}

export const PersonSchema = SchemaFactory.createForClass(Person);

export type PersonDocumentOverride = {
  name: Types.DocumentArray<Name>;
};

export type PersonDocument = HydratedDocument<Person, PersonDocumentOverride>;
@Schema()
export class Person {
  @Prop([NameSchema])
  name;
}

export const PersonSchema = SchemaFactory.createForClass(Person);

Virtuals #

Trong Mongoose, một virtual là property tồn tại trên document nhưng không được persist vào MongoDB. Nó không được lưu trong database mà được tính toán động bất cứ khi nào được truy cập. Virtual thường được dùng cho derived hoặc computed value, như kết hợp field (ví dụ: tạo property fullName bằng cách concatenate firstName và lastName), hoặc để tạo property dựa trên data có sẵn trong document.

class Person {
  @Prop()
  firstName: string;

  @Prop()
  lastName: string;

  @Virtual({
    get: function (this: Person) {
      return `${this.firstName} ${this.lastName}`;
    },
  })
  fullName: string;
}
💡 Hint

Decorator @Virtual() được import từ package @nestjs/mongoose.

Trong ví dụ này, virtual fullName được derive từ firstName và lastName. Mặc dù nó hoạt động như normal property khi được truy cập, nó không bao giờ được lưu vào MongoDB document.
