Session

HTTP session cung cấp cách lưu trữ thông tin về user qua nhiều request, điều này đặc biệt hữu ích cho các ứng dụng MVC.

Use with Express (default) #

Trước tiên, cài đặt package cần thiết (và type của nó cho người dùng TypeScript):

$ npm i express-session
$ npm i -D @types/express-session

Sau khi cài đặt xong, apply express-session middleware như global middleware (ví dụ, trong file main.ts của bạn).

import * as session from 'express-session';

// somewhere in your initialization file
app.use(
  session({
    secret: 'my-secret',
    resave: false,
    saveUninitialized: false,
  }),
);
⚠️ Notice

Server-side session storage mặc định được thiết kế có chủ đích là không dành cho môi trường production. Nó sẽ bị rò rỉ bộ nhớ trong hầu hết các điều kiện, không scale được qua một process duy nhất, và chỉ dành cho debugging và developing. Đọc thêm trong official repository.

secret được sử dụng để sign session ID cookie. Đây có thể là một string cho single secret, hoặc một array gồm nhiều secret. Nếu một array các secret được cung cấp, chỉ phần tử đầu tiên sẽ được sử dụng để sign session ID cookie, trong khi tất cả các phần tử sẽ được xem xét khi verifying signature trong request. Bản thân secret không nên dễ dàng bị đoán bởi con người và tốt nhất nên là một tập hợp ngẫu nhiên các ký tự.

Bật option resave buộc session được lưu lại vào session store, ngay cả khi session không bao giờ bị sửa đổi trong suốt request. Giá trị mặc định là true, nhưng việc sử dụng giá trị mặc định đã bị deprecated, vì giá trị mặc định sẽ thay đổi trong tương lai.

Tương tự, bật option saveUninitialized buộc một session "chưa được khởi tạo" được lưu vào store. Một session là chưa được khởi tạo khi nó mới nhưng chưa được sửa đổi. Chọn false hữu ích cho việc implement login session, giảm sử dụng storage trên server, hoặc tuân thủ luật yêu cầu phải xin phép trước khi set cookie. Chọn false cũng sẽ giúp xử lý race condition khi client thực hiện nhiều request song song mà không có session (source).

Bạn có thể truyền nhiều option khác vào session middleware, đọc thêm về chúng trong tài liệu API.

💡 Hint

Xin lưu ý rằng secure: true là option được khuyến nghị. Tuy nhiên, nó yêu cầu website hỗ trợ https, tức là HTTPS là cần thiết cho secure cookie. Nếu secure được set, và bạn truy cập site qua HTTP, cookie sẽ không được set. Nếu bạn đặt node.js phía sau proxy và đang sử dụng secure: true, bạn cần set "trust proxy" trong express.

Với cấu hình này, bạn bây giờ có thể set và đọc session value từ bên trong route handler, như sau:

@Get()
findAll(@Req() request: Request) {
  request.session.visits = request.session.visits ? request.session.visits + 1 : 1;
}
💡 Hint

Decorator @Req() được import từ @nestjs/common, trong khi Request từ package express. Property request.session không khả dụng nếu bạn chưa apply session middleware. Ngoài ra, nếu bạn sử dụng TypeScript, bạn có thể cần extend type của session object bằng cách thêm property mới vào interface SessionData (tìm hiểu thêm tại đây).

Ngoài ra, bạn có thể sử dụng decorator @Session() để extract session object từ request, như sau:

@Get()
findAll(@Session() session: Record<string, any>) {
  session.visits = session.visits ? session.visits + 1 : 1;
}
💡 Hint

Decorator @Session() được import từ package @nestjs/common.

Use with Fastify #

Trước tiên, cài đặt package cần thiết:

$ npm i @fastify/secure-session

Sau khi cài đặt xong, register plugin fastify-secure-session:

import secureSession from '@fastify/secure-session';

// somewhere in your initialization file
const app = await NestFactory.create<NestFastifyApplication>(
  AppModule,
  new FastifyAdapter(),
);
await app.register(secureSession, {
  secret: 'averylogphrasebiggerthanthirtytwochars',
  salt: 'mq9hDxBVDbdspDR6n',
});
💡 Hint

Bạn cũng có thể tạo trước (pregenerate) một key (xem hướng dẫn) hoặc sử dụng keys rotation.

Đọc thêm về các option có sẵn trong official repository.

Với cấu hình này, bạn bây giờ có thể set và đọc session value từ bên trong route handler, như sau:

@Get()
findAll(@Req() request: FastifyRequest) {
  const visits = request.session.get('visits');
  request.session.set('visits', visits ? visits + 1 : 1);
}

Ngoài ra, bạn có thể sử dụng decorator @Session() để extract session object từ request, như sau:

@Get()
findAll(@Session() session: secureSession.Session) {
  const visits = session.get('visits');
  session.set('visits', visits ? visits + 1 : 1);
}
💡 Hint

Decorator @Session() được import từ @nestjs/common, trong khi secureSession.Session từ package @fastify/secure-session (import statement: import * as secureSession from '@fastify/secure-session').
