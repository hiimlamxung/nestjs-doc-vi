Events

Event Emitter package (@nestjs/event-emitter) cung cấp một observer implementation đơn giản, cho phép bạn subscribe và listen các event khác nhau xảy ra trong ứng dụng của bạn. Event đóng vai trò là một cách tuyệt vời để decouple các khía cạnh khác nhau của ứng dụng, vì một event đơn lẻ có thể có nhiều listener không phụ thuộc lẫn nhau.

📘 Note

EventEmitterModule sử dụng nội bộ package eventemitter2.

Getting started #

Trước tiên, cài đặt package cần thiết:

$ npm i --save @nestjs/event-emitter

Sau khi cài đặt xong, import EventEmitterModule vào root AppModule và chạy static method forRoot() như bên dưới:

app.module.ts
import { Module } from '@nestjs/common';
import { EventEmitterModule } from '@nestjs/event-emitter';

@Module({
  imports: [
    EventEmitterModule.forRoot()
  ],
})
export class AppModule {}

Lệnh gọi .forRoot() khởi tạo event emitter và đăng ký bất kỳ declarative event listener nào tồn tại trong ứng dụng của bạn. Việc đăng ký xảy ra khi lifecycle hook onApplicationBootstrap được thực thi, đảm bảo rằng tất cả các module đã được load và khai báo bất kỳ scheduled job nào.

Để cấu hình EventEmitter instance bên dưới, truyền configuration object vào method .forRoot(), như sau:

EventEmitterModule.forRoot({
  // set this to `true` to use wildcards
  wildcard: false,
  // the delimiter used to segment namespaces
  delimiter: '.',
  // set this to `true` if you want to emit the newListener event
  newListener: false,
  // set this to `true` if you want to emit the removeListener event
  removeListener: false,
  // the maximum amount of listeners that can be assigned to an event
  maxListeners: 10,
  // show event name in memory leak message when more than maximum amount of listeners is assigned
  verboseMemoryLeak: false,
  // disable throwing uncaughtException if an error event is emitted and it has no listeners
  ignoreErrors: false,
});

Dispatching events #

Để dispatch (tức là fire) một event, trước tiên inject EventEmitter2 sử dụng standard constructor injection:

constructor(private eventEmitter: EventEmitter2) {}
💡 Hint

Import EventEmitter2 từ package @nestjs/event-emitter.

Sau đó sử dụng nó trong class như sau:

this.eventEmitter.emit(
  'order.created',
  new OrderCreatedEvent({
    orderId: 1,
    payload: {},
  }),
);

Listening to events #

Để khai báo một event listener, trang trí method với decorator @OnEvent() đặt trước định nghĩa method chứa code sẽ được thực thi, như sau:

@OnEvent('order.created')
handleOrderCreatedEvent(payload: OrderCreatedEvent) {
  // handle and process "OrderCreatedEvent" event
}
⚠️ Warning

Event subscriber không thể là request-scoped.

Argument đầu tiên có thể là một string hoặc symbol cho simple event emitter và một string | symbol | Array<string | symbol> trong trường hợp wildcard emitter.

Argument thứ hai (tùy chọn) là một listener options object như sau:

export type OnEventOptions = OnOptions & {
  /**
   * If "true", prepends (instead of append) the given listener to the array of listeners.
   *
   * @see https://github.com/EventEmitter2/EventEmitter2#emitterprependlistenerevent-listener-options
   *
   * @default false
   */
  prependListener?: boolean;

  /**
   * If "true", the onEvent callback will not throw an error while handling the event. Otherwise, if "false" it will throw an error.
   *
   * @default true
   */
  suppressErrors?: boolean;
};
💡 Hint

Đọc thêm về OnOptions options object từ eventemitter2.

@OnEvent('order.created', { async: true })
handleOrderCreatedEvent(payload: OrderCreatedEvent) {
  // handle and process "OrderCreatedEvent" event
}

Để sử dụng namespace/wildcard, truyền option wildcard vào method EventEmitterModule#forRoot(). Khi namespace/wildcard được bật, event có thể là string (foo.bar) được phân cách bởi delimiter hoặc array (['foo', 'bar']). Delimiter cũng có thể cấu hình được như một configuration property (delimiter). Với tính năng namespace được bật, bạn có thể subscribe event sử dụng wildcard:

@OnEvent('order.*')
handleOrderEvents(payload: OrderCreatedEvent | OrderRemovedEvent | OrderUpdatedEvent) {
  // handle and process an event
}

Lưu ý rằng wildcard như vậy chỉ áp dụng cho một block. Argument order.* sẽ match, ví dụ, các event order.created và order.shipped nhưng không match order.delayed.out_of_stock. Để listen các event như vậy, sử dụng multilevel wildcard pattern (tức là **), được mô tả trong tài liệu EventEmitter2.

Với pattern này, bạn có thể, ví dụ, tạo một event listener bắt tất cả các event.

@OnEvent('**')
handleEverything(payload: any) {
  // handle and process an event
}
💡 Hint

Class EventEmitter2 cung cấp một số method hữu ích để tương tác với event, như waitFor và onAny. Bạn có thể đọc thêm về chúng tại đây.

Preventing event loss #

Các event được trigger trước hoặc trong lifecycle hook onApplicationBootstrap — chẳng hạn như từ module constructor hoặc method onModuleInit — có thể bị bỏ lỡ vì EventSubscribersLoader có thể chưa hoàn thành việc thiết lập các listener.

Để tránh vấn đề này, bạn có thể sử dụng method waitUntilReady của EventEmitterReadinessWatcher, trả về một promise resolve khi tất cả các listener đã được đăng ký. Method này có thể được gọi trong lifecycle hook onApplicationBootstrap của một module để đảm bảo rằng tất cả các event được capture đúng cách.

await this.eventEmitterReadinessWatcher.waitUntilReady();
this.eventEmitter.emit(
  'order.created',
  new OrderCreatedEvent({ orderId: 1, payload: {} }),
);
📘 Note

Điều này chỉ cần thiết cho các event được emit trước khi lifecycle hook onApplicationBootstrap hoàn tất.
