Configuration

Ứng dụng thường chạy trong các môi trường khác nhau. Tùy thuộc vào môi trường, các cài đặt cấu hình khác nhau nên được sử dụng. Ví dụ, thông thường môi trường local dựa vào các port và database credential cụ thể, chỉ hợp lệ cho local DB instance. Môi trường production sẽ sử dụng một tập database credential riêng biệt. Vì các biến cấu hình thay đổi, best practice là lưu trữ biến cấu hình trong environment.

Các biến environment được định nghĩa bên ngoài có thể nhìn thấy bên trong Node.js thông qua global process.env. Chúng ta có thể cố gắng giải quyết vấn đề nhiều môi trường bằng cách thiết lập environment variable riêng biệt trong mỗi môi trường. Điều này có thể nhanh chóng trở nên khó quản lý, đặc biệt trong các môi trường dev và testing nơi các giá trị này cần được mock và/hoặc thay đổi dễ dàng.

Trong ứng dụng Node.js, thường sử dụng file .env, chứa các cặp key-value trong đó mỗi key đại diện cho một giá trị cụ thể, để đại diện cho mỗi môi trường. Chạy ứng dụng trong các môi trường khác nhau chỉ đơn giản là hoán đổi đúng file .env.

Cách tiếp cận tốt nhất để sử dụng kỹ thuật này trong Nest là tạo một ConfigModule expose một ConfigService để load file .env phù hợp. Mặc dù bạn có thể tự viết module như vậy, để thuận tiện Nest cung cấp package @nestjs/config sẵn. Chúng ta sẽ tìm hiểu package này trong chương hiện tại.

Installation #

Để bắt đầu sử dụng nó, trước tiên chúng ta cài đặt dependency cần thiết.

$ npm i --save @nestjs/config
💡 Hint

Package @nestjs/config sử dụng nội bộ dotenv.

📘 Notice

Package @nestjs/config yêu cầu TypeScript 4.1 trở lên.

Getting started #

Sau khi quá trình cài đặt hoàn tất, chúng ta có thể import ConfigModule. Thông thường, chúng ta import nó vào root AppModule và kiểm soát behavior của nó bằng static method .forRoot(). Trong bước này, các cặp key/value của environment variable được parse và resolve. Sau đó, chúng ta sẽ thấy một số option để truy cập ConfigService class của ConfigModule trong các feature module khác.

app.module.ts
TS
JS
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';

@Module({
  imports: [ConfigModule.forRoot()],
})
export class AppModule {}
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';

@Module({
  imports: [ConfigModule.forRoot()],
})
export class AppModule {}

Đoạn code trên sẽ load và parse file .env từ vị trí mặc định (thư mục gốc dự án), merge các cặp key/value từ file .env với environment variable được gán cho process.env, và lưu kết quả trong một cấu trúc private mà bạn có thể truy cập qua ConfigService. Method forRoot() đăng ký ConfigService provider, cung cấp method get() thuận tiện để đọc các biến cấu hình đã được parse/merge. Vì @nestjs/config dựa vào dotenv, nó sử dụng các quy tắc của package đó để resolve xung đột khi có tên biến trùng. Khi một key tồn tại cả trong runtime environment dưới dạng environment variable (ví dụ: qua OS shell export như export DATABASE_USER=test) và trong file .env, runtime environment variable được ưu tiên.

Một file .env mẫu trông giống như thế này:

DATABASE_HOST=localhost
DATABASE_PORT=5432

Custom env file path #

Mặc định, package tìm file .env trong thư mục gốc của ứng dụng. Để chỉ định một đường dẫn khác cho file .env, set property envFilePath của object options (tùy chọn) được truyền vào forRoot(), như sau:

ConfigModule.forRoot({
  envFilePath: '.development.env',
});

Bạn cũng có thể chỉ định nhiều đường dẫn cho file .env như thế này:

ConfigModule.forRoot({
  envFilePath: ['.env.development.local', '.env.development'],
});

Nếu một biến được tìm thấy trong nhiều file, file đầu tiên được ưu tiên.

Disable env variables loading #

Nếu bạn không muốn load file .env, mà thay vào đó muốn đơn giản truy cập environment variable từ runtime environment (như với OS shell export như export DATABASE_USER=test), set property ignoreEnvFile của object options thành true, như sau:

ConfigModule.forRoot({
  ignoreEnvFile: true,
});

Use module globally #

Khi bạn muốn sử dụng ConfigModule trong các module khác, bạn cần import nó (như tiêu chuẩn với bất kỳ module Nest nào). Ngoài ra, bạn có thể khai báo nó là global module bằng cách set property isGlobal thành true, như dưới đây. Trong trường hợp đó, bạn sẽ không cần import ConfigModule trong các module khác sau khi nó đã được load trong root module (ví dụ: AppModule).

ConfigModule.forRoot({
  isGlobal: true,
});

Custom configuration files #

Đối với các dự án phức tạp hơn, bạn có thể sử dụng custom configuration file để trả về các object cấu hình lồng nhau (nested). Điều này cho phép bạn nhóm các cài đặt cấu hình liên quan theo chức năng (ví dụ: các cài đặt liên quan đến database), và lưu trữ các cài đặt liên quan trong các file riêng biệt để quản lý chúng độc lập.

Một custom configuration file export một factory function trả về một configuration object. Configuration object có thể là bất kỳ object JavaScript lồng nhau nào. process.env object sẽ chứa các cặp key/value environment variable đã được resolve đầy đủ (với file .env và các biến được xác định bên ngoài đã được resolve và merge như mô tả ở trên). Vì bạn kiểm soát object cấu hình trả về, bạn có thể thêm bất kỳ logic cần thiết nào để ép kiểu giá trị sang type phù hợp, set giá trị mặc định, v.v. Ví dụ:

config/configuration.ts
TS
JS
export default () => ({
  port: parseInt(process.env.PORT, 10) || 3000,
  database: {
    host: process.env.DATABASE_HOST,
    port: parseInt(process.env.DATABASE_PORT, 10) || 5432,
  },
});
export default () => ({
  port: parseInt(process.env.PORT, 10) || 3000,
  database: {
    host: process.env.DATABASE_HOST,
    port: parseInt(process.env.DATABASE_PORT, 10) || 5432,
  },
});

Chúng ta load file này bằng property load của object options được truyền vào method ConfigModule.forRoot():

app.module.ts
TS
JS
import configuration from './config/configuration';

@Module({
  imports: [
    ConfigModule.forRoot({
      load: [configuration],
    }),
  ],
})
export class AppModule {}
import configuration from './config/configuration';

@Module({
  imports: [
    ConfigModule.forRoot({
      load: [configuration],
    }),
  ],
})
export class AppModule {}
📘 Notice

Giá trị được gán cho property load là một array, cho phép bạn load nhiều configuration file (ví dụ: load: [databaseConfig, authConfig]).

Using the ConfigService #

Để truy cập các giá trị cấu hình từ ConfigService, trước tiên chúng ta cần inject ConfigService. Như với bất kỳ provider nào, chúng ta cần import module chứa nó — ConfigModule — vào module sử dụng nó (trừ khi bạn set property isGlobal trong object options được truyền vào method ConfigModule.forRoot() thành true). Import nó vào feature module như dưới đây.

feature.module.ts
TS
JS
@Module({
  imports: [ConfigModule],
  // ...
})
@Module({
  imports: [ConfigModule],
  // ...
})

Sau đó inject nó bằng constructor injection tiêu chuẩn:

constructor(private configService: ConfigService) {}
💡 Hint

Import ConfigService từ package @nestjs/config.

Và sử dụng nó trong class:

// lấy environment variable
const dbHost = this.configService.get<string>('DATABASE_HOST');
// lấy custom configuration value
const dbHost = this.configService.get<string>('database.host');

Như ở trên, sử dụng method configService.get() để lấy một biến cấu hình đơn giản bằng cách truyền tên biến. Bạn có thể thực hiện TypeScript type hinting bằng cách truyền type, như ở trên (ví dụ: get<string>(...)). Method get() cũng có thể duyệt qua nested custom configuration object (được tạo qua Custom configuration file), như ở ví dụ thứ hai ở trên.

Bạn cũng có thể lấy toàn bộ nested custom configuration object bằng cách sử dụng dot notation làm interface token:

const dbConfig = this.configService.get<string>('database');
// bạn giờ có thể sử dụng `dbConfig.host` và `dbConfig.port`
const dbHost = dbConfig.host;

Method ConfigService cũng có method getOrThrow() tùy chọn, sẽ throw nếu giá trị cấu hình không được định nghĩa, kích hoạt resolve type dễ dàng hơn. Ví dụ:

// throw nếu "database.host" không được định nghĩa
const dbHost = this.configService.getOrThrow<string>('database.host');

Configuration namespaces #

Class ConfigModule cho phép bạn định nghĩa và load nhiều custom configuration file, như trong Custom configuration files ở trên. Bạn có thể quản lý hierarchies phức tạp của configuration object với nested configuration object như trong section đó. Ngoài ra, bạn có thể trả về "namespaced" configuration object với function registerAs() như dưới đây:

config/database.config.ts
TS
JS
export default registerAs('database', () => ({
  host: process.env.DATABASE_HOST,
  port: process.env.DATABASE_PORT || 5432,
}));
export default registerAs('database', () => ({
  host: process.env.DATABASE_HOST,
  port: process.env.DATABASE_PORT || 5432,
}));

Load namespaced configuration bằng property load của method forRoot(), giống hệt cách load custom configuration file:

import databaseConfig from './config/database.config';

@Module({
  imports: [
    ConfigModule.forRoot({
      load: [databaseConfig],
    }),
  ],
})
export class AppModule {}

Bây giờ, để lấy giá trị host từ namespace database, sử dụng dot notation. Sử dụng 'database' làm prefix cho tên property, tương ứng với tên namespace (truyền làm đối số đầu tiên cho function registerAs()):

const dbHost = this.configService.get<string>('database.host');

Một giải pháp thay thế hợp lý là inject namespace 'database' trực tiếp. Điều này cho phép chúng ta hưởng lợi từ strong typing:

constructor(
  @Inject(databaseConfig.KEY)
  private dbConfig: ConfigType<typeof databaseConfig>,
) {}
💡 Hint

Import ConfigType từ package @nestjs/config.

Cache environment variables #

Vì truy cập vào process.env có thể chậm, bạn có thể set property cache của object options được truyền vào ConfigModule.forRoot() để tăng hiệu năng của method ConfigService#get khi xử lý các biến được lưu trong process.env.

ConfigModule.forRoot({
  cache: true,
});

Partial registration #

Cho đến nay, chúng ta đã xử lý configuration file trong root module (ví dụ: AppModule), qua method forRoot(). Có lẽ bạn có cấu trúc dự án phức tạp hơn, với feature-specific configuration file nằm trong nhiều thư mục khác nhau. Thay vì load tất cả các file này trong root module, package @nestjs/config cung cấp tính năng gọi là partial registration, chỉ tham chiếu đến các configuration file liên kết với mỗi feature module. Sử dụng static method forFeature() trong một feature module để thực hiện partial registration này, như sau:

import databaseConfig from './config/database.config';

@Module({
  imports: [ConfigModule.forFeature(databaseConfig)],
})
export class DatabaseModule {}
⚠️ Warning

Trong một số trường hợp, bạn có thể cần truy cập property được load qua partial registration bằng cách sử dụng hook onModuleInit(), thay vì trong constructor. Điều này là do method forFeature() được thực thi trong quá trình khởi tạo module, và thứ tự khởi tạo module là không xác định. Nếu bạn truy cập các giá trị được load bởi module khác, trong constructor, configuration module đó có thể chưa được khởi tạo xong. Method onModuleInit() chỉ chạy sau khi tất cả module mà nó phụ thuộc đã khởi tạo xong, nên đây là kỹ thuật an toàn.

Schema validation #

Thông thường, việc throw exception trong lúc khởi động ứng dụng nếu các environment variable cần thiết không được cung cấp hoặc không đáp ứng một số quy tắc validation là standard practice. Package @nestjs/config cho phép hai cách khác nhau để làm điều này:

  • Joi built-in validator. Với Joi, bạn định nghĩa một object schema và validate các JavaScript object dựa trên nó.
  • Custom validate() function nhận environment variable làm đầu vào.

Để sử dụng Joi, chúng ta cần cài đặt package Joi:

$ npm install --save joi

Bây giờ chúng ta có thể định nghĩa validation schema Joi và truyền nó qua property validationSchema của object options của method forRoot(), như dưới đây:

app.module.ts
TS
JS
import * as Joi from 'joi';

@Module({
  imports: [
    ConfigModule.forRoot({
      validationSchema: Joi.object({
        NODE_ENV: Joi.string()
          .valid('development', 'production', 'test', 'provision')
          .default('development'),
        PORT: Joi.number().port().default(3000),
      }),
    }),
  ],
})
export class AppModule {}
const Joi = require('joi');

@Module({
  imports: [
    ConfigModule.forRoot({
      validationSchema: Joi.object({
        NODE_ENV: Joi.string()
          .valid('development', 'production', 'test', 'provision')
          .default('development'),
        PORT: Joi.number().port().default(3000),
      }),
    }),
  ],
})
export class AppModule {}

Mặc định, tất cả environment variable được coi là tùy chọn. Ở đây, chúng ta set giá trị mặc định cho NODE_ENV và PORT, đó thường là đủ khi bạn không cung cấp chúng trong file .env. Ngoài ra, bạn có thể sử dụng validation method required() để yêu cầu bắt buộc một giá trị phải được định nghĩa trong environment (file .env hoặc process environment).

💡 Hint

Mặc định, các unknown environment variable (biến không có mặt trong schema) được cho phép và không kích hoạt validation exception. Mặc định, tất cả validation error đều được báo cáo. Bạn có thể thay đổi các behavior này bằng cách truyền object validationOptions qua key validationOptions của object options forRoot(). Object validation options tuân theo Joi validation options tiêu chuẩn.

Custom validate function #

Ngoài ra, bạn có thể chỉ định một synchronous validate function nhận một object chứa các environment variable (từ env file và process) và trả về một object chứa các environment variable đã validate để bạn có thể convert/mutate chúng nếu cần. Nếu function throw error, nó sẽ ngăn ứng dụng bootstrap.

Trong ví dụ này, chúng ta sẽ sử dụng class-transformer và class-validator package:

env.validation.ts
import { plainToInstance } from 'class-transformer';
import { IsEnum, IsNumber, validateSync } from 'class-validator';

enum Environment {
  Development = 'development',
  Production = 'production',
  Test = 'test',
  Provision = 'provision',
}

class EnvironmentVariables {
  @IsEnum(Environment)
  NODE_ENV: Environment;

  @IsNumber()
  PORT: number;
}

export function validate(config: Record<string, unknown>) {
  const validatedConfig = plainToInstance(
    EnvironmentVariables,
    config,
    { enableImplicitConversion: true },
  );
  const errors = validateSync(validatedConfig, { skipMissingProperties: false });

  if (errors.length > 0) {
    throw new Error(errors.toString());
  }
  return validatedConfig;
}

Với điều này, bạn sử dụng function validate như một configuration option của ConfigModule, như sau:

app.module.ts
TS
JS
import { validate } from './env.validation';

@Module({
  imports: [
    ConfigModule.forRoot({
      validate,
    }),
  ],
})
export class AppModule {}
import { validate } from './env.validation';

@Module({
  imports: [
    ConfigModule.forRoot({
      validate,
    }),
  ],
})
export class AppModule {}

Custom getter functions #

ConfigService định nghĩa một method get() generic để lấy giá trị cấu hình theo key. Chúng ta cũng có thể thêm getter function để cho phép cú pháp tự nhiên hơn một chút:

config/http.config.ts
export default registerAs('http', () => ({
  host: process.env.HTTP_HOST,
  port: process.env.HTTP_PORT,
}));
api-config.service.ts
@Injectable()
export class ApiConfigService {
  constructor(private configService: ConfigService) {}

  get isAuthEnabled(): boolean {
    return this.configService.get<string>('AUTH_ENABLED') === 'true';
  }
}

Environment variables loaded hook #

Nếu cấu hình module phụ thuộc vào các environment variable, và các biến đó được load từ file .env, bạn có thể sử dụng hook ConfigModule.envVariablesLoaded để đảm bảo file đã được load trước khi tương tác với object process.env, xem ví dụ Conditional module configuration.

Conditional module configuration #

Có thể có lúc bạn muốn load module có điều kiện và chỉ định điều kiện trong env variable. May mắn thay, @nestjs/config cung cấp ConditionalModule cho phép bạn làm điều đó.

@Module({
  imports: [
    ConfigModule.forRoot(),
    ConditionalModule.registerWhen(TerminusModule, 'USE_TERMINUS'),
  ],
})
export class AppModule {}

Module trên chỉ load TerminusModule khi file .env cung cấp giá trị non-false cho biến USE_TERMINUS. Bạn cũng có thể tự truyền một điều kiện tùy chỉnh, một function nhận tham chiếu process.env và nên trả về boolean cho ConditionalModule xử lý:

@Module({
  imports: [
    ConfigModule.forRoot(),
    ConditionalModule.registerWhen(
      TerminusModule,
      (env: NodeJS.ProcessEnv) => !!env['USE_TERMINUS'],
    ),
  ],
})
export class AppModule {}
⚠️ Warning

Hãy đảm bảo khi sử dụng ConditionalModule, bạn cũng có ConfigModule được load trong ứng dụng, để hook ConfigModule.envVariablesLoaded có thể được tham chiếu và sử dụng đúng cách. Nếu hook không được kích hoạt trong 5 giây, hoặc trong khoảng thời gian timeout bằng millisecond mà người dùng set ở tham số thứ ba của registerWhen, thì ConditionalModule sẽ throw error và Nest sẽ hủy quá trình khởi động ứng dụng.

Expandable variables #

Package @nestjs/config hỗ trợ variable expansion. Với kỹ thuật này, bạn có thể tạo các biến lồng nhau, trong đó một biến được tham chiếu trong định nghĩa của biến khác. Ví dụ:

.env
APP_URL=mywebsite.com
SUPPORT_EMAIL=support@${APP_URL}

Với construction này, biến SUPPORT_EMAIL resolve thành 'support@mywebsite.com'. Lưu ý sử dụng cú pháp ${...} để kích hoạt resolve giá trị APP_URL bên trong định nghĩa SUPPORT_EMAIL.

💡 Hint

Đối với tính năng này, package @nestjs/config sử dụng nội bộ dotenv-expand.

Bật tính năng variable expansion bằng cách sử dụng property expandVariables trong object options được truyền vào method forRoot() của ConfigModule, như dưới đây:

app.module.ts
TS
JS
@Module({
  imports: [
    ConfigModule.forRoot({
      expandVariables: true,
    }),
  ],
})
export class AppModule {}
@Module({
  imports: [
    ConfigModule.forRoot({
      expandVariables: true,
    }),
  ],
})
export class AppModule {}

Using in the main.ts #

Mặc dù config của chúng ta được lưu trong một service, nó vẫn có thể được sử dụng trong file main.ts. Bằng cách đó, bạn có thể sử dụng nó để lưu các biến như port ứng dụng hoặc CORS host.

Để truy cập nó, bạn sử dụng method app.get(), theo sau là service reference:

const configService = app.get(ConfigService);

Sau đó bạn có thể sử dụng nó như bình thường, bằng cách gọi method get với configuration key:

const port = configService.get('PORT');
