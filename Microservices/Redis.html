Redis

Redis transporter triển khai mô hình messaging publish/subscribe, tận dụng tính năng pub/sub của Redis. Các message đã publish được phân loại vào các channel, mà không cần biết subscriber nào (nếu có) sẽ nhận message. Mỗi microservice có thể subscribe vào bất kỳ số lượng channel nào. Ngoài ra, nhiều hơn một channel có thể được subscribe cùng lúc. Các message được trao đổi là fire-and-forget, nghĩa là nếu một message được publish và không có subscriber nào quan tâm đến nó, message sẽ bị loại bỏ và không thể khôi phục. Do đó, bạn không có đảm bảo rằng message hoặc event sẽ được xử lý bởi ít nhất một service. Một message có thể được subscribe (và nhận) bởi nhiều subscriber.

Installation #

Để bắt đầu xây dựng microservices dựa trên Redis, trước tiên hãy cài đặt package cần thiết:

$ npm i --save ioredis

Overview #

Để sử dụng Redis transporter, truyền object options sau vào phương thức createMicroservice():

main
import { NestFactory } from '@nestjs/core';
import { Transport, MicroserviceOptions } from '@nestjs/microservices';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
    transport: Transport.REDIS,
    options: {
      host: 'localhost',
      port: 6379,
    },
  });
  await app.listen();
}
bootstrap();

Gợi ý

Enum Transport được import từ package @nestjs/microservices.

Options #

Object options là riêng cho transporter đã chọn. Redis transporter cung cấp các thuộc tính được mô tả dưới đây.

hostURL kết nối (mặc định: 'localhost')
portCổng kết nối (mặc định: 6379)
retryAttemptsSố lần thử lại message (mặc định: 0)
retryDelayĐộ trễ giữa các lần thử lại message (ms) (mặc định: 0)
wildcardsBật subscription dựa trên pattern của Redis, cho phép subscribe bằng wildcard (mặc định: false)

Tất cả các thuộc tính được hỗ trợ bởi driver ioredis chính thức cũng được transporter này hỗ trợ.

Client #

Giống như các transporter của microservice khác, bạn có một số tùy chọn để tạo một Redis ClientProxy instance.

Một phương thức để tạo instance là sử dụng ClientsModule. Để tạo một client instance với ClientsModule, import nó và sử dụng phương thức register() để truyền một object options có cùng thuộc tính như đã trình bày ở trên trong phương thức createMicroservice(), cùng với thuộc tính name được sử dụng làm injection token. Đọc thêm về ClientsModule tại đây.

@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'MATH_SERVICE',
        transport: Transport.REDIS,
        options: {
          host: 'localhost',
          port: 6379,
        }
      },
    ]),
  ],
  ...
})
@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'MATH_SERVICE',
        transport: Transport.REDIS,
        options: {
          host: 'localhost',
          port: 6379,
        }
      },
    ]),
  ],
  ...
})

Các factory khác cũng có thể được sử dụng để tạo client (ClientProxyFactory hoặc @Client()). Bạn có thể đọc thêm về chúng tại đây.

Context #

Trong các pattern phức tạp hơn, bạn có thể muốn truy cập thêm thông tin về request đến. Khi sử dụng Redis transporter, bạn có thể truy cập object RedisContext.

@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RedisContext) {
  console.log(`Channel: ${context.getChannel()}`);
}
@MessagePattern('notifications')
getNotifications(@Payload() data, @Ctx() context) {
  console.log(`Channel: ${context.getChannel()}`);
}
Gợi ý

@Payload(), @Ctx() và RedisContext được import từ package @nestjs/microservices.

Wildcards #

Một subscription có thể sử dụng glob-style pattern (wildcard). Đây là tính năng psubscribe / pmessage gốc của Redis. Ví dụ, để subscribe vào tất cả các channel bắt đầu bằng order., bạn sẽ sử dụng pattern sau:

@MessagePattern('order.*')
getOrders(@Payload() data: number[], @Ctx() context: RedisContext) {
  console.log(`Channel: ${context.getChannel()}`);
}
@MessagePattern('order.*')
getOrders(@Payload() data, @Ctx() context) {
  console.log(`Channel: ${context.getChannel()}`);
}

Để bật tính năng wildcard subscription, bạn cần thiết lập option wildcards thành true trong cấu hình transporter:

main
const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  transport: Transport.REDIS,
  options: {
    host: 'localhost',
    port: 6379,
    wildcards: true,
  },
});

Instance status updates #

Để nhận thông tin cập nhật về trạng thái kết nối và hoạt động của driver, bạn có thể subscribe vào các stream status cụ thể, như sau:

main
const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  transport: Transport.REDIS,
  options: {
    host: 'localhost',
    port: 6379,
  },
});

const redisClient = app.status;
redisClient.subscribe((status) => {
  console.log(status);
});

Thuộc tính status là một Observable phát ra các sự kiện trạng thái của driver. Các trạng thái có thể bao gồm: connected, disconnected, reconnecting, ready. Điều này cho phép bạn theo dõi tình trạng kết nối của driver.

Tương tự, bạn cũng có thể lắng nghe trạng thái của client proxy:

const client = app.get<ClientProxy>('MATH_SERVICE');
client.status.subscribe((status) => {
  console.log(status);
});

Listening to Redis events #

Trong một số trường hợp, bạn có thể muốn lắng nghe các sự kiện nội bộ được phát ra bởi Redis client. Ví dụ, bạn có thể muốn lắng nghe sự kiện error để ghi log các lỗi xảy ra. Để làm điều này, sử dụng phương thức unwrap để lấy underlying driver instance.

const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  transport: Transport.REDIS,
  options: {
    host: 'localhost',
    port: 6379,
  },
});

const [subClient, pubClient] = app.unwrap<[Redis, Redis]>();
subClient.on('error', (err) => {
  console.error(err);
});

Tương tự, bạn cũng có thể lấy underlying driver instance từ client proxy:

const client = app.get<ClientProxy>('MATH_SERVICE');
const [subClient, pubClient] = client.unwrap<[Redis, Redis]>();
subClient.on('error', (err) => {
  console.error(err);
});

Underlying driver access #

Để truy cập trực tiếp vào driver instance (ioredis), sử dụng phương thức unwrap(). Phương thức này trả về một tuple gồm hai ioredis instance: cái đầu tiên cho subscriber và cái thứ hai cho publisher.

const [subClient, pubClient] = app.unwrap<[Redis, Redis]>();

Phương thức unwrap() cho phép bạn truy cập trực tiếp vào các ioredis client bên dưới, cho phép bạn thực hiện các thao tác nâng cao mà NestJS abstraction layer không trực tiếp hỗ trợ, chẳng hạn như lắng nghe các sự kiện Redis nội bộ hoặc sử dụng các lệnh Redis tùy chỉnh.

Lưu ý

Khi sử dụng unwrap(), bạn đang tương tác trực tiếp với driver, do đó NestJS sẽ không quản lý kết nối hay theo dõi trạng thái của nó nữa.
