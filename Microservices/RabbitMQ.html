RabbitMQ

RabbitMQ là một open-source message-broker phổ biến và nhẹ, hỗ trợ nhiều messaging protocol. Nó có thể được triển khai trong các cấu hình distributed và federated để đáp ứng yêu cầu high-scale, high-availability. Ngoài ra, nó là message broker được triển khai rộng rãi nhất, được sử dụng trên toàn thế giới tại các doanh nghiệp nhỏ và lớn.

Installation #

Để bắt đầu xây dựng microservices dựa trên RabbitMQ, trước tiên hãy cài đặt các package cần thiết:

$ npm i --save amqplib amqp-connection-manager

Overview #

Để sử dụng RabbitMQ transporter, hãy truyền object options sau vào phương thức createMicroservice():

main
import { NestFactory } from '@nestjs/core';
import { Transport, MicroserviceOptions } from '@nestjs/microservices';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
    transport: Transport.RMQ,
    options: {
      urls: ['amqp://localhost:5672'],
      queue: 'cats_queue',
      queueOptions: {
        durable: false
      },
    },
  });
  await app.listen();
}
bootstrap();

Options #

Thuộc tính options là riêng cho transporter đã chọn. RabbitMQ transporter cung cấp các thuộc tính được mô tả bên dưới.

urlsMảng các connection URL
queueTên queue mà server sẽ lắng nghe
prefetchCountSố lượng message tối đa được gửi đồng thời trên channel. Mặc định là 0
isGlobalPrefetchCountBật global prefetch count cho channel. Mặc định là false
noAckNếu false, chế độ acknowledgement thủ công được bật. Mặc định là false
consumerTagTên tag cho consumer (xem tại đây)
queueOptionsCác tùy chọn bổ sung cho queue (xem tại đây)
socketOptionsCác tùy chọn bổ sung cho socket (xem tại đây)
headersHeaders được gửi cùng với mỗi message
replyQueueTên reply queue để ghi đè. Nếu không được cung cấp, reply queue sẽ được tạo tự động theo chế độ "assert" với tên được tạo tự động
persistentNếu true, message sẽ được lưu trữ persistent. Mặc định là false
noAssertNếu true, server sẽ không assert queue khi kết nối. Mặc định là false
wildcardsBật hỗ trợ pattern-matching wildcard. Mặc định là false
exchangeTên exchange
exchangeTypeLoại exchange. Mặc định là 'direct'
routingKeyRouting key cho exchange
maxConnectionAttemptsSố lần thử kết nối tối đa. Mặc định là -1 (vô hạn)

Client #

Giống như các Nest microservice transport layer khác, bạn có một số tùy chọn để tạo RabbitMQ ClientProxy instance.

Một phương pháp để tạo instance là sử dụng ClientsModule. Để tạo client instance với ClientsModule, import nó và sử dụng phương thức register() để truyền một object options có cùng thuộc tính như đã trình bày ở trên trong phương thức createMicroservice(), cùng với thuộc tính name dùng làm injection token. Đọc thêm về ClientsModule tại đây.

app.module
@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'MATH_SERVICE',
        transport: Transport.RMQ,
        options: {
          urls: ['amqp://localhost:5672'],
          queue: 'cats_queue',
          queueOptions: {
            durable: false
          },
        },
      },
    ]),
  ],
})
export class AppModule {}

Context #

Trong các pattern phức tạp hơn, bạn có thể muốn truy cập thêm thông tin về request đến. Khi sử dụng RabbitMQ transporter, bạn có thể truy cập object RmqContext.

app.controller
@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RmqContext) {
  console.log(context.getPattern());
}
@Bind(Payload(), Ctx())
@MessagePattern('notifications')
getNotifications(data, context) {
  console.log(context.getPattern());
}
Gợi ý

@Payload(), @Ctx() và RmqContext được import từ package @nestjs/microservices.

Để truy cập RabbitMQ message gốc (với properties, fields, và content), sử dụng phương thức getMessage() của object RmqContext, như sau:

app.controller
@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RmqContext) {
  console.log(context.getMessage());
}
@Bind(Payload(), Ctx())
@MessagePattern('notifications')
getNotifications(data, context) {
  console.log(context.getMessage());
}

Để lấy tham chiếu đến RabbitMQ channel, sử dụng phương thức getChannelRef() của object RmqContext:

app.controller
@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RmqContext) {
  console.log(context.getChannelRef());
}
@Bind(Payload(), Ctx())
@MessagePattern('notifications')
getNotifications(data, context) {
  console.log(context.getChannelRef());
}

Message acknowledgement #

Để đảm bảo message không bao giờ bị mất, RabbitMQ hỗ trợ message acknowledgement. Acknowledgement được consumer gửi lại cho RabbitMQ để thông báo rằng một message cụ thể đã được nhận, xử lý và RabbitMQ có thể xóa nó. Nếu consumer chết (channel bị đóng, connection bị đóng, hoặc TCP connection bị mất) mà không gửi ack, RabbitMQ sẽ hiểu rằng message chưa được xử lý hoàn toàn và sẽ re-queue nó.

Để bật chế độ acknowledgement thủ công, hãy đặt noAck thành false:

options: {
  urls: ['amqp://localhost:5672'],
  queue: 'cats_queue',
  noAck: false,
  queueOptions: {
    durable: false
  },
}

Khi chế độ acknowledgement thủ công được bật, chúng ta phải gửi một ACK thích hợp từ worker để thông báo rằng đã hoàn tất xử lý một task.

app.controller
@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: RmqContext) {
  const channel = context.getChannelRef();
  const originalMsg = context.getMessage();

  channel.ack(originalMsg);
}
@Bind(Payload(), Ctx())
@MessagePattern('notifications')
getNotifications(data, context) {
  const channel = context.getChannelRef();
  const originalMsg = context.getMessage();

  channel.ack(originalMsg);
}

Record builders #

Để cấu hình các message options, bạn có thể sử dụng class RmqRecordBuilder. Ví dụ, để thiết lập headers và thuộc tính priority, sử dụng phương thức setOptions, như sau:

const message = ':cat:';
const record = new RmqRecordBuilder(message)
  .setOptions({
    headers: {
      ['x-version']: '1.0.0',
    },
    priority: 3,
  })
  .build();

this.client.send('replace-emoji', record).subscribe(...);

RmqRecordBuilder class được export từ package @nestjs/microservices.

Và bạn cũng có thể đọc các giá trị này ở phía server bằng cách truy cập RmqContext, như sau:

app.controller
@MessagePattern('replace-emoji')
replaceEmoji(@Payload() data: string, @Ctx() context: RmqContext): string {
  const { properties: { headers } } = context.getMessage();
  return {
    data: data.replace(':cat:', '🐱'),
    headers,
  };
}
@Bind(Payload(), Ctx())
@MessagePattern('replace-emoji')

  return {
    data: data.replace(':cat:', '🐱'),
    headers,
  };
}

Instance status updates #

Để nhận cập nhật về trạng thái kết nối và trạng thái của server instance, bạn có thể subscribe vào các event tương ứng. Có hai event liên quan ở đây: connected và disconnected. Các event này được phát ra khi server instance kết nối hoặc ngắt kết nối khỏi RabbitMQ broker.

const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  transport: Transport.RMQ,
  options: {
    urls: ['amqp://localhost:5672'],
    queue: 'cats_queue',
  },
});

const rmqServer = app.status;
rmqServer.subscribe((status) => {
  console.log(status);
});

Tương tự, đối với client proxy, bạn có thể subscribe vào các event trạng thái để nhận thông tin về trạng thái kết nối của client:

const client = app.get<ClientProxy>('MATH_SERVICE');

client.status.subscribe((status) => {
  console.log(status);
});

Listening to RabbitMQ events #

Trong một số trường hợp, bạn có thể muốn lắng nghe các event nội bộ được phát ra bởi amqp-connection-manager (ví dụ: khi connection được thiết lập hoặc xảy ra lỗi). Để làm điều đó, hãy đăng ký một event listener, như sau:

const rmqServer = app.unwrap<ServerRMQ>();

rmqServer.on('connect', () => {
  console.log('Connected to RabbitMQ');
});

rmqServer.on('disconnect', (err) => {
  console.log('Disconnected from RabbitMQ', err);
});

rmqServer.on('connectFailed', (err) => {
  console.log('Connection to RabbitMQ failed', err);
});

Tương tự, bạn có thể lắng nghe các event từ phía client proxy:

const client = app.get<ClientProxy>('MATH_SERVICE');
const rmqClient = client.unwrap<ClientRMQ>();

rmqClient.on('connect', () => {
  console.log('Connected to RabbitMQ');
});

rmqClient.on('disconnect', (err) => {
  console.log('Disconnected from RabbitMQ', err);
});

Underlying driver access #

Để truy cập driver instance bên dưới (amqp-connection-manager), sử dụng phương thức unwrap(). Đọc thêm về pattern này tại đây.

const rmqServer = app.unwrap<ServerRMQ>();
const server = rmqServer.server; // amqp-connection-manager instance (AmqpConnectionManager)
const channel = rmqServer.channel; // channel wrapper (ChannelWrapper)

Đối với client proxy:

const client = app.get<ClientProxy>('MATH_SERVICE');
const rmqClient = client.unwrap<ClientRMQ>();
const client = rmqClient.client; // amqp-connection-manager instance (AmqpConnectionManager)
const channel = rmqClient.channel; // channel wrapper (ChannelWrapper)

Wildcards #

Một pattern có thể là một explicit string nhưng cũng có thể là một wildcard. RabbitMQ hỗ trợ hai kiểu wildcard: * (star) khớp với đúng một từ, và # (hash) khớp với không hoặc nhiều từ. Để sử dụng tính năng này, bạn cần cấu hình topic exchange.

Để bật wildcards, bạn cần thiết lập tùy chọn wildcards thành true trong cấu hình transporter, đồng thời cung cấp exchange và exchangeType:

main
const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  transport: Transport.RMQ,
  options: {
    urls: ['amqp://localhost:5672'],
    queue: 'cats_queue',
    wildcards: true,
    exchange: 'my_exchange',
    exchangeType: 'topic',
    queueOptions: {
      durable: false
    },
  },
});
const app = await NestFactory.createMicroservice(AppModule, {
  transport: Transport.RMQ,
  options: {
    urls: ['amqp://localhost:5672'],
    queue: 'cats_queue',
    wildcards: true,
    exchange: 'my_exchange',
    exchangeType: 'topic',
    queueOptions: {
      durable: false
    },
  },
});

Khi wildcards được bật, bạn có thể sử dụng wildcard patterns trong message pattern của handler:

app.controller
@MessagePattern('animals.*.info')
getAnimalInfo(@Payload() data: any, @Ctx() context: RmqContext) {
  console.log(`Pattern: ${context.getPattern()}`);
  return data;
}

@MessagePattern('animals.#')
getAllAnimals(@Payload() data: any, @Ctx() context: RmqContext) {
  console.log(`Pattern: ${context.getPattern()}`);
  return data;
}
@Bind(Payload(), Ctx())
@MessagePattern('animals.*.info')
getAnimalInfo(data, context) {
  console.log(`Pattern: ${context.getPattern()}`);
  return data;
}

@Bind(Payload(), Ctx())
@MessagePattern('animals.#')
getAllAnimals(data, context) {
  console.log(`Pattern: ${context.getPattern()}`);
  return data;
}

Trong ví dụ trên, pattern animals.*.info sẽ khớp với routing key như animals.cat.info hoặc animals.dog.info. Pattern animals.# sẽ khớp với bất kỳ routing key nào bắt đầu bằng animals., bao gồm animals.cat, animals.cat.info, và animals.cat.info.details.
