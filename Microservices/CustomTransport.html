Custom Transporters

Nest cung cấp sẵn một số transport layer implementations, cũng như khả năng tạo custom transport strategies của riêng bạn, cho phép bạn tích hợp với bất kỳ messaging system hoặc platform nào.

Các transporter được xây dựng sẵn bao gồm TCP, Redis, NATS, MQTT, Kafka, và gRPC. Để tạo một custom transport strategy, bạn cần implement các protocol giao tiếp cần thiết được Nest sử dụng.

Một Nest microservice là một ứng dụng sử dụng một transport layer khác so với HTTP. Nest trừu tượng hoá các chi tiết implementation của mỗi transporter đằng sau một interface được gọi là CustomTransportStrategy, đồng thời cung cấp một client proxy để gửi và nhận messages/events tới microservice từ xa.

Gợi ý

Việc tạo custom transporter từ đầu không bắt buộc. Nếu bạn chỉ cần tích hợp với một messaging system hiện có (như RabbitMQ hoặc Amazon SQS), hãy cân nhắc sử dụng các Nest community packages có sẵn trước khi tự tạo custom strategy.

Creating a strategy #

Để định nghĩa một custom transport strategy, trước tiên bạn cần tạo một class extend từ Server (được import từ @nestjs/microservices) và implement interface CustomTransportStrategy.

Interface CustomTransportStrategy bắt buộc bạn phải implement các method sau:

  • listen(callback: (...optionalParams: unknown[]) => any) - Khởi động server transport và bắt đầu lắng nghe các message đến.
  • close() - Huỷ server transport.

Ngoài ra, class Server cũng cung cấp một số method hữu ích, bao gồm khả năng đăng ký message handlers với Nest runtime.

google-cloud-pub-sub-server.ts
import { CustomTransportStrategy, Server } from '@nestjs/microservices';

export class GoogleCloudPubSubServer
  extends Server
  implements CustomTransportStrategy
{
  /**
   * This method is triggered when you run "app.listen()".
   */
  listen(callback: () => void) {
    callback();
  }

  /**
   * This method is triggered on application shutdown.
   */
  close() {}
}
import { Server } from '@nestjs/microservices';

export class GoogleCloudPubSubServer extends Server {
  /**
   * This method is triggered when you run "app.listen()".
   */
  listen(callback) {
    callback();
  }

  /**
   * This method is triggered on application shutdown.
   */
  close() {}
}

Với class này đã được định nghĩa, chúng ta có thể sử dụng custom strategy của mình trong method createMicroservice(), thay vì truyền một transport enum thông thường.

main.ts
const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  strategy: new GoogleCloudPubSubServer(),
});
const app = await NestFactory.createMicroservice(AppModule, {
  strategy: new GoogleCloudPubSubServer(),
});

Thay vì truyền thuộc tính transport cùng với các options, chúng ta truyền một thuộc tính duy nhất: strategy, có giá trị là một instance của custom transport strategy class của chúng ta.

Quay lại class GoogleCloudPubSubServer, trong một ứng dụng thực tế, chúng ta sẽ thiết lập kết nối tới message broker/external service của mình và đăng ký subscribers/listeners trong method listen() (sau đó gọi callback), và ngắt kết nối trong method close().

Class Server của @nestjs/microservices có vai trò quan trọng vì nó cung cấp một số method hữu ích, ví dụ, khả năng truy cập các message handler đã được đăng ký bởi Nest runtime. Nó cũng cung cấp một số method tiện ích cơ bản để implement các custom transport strategies. Hãy xem xét chi tiết hơn.

Thuộc tính messageHandlers là một Map chứa tất cả các message handlers đã được đăng ký bởi Nest runtime (thông qua các decorator như @MessagePattern() và @EventPattern()), trong đó key là pattern tương ứng và value là handler function.

google-cloud-pub-sub-server.ts
import { CustomTransportStrategy, Server } from '@nestjs/microservices';

export class GoogleCloudPubSubServer
  extends Server
  implements CustomTransportStrategy
{
  listen(callback: () => void) {
    // Create the subscription to the Google Cloud Pub/Sub topic
    // In a real application, here you would set up the connection
    // to the message broker and register subscribers.

    // For each received message, use the "handleEvent"
    // or "handleMessage" method (provided by the "Server" class)
    // to delegate processing to the appropriate handler
    // registered in your Nest application.
    this.messageHandlers.forEach((handler, pattern) => {
      // The "handler" here is a reference to the handler registered
      // through the "@MessagePattern" and "@EventPattern" decorators
      // in your Nest application.
    });

    callback();
  }

  close() {}
}
import { Server } from '@nestjs/microservices';

export class GoogleCloudPubSubServer extends Server {
  listen(callback) {
    this.messageHandlers.forEach((handler, pattern) => {
      // The "handler" here is a reference to the handler registered
      // through the "@MessagePattern" and "@EventPattern" decorators
      // in your Nest application.
    });

    callback();
  }

  close() {}
}

Mỗi handler được đăng ký qua @MessagePattern() và @EventPattern() sẽ có một entry tương ứng trong messageHandlers Map, với pattern được sử dụng làm key. Method on() được kế thừa từ class Server cho phép bạn đăng ký thêm các handlers cho các pattern cụ thể.

Để phân biệt giữa message handler (request-response) và event handler (event-based), bạn có thể sử dụng thuộc tính isEventHandler của handler. Nếu handler.isEventHandler là true, nó là event handler (được đăng ký với @EventPattern()). Ngược lại, nó là message handler (được đăng ký với @MessagePattern()).

Method unwrap() được sử dụng để giải nén (unwrap) các incoming payloads. Các transporter có thể gửi message dưới nhiều format khác nhau, và method này cho phép bạn chuyển đổi payload thành định dạng mà Nest có thể xử lý.

Client proxy #

Để gửi messages (hoặc events) tới custom transporter của bạn từ một ứng dụng Nest khác (hoặc từ cùng ứng dụng), bạn cần tạo một custom ClientProxy class.

Để tạo một custom client, bạn extend class ClientProxy và implement các method connect(), close(), publish() (cho messages) và dispatchEvent() (cho events).

google-cloud-pub-sub-client.ts
import { ClientProxy, ReadPacket, WritePacket } from '@nestjs/microservices';

export class GoogleCloudPubSubClient extends ClientProxy {
  async connect(): Promise<any> {
    // Create the connection to the Google Cloud Pub/Sub
  }

  async close() {
    // Close the connection to the Google Cloud Pub/Sub
  }

  async dispatchEvent(packet: ReadPacket<any>): Promise<any> {
    // This method is triggered when you use "client.emit()".
    // It should publish an event to the Google Cloud Pub/Sub topic.
    return;
  }

  publish(
    packet: ReadPacket<any>,
    callback: (packet: WritePacket<any>) => void,
  ): () => void {
    // This method is triggered when you use "client.send()".
    // It should publish a message to the Google Cloud Pub/Sub topic
    // and subscribe to the response topic to receive the response.
    // The "callback" function should be called with the response packet.
    return () => {
      // This function is called when the subscriber unsubscribes.
      // Perform any cleanup here.
    };
  }
}
import { ClientProxy } from '@nestjs/microservices';

export class GoogleCloudPubSubClient extends ClientProxy {
  async connect() {
    // Create the connection to the Google Cloud Pub/Sub
  }

  async close() {
    // Close the connection to the Google Cloud Pub/Sub
  }

  async dispatchEvent(packet) {
    // This method is triggered when you use "client.emit()".
    return;
  }

  publish(packet, callback) {
    // This method is triggered when you use "client.send()".
    return () => {
      // Cleanup on unsubscribe.
    };
  }
}

Method connect() được gọi khi khởi tạo kết nối tới transport layer. Nó trả về một Promise resolve khi kết nối thành công.

Method close() được gọi khi ngắt kết nối từ transport layer.

Method publish() được kích hoạt khi bạn sử dụng client.send(). Nó gửi một message và nhận response từ microservice. Method này trả về một function cleanup được gọi khi subscriber huỷ đăng ký.

Method dispatchEvent() được kích hoạt khi bạn sử dụng client.emit(). Nó gửi một event tới microservice mà không chờ đợi response.

Method unwrap() được sử dụng để giải nén (unwrap) các outgoing payloads, tương tự như phía server. Nó cho phép bạn chuyển đổi payload trước khi gửi đi.

Khi custom client proxy đã được tạo, bạn có thể sử dụng nó trực tiếp với decorator @Client() hoặc đăng ký nó như một custom provider.

app.controller.ts
@Controller()
export class AppController {
  constructor(
    @Inject('GOOGLE_CLOUD_PUB_SUB')
    private readonly client: GoogleCloudPubSubClient,
  ) {}

  @Get()
  sendMessage() {
    // Use "send" for request-response messaging
    return this.client.send('pattern', { data: 'value' });
  }

  @Get('emit')
  emitEvent() {
    // Use "emit" for event-based messaging
    this.client.emit('event', { data: 'value' });
  }
}
@Controller()
export class AppController {
  client;

  constructor(client) {
    this.client = client;
  }

  @Get()
  sendMessage() {
    return this.client.send('pattern', { data: 'value' });
  }

  @Get('emit')
  emitEvent() {
    this.client.emit('event', { data: 'value' });
  }
}

Method send() trả về một Observable mà bạn có thể subscribe để nhận response từ microservice. Method emit() gửi một event theo kiểu fire-and-forget và không chờ đợi response.

Message serialization #

Nếu bạn cần thêm logic tuỳ chỉnh xung quanh việc serialization của responses phía server, bạn có thể sử dụng các method serializeResponse() và serializeError() được cung cấp bởi class Server.

Method serializeResponse() được gọi để serialize response data trước khi gửi lại cho client. Method serializeError() được gọi để serialize một error response.

google-cloud-pub-sub-server.ts
import {
  CustomTransportStrategy,
  IncomingResponse,
  Server,
} from '@nestjs/microservices';

export class GoogleCloudPubSubServer
  extends Server
  implements CustomTransportStrategy
{
  listen(callback: () => void) {
    callback();
  }

  close() {}

  sendMessage(response: IncomingResponse) {
    // Here you can see how to use the "serializeResponse"
    // and "serializeError" methods to serialize the response.
    const serializedResponse = response.err
      ? this.serializeError(response.err)
      : this.serializeResponse(response.response);
    // ... send the serialized response to the client
  }
}
import { Server } from '@nestjs/microservices';

export class GoogleCloudPubSubServer extends Server {
  listen(callback) {
    callback();
  }

  close() {}

  sendMessage(response) {
    const serializedResponse = response.err
      ? this.serializeError(response.err)
      : this.serializeResponse(response.response);
    // ... send the serialized response to the client
  }
}

Để tuỳ chỉnh hoàn toàn việc serialization, bạn có thể cung cấp một custom serializer class implement interface Serializer. Điều này cho phép bạn kiểm soát cách messages được serialized trước khi gửi qua transport layer.

custom-serializer.ts
import { Serializer, OutgoingResponse } from '@nestjs/microservices';

export class CustomSerializer implements Serializer {
  serialize(value: OutgoingResponse): OutgoingResponse {
    // Custom serialization logic
    // For example, adding metadata, transforming the payload, etc.
    return value;
  }
}
export class CustomSerializer {
  serialize(value) {
    // Custom serialization logic
    return value;
  }
}

Để sử dụng custom serializer, truyền nó vào constructor options của Server hoặc ClientProxy:

main.ts
const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  strategy: new GoogleCloudPubSubServer({
    serializer: new CustomSerializer(),
  }),
});
const app = await NestFactory.createMicroservice(AppModule, {
  strategy: new GoogleCloudPubSubServer({
    serializer: new CustomSerializer(),
  }),
});
Lưu ý

Ngoài custom serializer, bạn cũng có thể implement một custom Deserializer bằng cách implement interface Deserializer để tuỳ chỉnh cách messages được deserialized khi nhận.
