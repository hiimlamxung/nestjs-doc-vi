MQTT

MQTT (Message Queuing Telemetry Transport) là một giao thức messaging mã nguồn mở, nhẹ, được tối ưu hóa cho độ trễ thấp. Giao thức này cung cấp một cách kết nối thiết bị có khả năng mở rộng và tiết kiệm chi phí sử dụng mô hình publish/subscribe. Một hệ thống giao tiếp được xây dựng trên MQTT bao gồm publishing server, một broker và một hoặc nhiều client. Nó được thiết kế cho các thiết bị có tài nguyên hạn chế và các mạng có băng thông thấp, độ trễ cao hoặc không đáng tin cậy.

Installation #

Để bắt đầu xây dựng các microservice dựa trên MQTT, trước tiên hãy cài đặt package cần thiết:

$ npm i --save mqtt

Overview #

Để sử dụng MQTT transporter, truyền object options sau vào phương thức createMicroservice():

main
const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  transport: Transport.MQTT,
  options: {
    url: 'mqtt://localhost:1883',
  },
});
const app = await NestFactory.createMicroservice(AppModule, {
  transport: Transport.MQTT,
  options: {
    url: 'mqtt://localhost:1883',
  },
});
Gợi ý

Enum Transport được import từ package @nestjs/microservices.

Options #

Object options dành riêng cho transporter đã chọn. Transporter MQTT expose các thuộc tính được mô tả tại đây.

Client #

Giống như các microservice transporter khác, bạn có nhiều tùy chọn để tạo một instance ClientProxy cho MQTT.

Một phương pháp để tạo instance là sử dụng ClientsModule. Để tạo client instance với ClientsModule, import nó và sử dụng phương thức register() để truyền một object options có cùng các thuộc tính đã thấy ở trên trong phương thức createMicroservice(), cùng với thuộc tính name được sử dụng làm injection token. Đọc thêm về ClientsModule tại đây.

@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'MATH_SERVICE',
        transport: Transport.MQTT,
        options: {
          url: 'mqtt://localhost:1883',
        }
      },
    ]),
  ]
  ...
})

Các tùy chọn khác để tạo client (dùng ClientProxyFactory hoặc @Client()) cũng có thể được sử dụng. Bạn có thể đọc thêm về chúng tại đây.

Context #

Trong các tình huống phức tạp hơn, bạn có thể cần truy cập thông tin bổ sung về incoming request. Khi sử dụng MQTT transporter, bạn có thể truy cập object MqttContext.

@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: MqttContext) {
  console.log(`Topic: ${context.getTopic()}`);
}
@Bind(Payload(), Ctx())
@MessagePattern('notifications')
getNotifications(data, context) {
  console.log(`Topic: ${context.getTopic()}`);
}
Gợi ý

@Payload(), @Ctx() và MqttContext được import từ package @nestjs/microservices.

Để truy cập mqtt packet gốc, sử dụng phương thức getPacket() của object MqttContext, như sau:

@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: MqttContext) {
  console.log(context.getPacket());
}
@Bind(Payload(), Ctx())
@MessagePattern('notifications')
getNotifications(data, context) {
  console.log(context.getPacket());
}

Wildcards #

Một subscription có thể là một topic cụ thể, hoặc có thể bao gồm các wildcard. Có hai loại wildcard khả dụng, + và #. + là wildcard single-level, trong khi # là wildcard multi-level bao phủ nhiều level topic.

@MessagePattern('sensors/+/temperature/+')
getTemperature(@Ctx() context: MqttContext) {
  console.log(`Topic: ${context.getTopic()}`);
}
@Bind(Ctx())
@MessagePattern('sensors/+/temperature/+')
getTemperature(context) {
  console.log(`Topic: ${context.getTopic()}`);
}

Quality of Service (QoS) #

Bất kỳ subscription nào được tạo với decorator @MessagePattern hoặc @EventPattern sẽ subscribe với QoS 0. Nếu cần QoS cao hơn, có thể thiết lập toàn cục bằng cách sử dụng block subscribeOptions khi thiết lập kết nối như sau:

main
const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
  transport: Transport.MQTT,
  options: {
    url: 'mqtt://localhost:1883',
    subscribeOptions: {
      qos: 2
    },
  },
});
const app = await NestFactory.createMicroservice(AppModule, {
  transport: Transport.MQTT,
  options: {
    url: 'mqtt://localhost:1883',
    subscribeOptions: {
      qos: 2
    },
  },
});

Per-pattern QoS #

Bạn có thể ghi đè MQTT subscription QoS trên cơ sở từng pattern bằng cách cung cấp qos trong trường extras của pattern decorator. Khi không được chỉ định, giá trị subscribeOptions.qos toàn cục sẽ được sử dụng làm mặc định.

@EventPattern('critical-events', { extras: { qos: 2 } })
handleCriticalEvent(@Payload() data: any) {
  // Subscription này sử dụng QoS 2
}

@EventPattern('metrics', { extras: { qos: 0 } })
handleMetrics(@Payload() data: any) {
  // Subscription này sử dụng QoS 0
}
@Bind(Payload())
@EventPattern('critical-events', { extras: { qos: 2 } })
handleCriticalEvent(data) {
  // Subscription này sử dụng QoS 2
}

@Bind(Payload())
@EventPattern('metrics', { extras: { qos: 0 } })
handleMetrics(data) {
  // Subscription này sử dụng QoS 0
}
Gợi ý

Cấu hình QoS theo từng pattern không ảnh hưởng đến hành vi hiện tại. Khi extras.qos không được chỉ định, subscription sử dụng giá trị subscribeOptions.qos toàn cục.

Record builders #

Để cấu hình message options (điều chỉnh mức QoS, thiết lập cờ Retain hoặc DUP, hoặc thêm các thuộc tính bổ sung vào payload), bạn có thể sử dụng class MqttRecordBuilder. Ví dụ, để thiết lập QoS thành 2, sử dụng phương thức setQoS, như sau:

const userProperties = { 'x-version': '1.0.0' };
const record = new MqttRecordBuilder(':cat:')
  .setProperties({ userProperties })
  .setQoS(1)
  .build();
client.send('replace-emoji', record).subscribe(...);
Gợi ý

Class MqttRecordBuilder được export từ package @nestjs/microservices.

Và bạn cũng có thể đọc các options này ở phía server, bằng cách truy cập MqttContext.

@MessagePattern('replace-emoji')
replaceEmoji(@Payload() data: string, @Ctx() context: MqttContext): string {
  const { properties: { userProperties } } = context.getPacket();
  return userProperties['x-version'] === '1.0.0' ? '🐱' : '🐈';
}
@Bind(Payload(), Ctx())
@MessagePattern('replace-emoji')
replaceEmoji(data, context) {
  const { properties: { userProperties } } = context.getPacket();
  return userProperties['x-version'] === '1.0.0' ? '🐱' : '🐈';
}

Trong một số trường hợp bạn có thể muốn cấu hình user properties cho nhiều request, bạn có thể truyền các options này vào ClientProxyFactory.

import { Module } from '@nestjs/common';
import { ClientProxyFactory, Transport } from '@nestjs/microservices';

@Module({
  providers: [
    {
      provide: 'API_v1',
      useFactory: () =>
        ClientProxyFactory.create({
          transport: Transport.MQTT,
          options: {
            url: 'mqtt://localhost:1833',
            userProperties: { 'x-version': '1.0.0' },
          },
        }),
    },
  ],
})
export class ApiModule {}

Instance status updates #

Để nhận cập nhật theo thời gian thực về kết nối và trạng thái của driver instance bên dưới, bạn có thể subscribe vào stream status. Stream này cung cấp các cập nhật trạng thái dành riêng cho driver đã chọn. Đối với MQTT driver, stream status phát ra các sự kiện connected, disconnected, reconnecting, và closed.

this.client.status.subscribe((status: MqttStatus) => {
  console.log(status);
});
Gợi ý

Type MqttStatus được import từ package @nestjs/microservices.

Tương tự, bạn có thể subscribe vào stream status của server để nhận thông báo về trạng thái của server.

const server = app.connectMicroservice<MicroserviceOptions>(...);
server.status.subscribe((status: MqttStatus) => {
  console.log(status);
});

Listening to MQTT events #

Trong một số trường hợp, bạn có thể muốn lắng nghe các sự kiện nội bộ được phát ra bởi microservice. Ví dụ, bạn có thể lắng nghe sự kiện error để kích hoạt các thao tác bổ sung khi xảy ra lỗi. Để làm điều này, sử dụng phương thức on(), như dưới đây:

this.client.on('error', (err) => {
  console.error(err);
});

Tương tự, bạn có thể lắng nghe các sự kiện nội bộ của server:

server.on<MqttEvents>('error', (err) => {
  console.error(err);
});
Gợi ý

Type MqttEvents được import từ package @nestjs/microservices.

Underlying driver access #

Đối với các trường hợp sử dụng nâng cao hơn, bạn có thể cần truy cập vào driver instance bên dưới. Điều này có thể hữu ích cho các tình huống như đóng kết nối thủ công hoặc sử dụng các phương thức dành riêng cho driver. Tuy nhiên, hãy lưu ý rằng trong hầu hết các trường hợp, bạn không cần truy cập driver trực tiếp.

Để làm điều đó, bạn có thể sử dụng phương thức unwrap(), trả về driver instance bên dưới. Tham số generic type nên chỉ định kiểu driver instance bạn mong đợi.

const mqttClient = this.client.unwrap<import('mqtt').MqttClient>();

Tương tự, bạn có thể truy cập driver instance bên dưới của server:

const mqttClient = server.unwrap<import('mqtt').MqttClient>();
